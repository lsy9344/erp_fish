## 1. 목표

아침 회의에서 화면을 띄워 둔 채 지점별 변화와 예외를 바로 읽을 수 있도록 일일 리포트를 다음과 같이 정리한다.

1. 이미 존재하는 실제·예상 마진율 1.5%p 이상 경고를 지점별 설정값으로 전환한다.
2. `매출 분석`의 표 3개를 회의용 차트 3개로 전환한다.
3. 전일 대비 증감은 퍼센트와 증감 금액을 한 문장으로 함께 표시한다.
4. 재고 지표를 0 기준 양·음수 편차로 정의하고 데이터 분포에 맞게 축을 자동 조정한다.
5. 매장 매출 포지션에서 `전체 평균 대비`와 원시 `매출액` 표시를 제거하고, 현재 매출 구성 도넛과 전일 대비 변화를 함께 제공한다.
6. 직원 근태는 지각·조퇴·특이사항이 있는 직원만 노출하고 `품목별 판매 현황` 위로 이동한다.
7. 현재의 `일일 운영 > 아침 회의` 진입 동선은 유지한다.

## 3. 확정 계산 규칙

### 3.1 지점별 마진 차이 경고

```text
마진 차이(%p) = |실제 마진율 - 예상 마진율|
경고 = 마진 차이 >= 해당 지점의 경고 기준값
```

- 지점별 기본값은 현재 동작과 같은 `1.50%p`, DB 저장값은 `150bp`다.
- 경계값은 포함한다. 예: 기준 1.50%p에서 정확히 1.50%p 차이면 경고다.
- 미설정 지점도 1.50%p 기본값을 사용해 기존 화면의 결과가 바뀌지 않게 한다.
- 실제 또는 예상 마진율이 계산 불가이면 빨간색으로 추정하지 않고 기존 `판정 불가` 상태를 유지한다.
- 현재 전역 `marginRateBps`는 “최저 마진율” 정책이며 이번 “실제·예상 차이 허용폭”과 의미가 다르므로 재사용하지 않는다.

### 3.2 전일 대비 매출액 증감률

```text
증감액 = 선택일 정정 반영 매출 - 전일 정정 반영 매출
증감률 = 증감액 / 전일 정정 반영 매출
```

- 차트의 주 값은 증감률이며 양수와 음수를 모두 표시한다.
- 각 지점 라벨은 `+22.9% 증가 (332,500원)` 또는 `-8.4% 감소 (-120,000원)` 형식으로 표시한다.
- 선택일 매출과 전일 매출은 계산에는 유지하지만 별도 화면 열로 노출하지 않는다.
- 전일 매출 0원, 휴무, 장부 미입력, 계산 불가는 0% 막대로 만들지 않고 이유 목록에 표시한다.

### 3.3 매출 대비 재고 편차율

기존 단순 비율은 항상 양수이므로 0선 양·음수 차트와 맞지 않는다. 일일 아침 회의 카드의 지표를 다음처럼 명확히 변경한다.

```text
매출 대비 재고 편차율 = (재고금액 - 선택일 매출액) / 선택일 매출액
```

- 재고금액 < 매출액: 음수
- 재고금액 = 매출액: 0%
- 재고금액 > 매출액: 양수
- 예: 재고 30,000원, 매출 120,000원은 기존 `+25.0%`가 아니라 `-75.0%`다.
- 이 정의는 기간·월간 리포트의 일반적인 `재고금액 / 매출액`과 다르므로 일일 카드 제목을 `재고비율`로 유지하더라도 설명에 `매출 대비 재고 편차율` 공식을 노출한다.
- FIFO 재고금액 누락, 재고 수량 정정으로 금액 확정 불가, 선택일 매출 0원은 기존처럼 계산 불가다.

### 3.4 매장 매출 포지션

- `전체 평균 대비` 계산과 표시를 제거한다.
- 별도 원시 `매출액` 열을 제거한다.
- 현재 선택일 매출 비중은 도넛 조각 계산에만 사용하고, 화면 범례에는 지점명·전체 비중·전일 대비 증감률·증감액을 표시한다.
- 도넛은 `positions[].share`의 현재 매출 구성만 표현한다. 양·음수 증감액을 파이 조각 크기로 사용하지 않는다.
- 전일 대비 값은 `salesChanges`에서 같은 `storeId`의 서버 계산 결과를 결합해 사용하며 클라이언트에서 재계산하지 않는다.
- 매출 합계가 0이거나 계산 가능한 지점이 없으면 도넛을 숨기고 원인을 표시한다.

## 4. 화면 설계

### 4.1 매출 분석 카드 배치

데스크톱에서는 다음 순서로 동일한 높이의 3열 카드로 배치하고, 좁은 화면에서는 세로로 쌓는다.

1. `전일 대비 매출액 증감률`
2. `매장 매출 포지션`
3. `재고비율`

구현은 이미 설치된 shadcn `Card`, `ChartContainer`, `ChartTooltipContent`와 Recharts를 재사용한다. 신규 차트 패키지는 추가하지 않는다.

- 레이아웃: `grid gap-4 lg:grid-cols-3`
- 각 카드는 `CardHeader`, `CardTitle`, `CardDescription`, `CardContent` 전체 구성을 사용한다.
- 핵심 퍼센트와 금액은 hover 전용으로 숨기지 않고 라벨 또는 범례에 항상 보이게 한다.
- 툴팁에는 같은 수치와 계산 기준을 반복 제공한다.
- 계산 불가 지점은 차트 아래 별도 목록으로 유지한다.
- 모바일에서는 지점명이 잘리지 않도록 차트 최소 너비 또는 카드 내부 가로 스크롤을 제한적으로 사용한다.

### 4.2 양·음수 막대의 자동 축

Recharts 숫자 축은 함수형 `domain`을 지원하므로 데이터의 실제 최솟값·최댓값으로 비대칭 범위를 계산한다.

```text
최소축 = min(데이터 최솟값, 0) - 여백
최대축 = max(데이터 최댓값, 0) + 여백
```

- 0 기준선은 `ReferenceLine x={0}`으로 유지하되 차트 가운데에 고정하지 않는다.
- 값이 대부분 음수면 0선은 오른쪽으로, 대부분 양수면 왼쪽으로 이동한다.
- 한쪽 부호만 있는 경우에도 0은 도메인에 포함해 방향을 해석할 수 있게 한다.
- 모든 값이 0인 경우에만 작은 고정 범위를 사용해 축이 붕괴하지 않게 한다.
- 매출 감소는 위험 색상, 증가는 기본/긍정 차트 색상을 사용하며 색상 외에 `증가`·`감소` 문구를 함께 표시한다.

공식 Recharts 문서상 `XAxis.domain`은 `[dataMin, dataMax]`를 받는 함수형 범위를 지원하고, `ReferenceLine`은 데이터 좌표의 0선을 그릴 수 있다.

- https://recharts.github.io/en-US/api/XAxis/
- https://recharts.github.io/en-US/api/ReferenceLine/

### 4.3 매장 매출 포지션 도넛

저장소의 기존 도넛 구현 `hq-report-overview.tsx:379-503`을 재사용 패턴으로 삼는다.

- `PieChart`, `Pie`, `Cell`, `ChartTooltip`, `ChartLegend` 사용
- 조각: 계산 가능한 지점별 선택일 매출 비중
- 범례: `지점명 · 비중 · 전일 대비 증감률 (증감액)`
- 조각이 너무 많으면 작은 조각을 임의로 숨기지 않고 범례 가독성을 우선하며, 실제 지점 수를 기준으로 높이를 조정한다.
- 도넛과 동일한 데이터를 화면 밖 접근성 표로 제공한다.
- `accessibilityLayer`를 켜고 색상만으로 지점을 구분하지 않도록 지점명 범례를 유지한다.

### 4.4 직원 근태 현황

- `lateMemo`, `earlyLeaveMemo`, `specialMemo` 중 trim 후 하나라도 값이 있는 실제 직원만 DTO에 포함한다.
- 정상 직원, `직원 미연결`만 있는 직원, `명단 부족`, `근태 미입력` 합성 행은 화면으로 보내지 않는다.
- 한 직원에게 지각·조퇴·특이사항이 여러 개 있으면 한 행에 상태 배지와 메모를 모두 표시한다.
- 요약은 `이상 근태 인원`, `지각`, `조퇴`, `특이사항` 4개로 축소한다.
- 표시 대상이 없으면 `선택일에 지각·조퇴·특이사항이 없습니다.`를 표시한다.
- 섹션 순서는 `매출 분석 → 직원 근태 현황 → 품목별 판매 현황 → 마감·이상 신호 현황`으로 바꾼다.

### 4.5 회의 사용성

- 현재 `/app/reports/daily`와 `일일 운영 > 아침 회의` 동선을 유지한다.
- 회의 중 중요한 수치가 hover나 추가 클릭에만 의존하지 않게 한다.
- 데스크톱에서 3개 분석 카드를 한 줄로 비교할 수 있게 하고, 좁은 화면은 정보 손실 없이 세로 배치한다.
- 자동 새로고침, 새 회의 모드, 전체 화면 버튼은 이번 요청에 없으므로 추가하지 않는다.

## 7. 테스트 가능한 수용 기준

### 이상 신호

- [ ] 설정이 없는 지점은 1.50%p 이상에서 기존처럼 빨간 막대가 된다.
- [ ] 지점 A 기준 2.00%p, 지점 B 기준 1.00%p일 때 같은 1.50%p 차이가 A는 정상, B는 빨간 경고다.
- [ ] 정확히 기준값과 같은 차이는 경고다.
- [ ] 실제/예상 마진이 계산 불가이면 경고를 추정하지 않는다.
- [ ] 설정 화면에서 지점별 값을 저장하고 새로고침 후 같은 값이 보인다.
- [ ] 권한 없는 사용자는 설정 조회·수정을 할 수 없다.
- [ ] 변경 전후 값과 사유가 감사로그에 남는다.

### 매출 분석 차트

- [ ] 데스크톱에서 `증감률 → 매장 포지션 → 재고비율` 카드가 한 줄 3등분으로 보인다.
- [ ] 모바일에서는 같은 순서로 세로 배치되고 핵심 값이 잘리지 않는다.
- [ ] 선택일 매출과 전일 매출의 별도 표시 열이 없다.
- [ ] 각 지점의 증감률 옆에 증감액이 `+22.9% 증가 (332,500원)` 형식으로 보인다.
- [ ] 전일 매출 0원/휴무/미입력은 0% 막대가 아니라 계산 불가 이유로 보인다.
- [ ] 양·음수 값이 섞이면 0선이 보이고, 한쪽 부호가 대부분이면 데이터 범위에 맞춰 0선 위치가 이동한다.

### 재고 편차

- [ ] 재고 30,000원·매출 120,000원은 -75.0%다.
- [ ] 재고와 매출이 같으면 0.0%다.
- [ ] 재고가 매출보다 크면 양수다.
- [ ] FIFO 금액 누락·정정 불확정·매출 0원은 계산 불가 이유를 유지한다.

### 매장 포지션

- [ ] `전체 평균 대비`가 화면과 DTO에서 제거된다.
- [ ] 원시 `매출액`이 별도 열로 보이지 않는다.
- [ ] 도넛 조각 합은 계산 가능한 지점의 현재 매출 비중 100%와 일치한다.
- [ ] 범례에서 각 지점의 비중, 전일 대비 증감률, 증감액을 확인할 수 있다.
- [ ] 매출 합계 0원 또는 계산 가능한 지점 없음은 빈 도넛이 아니라 이유 문구로 보인다.

### 직원 근태와 순서

- [ ] 지각·조퇴·특이사항 중 하나 이상 있는 직원만 보인다.
- [ ] 정상 직원, 직원 미연결만 있는 직원, 명단 부족, 근태 미입력 행은 보이지 않는다.
- [ ] 예외 직원이 없으면 명시적인 정상 빈 상태가 보인다.
- [ ] 직원 근태 현황이 품목별 판매 현황보다 위에 있다.
- [ ] 직원 급여 금액과 내부 직원 ID는 응답과 DOM에 없다.

### 회귀

- [ ] `일일 운영 > 아침 회의`와 좌측 `리포트` 기본 진입 경로가 `/app/reports/daily`로 유지된다.
- [ ] 정정 반영 매출, 권한 범위 지점 필터, 품목별 판매 상위 10개 동작이 유지된다.
- [ ] Excel/CSV export의 기존 형식은 변경되지 않는다.
