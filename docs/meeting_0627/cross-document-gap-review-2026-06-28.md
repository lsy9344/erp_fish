## 1. 결론 요약

현재 `implementation-work-order-2026-06-27.md`는 6/27 미팅 메모의 주요 구현 항목을 많이 흡수했다. 특히 Ecount 자동 매핑, 상태 한글화, 품목별 수량 표시, 변경이력 상세화, 전일재고, 냉동/생물 분류, 장기재고 알림, 마진율 재검토 같은 핵심 주제는 들어가 있다.

다만 미팅 폴더의 기존 문서들과 대조하면 다음 문제가 남아 있다.

1. 6/22 정책과 6/27 요청이 충돌하는 항목이 있다. 대표적으로 `근무/인건비 노출`, `단가 수정 권한`, `전일재고 상세 범위`가 그렇다.
2. 6/27 메모에는 있지만 작업지시서에서는 별도 작업으로 충분히 쪼개지지 않은 항목이 있었다. 현재는 `품목 구분 페이지/차트`, `월별 손익계산서`, `xlsx 다운로드`, `장기재고 기준일 관리`, `전체 셀 반영 검증`, `비용 항목명/기타 메모 정리`가 보강됐고, `진수산 재고이상`은 의뢰자 직접 분석으로 제외됐다.
3. `client-review-checklist-2026-06-27.md`가 최신 6/27 결론과 맞지 않는 부분이 있다. 특히 상태 문구, 근무/인건비 입력, 품목별 이익률 명칭이 그렇다. 작업지시서에는 이를 최신화하는 WO-19를 추가했다.
4. 급여/인건비, 단가 권한, 마진율 표시 방향, 전일재고 노출, 알림 채널, 장기재고 기준일 관리 메뉴, xlsx export, xlsx 컬럼, 월별 손익 조정 항목, 기존 급여액 처리, 사업자명은 확정됐다. 남은 자료는 냉동/생물 기준표다.

## 3. 문서 간 충돌 항목

### 3.1 급여/인건비 노출 정책

- 6/22 정책 문서: 매장관리자 화면에서 `근무/인건비` 단계와 일부 요약값을 유지하는 방향이다. `laborCount`, `payrollTotal` 같은 운영 보조 합계를 허용한다.
- 6/27 메모: 급여와 사람 등록은 본사에서 하고, 매장관리자는 선택만 하며 급여/인건비 금액은 보지 않는 방향으로 읽힌다.
- 현재 작업지시서: 2026-06-28 결정에 따라 `본사만 노출/수정`, `매장관리자는 선택만`으로 보강되어 있다.

확정:

- 급여액과 인건비 합계는 본사만 볼 수 있다.
- 직원/급여 기준은 본사만 등록하고 수정한다.
- 매장관리자는 근무자를 선택만 한다.
- 지점장 응답에는 `salaryAmount`, `payrollTotal`, 개인별 급여액을 내려보내지 않는다.
- 기존 저장 데이터는 본사 화면에서만 다룬다. 과거 데이터 migration 방식은 구현 설계에서 정한다.

### 3.2 Ecount 단가 수정 권한

- 6/22 정책 문서: Ecount 원본 단가는 보존하고, 매장관리자는 ledger 적용 단가만 수정할 수 있다는 방향이다.
- 6/27 메모: 본사 화면에서 재고 수량/가격 수정 이야기가 나왔고, 본사가 조정하는 흐름이 강조된다.
- 현재 작업지시서: 2026-06-28 결정에 따라 본사만 수정 가능으로 보강되어 있다.

확정:

- Ecount 원본 단가는 절대 수정하지 않는다.
- 실제 계산에 쓰는 적용 단가는 별도 필드로 둔다.
- 적용 단가 수정 권한은 본사만 가진다.
- 단가 수정 시 Ecount 원본과 적용 단가의 차이를 감사 로그에 남긴다.

### 3.3 전일재고 상세 범위

- 6/22 문서: 전일재고 상세 팝업은 기간 기본 1개월, 필터, FIFO 상세, 입출고 흐름까지 보는 방향이다.
- 6/27 메모: 매장관리자 로그인 후 전일재고 버튼/창으로 보는 요구가 강조된다.
- 현재 작업지시서: 2026-06-28 결정에 따라 지점장 노출 범위가 확정됐다.

확정:

- 매장관리자는 품목, 수량, FIFO 기준일/lot 식별 정보만 본다.
- 금액, 단가, FIFO 금액, 원가, 마진은 보지 않는다.
- 본사는 필요 시 상세 금액과 FIFO 금액을 볼 수 있다.
- 조회 기본 기간과 필터는 WO-11 구현 중 정한다.

### 3.4 알림 채널과 장기재고 알림

- 6/22 문서: LINE 아침 보고, 다중 수신자, Telegram/카카오톡 대안이 언급되어 있다.
- 6/27 작업지시서: 장기재고 알림과 오전 요약은 있으나, 알림 채널과 수신자 정책이 충분히 들어가 있지 않다.

확정:

- 알림 채널은 LINE만 사용한다.
- Telegram, 카카오톡, 이메일은 이번 범위에서 제외한다.
- 수신자 목록은 LINE 설정에서 관리한다.

### 3.5 `품목별 이익률` 명칭 변경 범위

- 6/27 메모: `품목별 이익률 추정`을 `품목별 매출`로 바꾸자는 요청이 있다.
- 기존 리뷰 체크리스트와 일부 문서에는 기존 명칭이 남아 있다.

권장 처리:

- 사용자에게 보이는 화면 제목, 문서, export 헤더는 `품목별 매출`로 통일한다.
- 테스트명과 내부 코드명은 위험이 낮을 때만 별도 정리한다.

추천:

- 사용자에게 보이는 모든 텍스트는 `품목별 매출`로 통일한다.
- 코드 내부 함수명은 위험이 낮을 때만 별도 리팩토링한다.

### 4.2 전체 셀 반영 검증표

6/27 메모의 `전체 셀 반영이 되었는가`는 단순 UI 수정이 아니라 계산 검증 작업이다. 별도 표가 필요하다.

반영 상태:

- 작업지시서에 WO-17 `전체 셀 반영 검증표 작성`으로 추가했다.
- WO-14는 WO-17에서 본사 홈 관련 셀 범위가 정리된 뒤 착수한다.

필요한 표:

| 기준 셀 | 의미 | 원천 데이터 | 현재 화면 위치 | 현재 값 | 기대 값 | 차이 | 조치 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| AE4 | 분석 매출 | 장부 `26` 시트 `AI36+AI63+AI76` | 본사 홈 | 확인됨 | 분석 매출 | 화면 라벨 미반영 가능 | `분석 매출`로 표시 |
| AE5 | 분석 이익률 | 장부 `26` 시트 `(AE4-(AG36+AG63+AG76))/AE4` | 본사 홈 | 확인됨 | 분석 이익률 | 화면 라벨 미반영 가능 | `분석 이익률`로 표시 |
| C5 | 장부 매출 | 장부 `26` 시트 `C22+C23+C24+C36` | 본사 홈 | 확인됨 | 장부 매출 | 화면 라벨 미반영 가능 | `장부 매출`로 표시 |
| C17 | 장부 이익률 | 장부 `26` 시트 `C15/C5` | 본사 홈 | 확인됨 | 장부 이익률 | 화면 라벨 미반영 가능 | `장부 이익률`로 표시 |

주의:

- AE4, AE5, C5, C17의 의미는 확인됐다.
- 남은 일은 WO-17에서 전체 셀 반영 검증표를 만들어 다른 셀까지 누락 여부를 확인하는 것이다.
- 마진율 반전 표시는 2026-06-30 지시로 삭제한다. 홈에서는 장부 기준과 분석 기준의 원값을 구분해 보여준다.

### 4.5 월별 손익계산서 별도 스펙

기존 문서에는 월별 손익계산서가 단순 보고서가 아니라 본사 조정, 고정비, 세금, 월세, 인건비, 기타 비용과 연결된다. 2026-06-28 결정으로 월별 손익계산서는 범위에 포함한다. 2026-06-28 추가 답변으로 설명은 더 쉽게 바꾼다.

쉽게 말하면 월별 손익계산서는 한 달 동안 지점이 얼마나 벌고, 얼마나 쓰고, 얼마가 남았는지 보는 표다.

- 매출: 한 달 동안 판 금액
- 매입 원가: 팔기 위해 사온 물건값
- 인건비: 직원 급여와 근무 비용
- 고정비: 월세, 관리비처럼 매달 나가는 돈
- 기타 비용: 그 밖에 본사가 직접 넣는 비용
- 남은 금액: 매출에서 위 비용을 뺀 금액

본사 조정 항목은 추천안으로 확정한다.

월별 손익 조정 항목:

- 월세
- 관리비
- 공과금
- 세금/수수료
- 포장/소모품
- 배송/운반
- 수선/유지보수
- 기타비용
- 본사조정
- 조정사유
- 메모

별도 스펙에 들어가야 할 항목:

- 월별 기준: 영업일 기준인지 달력 월 기준인지
- 매출 기준: 실제 POS, Ecount, 매장 입력, 추정 매출 중 무엇인지
- 원가 기준: FIFO인지 평균단가인지
- 인건비: 본사 등록 기준인지 매장 입력 기준인지
- 고정비: 월세, 세금, 관리비, 기타비용
- 본사 조정: 어떤 항목을 수동 수정할 수 있는지
- 잠금/마감: 월 마감 후 수정 가능 여부
- export: xlsx는 필수, CSV는 보조 가능
- xlsx 시트: `요약`, `기간조회_RAW`, `월별손익`, `재고현황`, `품목매출`

반영 상태:

- 작업지시서 WO-15에 위 기준을 착수 전 확정 기준으로 추가했다.
- xlsx API 계약 변경과 `tests/api/report-export.spec.ts` 수정 필요도 WO-15에 추가했다.

`client-review-checklist-2026-06-27.md`는 다음 이유로 최신 상태가 아니다.

15. 월별 손익 조정 항목은 추천안으로 확정한다.

18. 진수산 재고이상은 의뢰자가 직접 분석하므로 구현 범위에서 제외한다.
