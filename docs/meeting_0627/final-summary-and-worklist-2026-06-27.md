## 1. 한 줄 요약

이번 회의의 핵심은 **이카운트 업로드와 지점 장부를 더 자동으로 연결하고, 지점장은 단순 입력만 하며, 본사는 실제 매출과 품목 기준 추정 매출/마진 차이를 비교해 재고·손실·장기 보관 문제를 빠르게 보는 것**이다.

## 2. 최종 결론

1. 지점장 화면은 계속 단순해야 한다. 급여액, 민감한 마진 차이, 원가성 정보는 지점장에게 노출하지 않는 방향이다.
2. 본사 화면은 더 강해져야 한다. 실제 매출, 품목 판매가/수량 기준 추정 매출, 두 값의 차이 금액과 비율을 본사가 비교할 수 있어야 한다.
3. 이카운트 업로드는 지점/품목 매핑을 자동 재사용하고, 미매핑 항목만 본사가 확인하는 방식이 맞다.
4. 품목별 매출/이익률은 현재 POS 품목별 실제 판매 데이터가 없으므로 **추정값**이다. 화면명을 바꾸더라도 `추정`임은 유지해야 한다.
5. 제공 장부의 `26` 시트 기준으로 `AE4`는 오른쪽 `관리자모드 - 매출 분석` 영역의 `매출` 값이다. 다만 마진율 `79.5% -> 20.5%`, 좌/우 매출의 화면 라벨, 전체 셀 반영 범위는 원본 장부와 계산식 대조 후 확정해야 한다.
6. 냉동/생물 자동 분류와 장기 보관 알림은 의뢰자가 품목 기준표와 보관 기준일을 제공해야 정확하게 만들 수 있다.
7. 2026-06-30까지 정리본을 전달하고, 2026-07부터 사용 가능한 상태를 목표로 한다. 파일럿 대상 지점과 기존 방식 병행 기간은 의뢰자 확인이 필요하다.

## 3. 회의 내용 정리

### 3.1 이카운트 업로드와 매핑

- 이카운트 엑셀을 올리면 기존에 저장한 지점 매핑을 자동 적용해야 한다.
- 모든 라인이 자동 매핑되면 중간 확인 단계를 줄일 수 있다.
- 품목도 이카운트 원본 품목명/규격을 기준으로 ERP 품목에 자동 연결해야 한다.
- 신규 품목이 나오면 업로드 과정에서 추가할 수 있어야 한다.
- 업로드 상세 화면의 상태값은 영어 대신 한글로 표시한다. 예: `committed` -> `반영됨`.

### 3.2 품목, 규격, 세분화

- 품목은 대표 품목과 세부 규격을 함께 볼 수 있어야 한다.
- 예: 제주갈치 전체와 제주갈치 30미/41미 같은 세부 규격을 같이 다룬다.
- 본사 리포트에는 품목별 판매 수량을 추가한다.
- `품목별 이익률 (추정)`은 의뢰자 표현에 맞춰 `품목별 매출` 쪽으로 바꾸되, 실제 POS 매출이 아니면 `추정` 표시는 유지한다.
- 세부 규격별 조회는 차트 또는 별도 표/페이지로 제공하는 방향이 맞다.

### 3.3 비용, 급여, 인건비

- 비용 항목은 지점 자유 입력보다 본사가 관리하는 항목을 선택하는 방식이 맞다.
- 예외 비용은 `기타 + 메모`로 처리한다.
- 식대/점심 같은 항목명은 실제 현장 표현에 맞춰 조정한다.

- 지점 직원에게 급여액을 입력시키거나 보여주지 않는다.
- 본사가 사람과 급여 기준을 등록하고, 지점장은 근무자/근무 여부를 선택하는 방식이 맞다.
- 급여 계산과 인건비 집계는 본사 권한 화면에서 처리한다.

### 3.4 재고, 손실, 전일재고

- 전일 재고를 싸게 파는 문제는 우선 `떨이/손실` 입력으로 처리한다.
- 손실 금액은 단순 원가 기준이 아니라, 팔려고 했던 가격과 실제 회수액의 차이를 기준으로 봐야 한다.
- 복잡한 전일 재고 판매가 수정 기능은 바로 넣지 말고, 실제 사용 후 필요성을 다시 판단한다.
- 지점장은 전날 장부를 수정할 수 없어야 한다.
- 단, 전날 재고는 참고용으로 볼 수 있어야 한다. 버튼을 눌러 품목명과 수량만 보는 단순 창이 적합하다.
- 본사는 재고 항목, 수량, 장부 적용 단가를 수정할 수 있어야 하지만, 원본 이카운트 정보는 보존해야 한다.
- 장기 보관 재고는 본사와 지점장 모두 알아야 한다. 생물은 3~4일 이상 변동 없이 남는 경우가 우선 논의되었다.

### 3.5 본사 홈, 매출, 마진율

- 본사 홈에는 두 종류의 매출이 필요하다.
  - 실제 매출: 현금, 카드 등 실제 돈을 센 금액
  - 추정 매출: 품목 판매가와 판매량 기준으로 계산한 금액
- 두 매출 기준의 차이를 금액과 퍼센트로 보여줘야 한다.
- 제공 장부 `장부-202605현대 (1).xlsx`의 `26` 시트를 확인한 결과, 오른쪽 `관리자모드 - 매출 분석` 영역은 다음 구조다.
  - `AE4`: 매출 `3,514,811`, 수식 `AI36+AI63+AI76`
  - `AE5`: 이익률 `28.84%`, 수식 `(AE4-(AG36+AG63+AG76))/AE4`
  - `AI36`, `AI63`, `AI76`: 냉동/생물/판매완료 구간의 `판매금액` 합계
- 같은 시트의 왼쪽 요약은 다음 구조다.
  - `C5`: 매출 `3,491,000`, 수식 `C22+C23+C24+C36`
  - `C17`: 이익률 `28.35%`, 수식 `C15/C5`
- 따라서 관리자 홈에는 기존 실제 매출 기준 값 아래에 `AE4` 기준 매출을 함께 보여주고, 옆에는 `C17`과 `AE5`처럼 두 기준의 이익률을 나란히 보여주는 방향이 자연스럽다.
- 현재 `79.5%`처럼 보이는 마진율은 의뢰자가 기대한 `20.5%`와 다르므로 계산식 확인이 필요하다. 이번 확인으로 `AE4/AE5`의 장부상 의미는 확인됐지만, 앱의 기존 마진율 표시가 어떤 기준을 쓰고 있는지는 별도 대조가 필요하다.
- 이 비교 정보는 본사만 보며, 지점장에게는 보여주지 않는 방향이다.

### 3.6 리포트, 기간조회, 손익계산서

- 일별뿐 아니라 기간별, 월별 조회가 필요하다.
- 품목별 수량, 품목별 추정 매출, 지점별 매출, 지점별 이익률, 월별 손익을 볼 수 있어야 한다.
- 남아 있는 재고를 품목별, 지점별, 전체 합계로 볼 수 있어야 한다.
- 조회 결과는 엑셀로 다운로드할 수 있어야 한다. 목적은 피벗 분석용 기초 자료 제공이다.
- 지점별 월간 손익계산서는 기존 사양을 다시 검토해야 한다. 본사가 별도 조정해서 활용할 수 있어야 한다.

### 3.7 변경 이력과 본사 수정

- 본사가 장부를 수정할 때는 사유 입력이 필요하다.
- 변경 이력에는 누가, 언제, 무엇을, 왜 바꿨는지 남아야 한다.
- 아이디만으로 작성자나 수정자를 알아보기 어려우므로, 장부와 변경 이력에는 표시명 또는 실제 작성자명을 함께 보여주는 편이 좋다.
- 이력 삭제 기능은 만들지 않는 방향이 안전하다.
- 현재 변경 이력 상세보기는 무엇이 바뀌었는지 더 쉽게 보이도록 개선해야 한다.

| P2-8 | 장기 보관 재고 알림 | FIFO 입고일 또는 전일재고 이력을 기준으로 오래 남은 재고를 표시한다 | 생물/냉동별 기준일 확정 필요 |
| P2-9 | 현재 재고 조회 강화 | 품목별, 지점별, 전체 합계 기준으로 남은 재고를 볼 수 있게 한다 | 전일재고, 당일재고, FIFO lot 의미 구분 |
| P2-10 | 손실 금액 기준 정리 | 계획 판매가와 실제 회수액 차이를 손실 기준으로 표시한다 | 전일재고 떨이와 당일 매입 손실 구분 |

| 번호 | 작업 | 변경 방식 | 리스크 |

| P3-2 | 기간별 조회 | 기간, 지점, 품목, 세부 규격 기준으로 조회한다 | 데이터량과 필터 UX 설계 필요 |

| P3-5 | 품목 세분화 분석 화면 | 대표 품목과 세부 규격을 함께 볼 수 있는 표/차트를 추가한다 | 이카운트 규격과 ERP 품목 master 관계 정리 필요 |

## 5. 애매하거나 불분명한 부분

| 항목 | 왜 애매한가 | 현재 판단 |
|---|---|---|
| `품목묭도 자동화` | 메모상 오타로 보이며, 품목명/품목 용도/품목 등록 중 무엇인지 불명확 | 전사본 기준으로는 품목명·규격 매핑 자동화에 가까움 |
| `모두 적용` 버튼 | 어느 화면에서 어떤 값을 모두 적용할지 범위가 없음 | 품목 alias 일괄 적용 기능으로 추정 |
| `79.5% -> 20.5%` | 장부 `26` 시트의 `AE5`는 `28.84%`, `C17`은 `28.35%`로 확인됐으나 앱에서 `79.5%`가 나온 기준은 아직 불명확 | 공식 확정 전 수정 금지 |
| 좌/우 매출 | `26` 시트 기준 왼쪽은 `C5/C17`, 오른쪽은 `AE4/AE5`로 위치가 확인됨. 다만 화면에서 부를 업무명은 아직 정해야 함 | 예: `실제 매출/실제 이익률`, `품목 기준 매출/품목 기준 이익률` 등 라벨 확정 필요 |
| `AE4` | 셀 의미는 확인됨. `26` 시트 오른쪽 `관리자모드 - 매출 분석` 영역의 매출이며, 수식은 `AI36+AI63+AI76` | 관리자 홈 표시 라벨과 기존 매출 아래 배치 방식 확정 필요 |
| 지점장 노출 범위 | 본사 분석값을 보여주면 지점장이 숫자를 맞출 수 있다는 우려가 있음 | 민감 차이/마진/원가 정보는 본사 전용 |
| 지점별 월간 손익계산서 | 기존 사양과 이번 요청의 차이가 큼 | 별도 사양 재작성 필요 |
| 변경이력 상세보기 | “수정하기”만 있고 정확한 불편 지점이 없음 | 필드명/전후 비교/링크 개선부터 시작 |
| 회사명/사업자명 | 메모에는 `도원에스디`, 전사본에는 `도원` 흐름이 있음 | 화면 표시명과 사업자명을 분리해 확인 |

## 6. 불가능하거나 위험한 부분

1. POS 품목별 실제 판매 데이터 없이 `확정 품목별 매출`, `확정 품목별 이익률`을 만들 수 없다. 현재 가능한 것은 `전일재고 + 매입 - 당일재고`와 판매 예정가 기준의 추정값이다.
2. 마진율 공식을 확인하지 않고 `79.5%`를 `20.5%`로 바꾸면 다른 화면의 이익률과 알림 기준이 깨질 수 있다.
3. `AE4` 의미는 확인됐지만, 원본 장부와 전체 셀 매핑표 없이 `장부의 모든 셀 반영`은 완료 기준을 만들 수 없다.
4. 냉동/생물 분류를 품목명 문자열만으로 자동화하면 오분류가 생길 수 있다.
5. 전일 재고 가격을 자유롭게 수정하게 하면 원본 보존, FIFO 원가, 변경 이력, 손실 계산이 흔들릴 수 있다.
6. 미저장 경고를 단순히 꺼버리면 실제 입력값 유실이 생길 수 있다. 단계별 변경 감지를 고쳐야 한다.

## 7. 의뢰자에게 요청할 것

1. 최신 이카운트 샘플 xlsx.
2. 이카운트 거래처명과 ERP 지점명 매핑표.
3. 이카운트 품목명/규격과 ERP 표준품목 매핑표.
4. `모두 적용`이 적용되어야 하는 예시와 적용되면 안 되는 예시.
5. 전체 셀 반영 재검수용 셀 매핑표. `AE4/AE5`는 이번에 확인됐으나, 모든 셀 반영 여부는 별도 매핑표가 필요하다.
6. `C5/C17`과 `AE4/AE5`를 홈에서 어떤 이름으로 보여줄지, 기존 매출 아래에 어떤 순서로 배치할지에 대한 화면 표시 라벨.
7. 기존 장부에서 쓰는 마진율 공식과 예시 계산.
8. 진수산 `재고이상`이 나온 날짜, 품목, 화면 캡처, 기대값.
9. 냉동/생물 기준 품목표.
10. 장기 보관 재고 알림 기준일. 예: 생물 3일, 냉동 14일처럼 품목군별 기준.
11. 지점장에게 숨겨야 할 값 목록. 예: 급여액, 원가, 마진 차이, 추정 매출 차이.
12. 기간조회와 엑셀 다운로드에 필요한 컬럼 목록.
13. 지점별 월간 손익계산서에서 본사가 조정할 항목 목록.
14. 2026-07 파일럿 지점명과 병행 테스트 방식.
15. 화면 표시명과 사업자명 최종 표기. 예: 화면은 `도원`, 사업자명은 `도원에스디`, 또는 기존 `ERP Fish` 유지 여부.

## 8. 권장 실행 순서

1. **확정 회의 전 준비**
   - 원본 장부, 이카운트 샘플, 품목/지점 매핑표, 냉동/생물 기준표를 요청한다.
   - 마진율과 좌/우 매출 정의를 먼저 확정한다.

2. **1차 빠른 수정**
   - 이카운트 상태 한글화.
   - 본사 조회 모드의 7단계 탭 이동 수정.
   - 불필요한 미저장 경고 수정.
   - 품목별 리포트 수량 추가와 제목 정리.
   - 변경 이력 상세보기 1차 개선.

3. **2차 기준 데이터 정비**
   - 지점/품목 alias 자동 매핑.
   - 신규 품목 추가 흐름.
   - 냉동/생물 master 반영.
   - 급여/인건비 master 정리.

4. **3차 본사 분석 기능**
   - 실제 매출과 추정 매출을 함께 표시.
   - 차이 금액/차이율 표시.
   - 기간조회, 엑셀 다운로드, 월간 손익계산서 확장.

5. **파일럿**
   - 2026-07부터 한 지점에서 기존 장부와 ERP를 병행하는 것을 목표로 한다.
   - 병행 기간은 의뢰자와 확정한다.
   - 매일 실제 장부와 ERP 숫자를 비교해 마진율, 재고, 품목 수량, 손실, 알림 기준을 조정한다.
