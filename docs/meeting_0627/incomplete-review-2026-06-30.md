## 결론

6/27 회의 메모 20개 항목은 대부분 작업지시서나 코드에 반영됐다. 지금 남은 것은 "요구가 문서에 없는 상태"라기보다, 아래 세 가지다.

1. 실제 코드가 최신 결정과 완전히 맞는지 검증해야 하는 항목

3. 기준자료 반영은 되었지만 최종 용어와 분류 규칙을 검수해야 하는 항목

## 우선 미완료로 봐야 할 항목

### 1. 월별 손익의 `조정사유`와 `메모` 분리

월별 손익 계획 문서에는 `조정사유`와 `메모`를 분리하라고 되어 있다. 하지만 현재 `src/features/reports/monthly-profit-loss.ts`는 본사조정의 조정사유를 `expense.memo`에서 가져오고 있다.

필요 조치:

- `HeadquartersExpense` 또는 입력 흐름에서 `adjustmentReason`과 `memo`를 분리할지 확정한다.
- 월별 손익 builder가 `adjustmentReason`은 조정사유로, `memo`는 메모로만 쓰게 고친다.
- `tests/unit/monthly-profit-loss.test.mjs`에 이 둘이 섞이지 않는 테스트를 추가한다.

### 2. 월별 xlsx의 `품목매출` 시트가 기간 합산이 아님

회의에서는 기간별 조회와 xlsx 피벗 분석이 중요하다고 나왔다. 최신 계획도 `품목매출`은 조회 기간 합산이어야 한다고 정리했다. 하지만 현재 export route는 월 마지막 날의 일별 리포트 `productProfitability`를 대표값으로 사용한다.

필요 조치:

- 월 시작일과 종료일을 받는 품목매출 기간 집계 쿼리를 만든다.
- `품목매출` 시트가 마지막 하루가 아니라 조회 기간 전체를 기준으로 나오게 한다.
- 테스트에서 `startDate`, `endDate`, `storeId` 필터가 시트에 반영되는지 확인한다.

### 3. 냉동/생물 기준 검수

음성 내용에서는 냉동/생물 구분이 문자열 추측으로 끝나면 안 되고, 의뢰자가 가진 기초 자료를 기준으로 해야 한다고 정리됐다. 기준자료(`냉동_생물_자료.xlsx`) 도착으로 최종 용어는 `냉동/생물`로 확정했고 코드/문서도 이 용어로 통일한다. 남은 일은 자료 부재가 아니라 분류 규칙 검수다.

필요 조치:

- 최종 용어는 `냉동/생물`로 확정(`냉동/활어` 표현은 실행 문서/코드에서 제거).
- 현재 코드의 `냉)` 접두 분류 규칙이 제공 자료와 실제 운영 품목을 충분히 덮는지 확인한다.
- 품목명, 규격, 표준품목, 분류, 장기재고 기준일을 기준표와 화면/export 용어에 맞춘다.
- 품목 분류 자체가 애매한 신규 품목은 `기준 미정`으로 둔다.
- 장기재고 기준일이 없는 품목군은 `기준 확인 필요`로 표시하고 LINE 알림 대상에서 제외한다.

### 4. 장부 전체 셀 반영 검증표의 빈칸

`AE4/AE5`, `C5/C17`은 확인됐지만, 하위 셀과 추가 주요 셀은 아직 `앱 확인 필요` 상태다. 본사 홈의 이중 매출/마진 표시가 이미 코드에 일부 들어갔더라도, 원본 장부와 실제 화면값 대조는 끝났다고 보기 어렵다.

필요 조치:

- `C15`, `AG36/AG63/AG76`, `AI36/AI63/AI76`, `C22/C23/C24`, `C36`의 원본 값과 앱 표시값을 채운다.
- 앱에 반영하지 않을 셀은 제외 사유를 적는다.
- 본사 홈, 리포트, 월별 xlsx 중 어느 화면에서 쓰는 값인지 분리한다.

### 5. 고객 검수 체크리스트가 아직 전부 미체크

`client-review-checklist-2026-06-28.md`는 최신 정책으로 잘 정리되어 있지만, 체크박스는 모두 비어 있다. 코드 구현 흔적이 있어도 검수 완료로 표시된 것은 아니다.

필요 조치:

- 지점장 민감정보 미노출, 이카운트 상태 한글화, 전날 재고 보기, 장기재고 기준일, xlsx 5시트, 본사 홈 이중 매출/마진을 실제 계정으로 확인한다.
- 통과한 항목만 체크한다.

## 코드에 반영 흔적이 있는 항목

아래 항목은 문서 작성 시점에는 미완료였지만 현재 코드에는 구현 또는 테스트 흔적이 있다. 그래도 최종 검수는 필요하다.

- 이카운트 상태 한글화: `반영 가능`, `반영됨`, `매핑 필요`
- 이카운트 원본 단가 보존과 적용 단가 분리
- 지점장 응답에서 급여액, 원가, 마진 관련 민감값 제거
- 전날 재고 보기 모달
- 본사 홈의 장부 매출/분석 매출, 장부 이익률/분석 이익률 표시
- 장기재고 기준일 관리 메뉴와 설정 저장
- xlsx export와 월별 xlsx 5시트
- 품목별 판매 현황 `(추정)` 제목과 수량 표시

## 구현 범위에서 제외된 항목

- `진수산 재고이상`: 의뢰자가 직접 분석하기로 정리되어 구현 범위에서 제외됐다.
- `79.5% -> 20.5%` 마진율 반전: 최신 문서에서는 삭제됐다. 대신 장부 기준과 분석 기준의 원값 및 차이를 보여준다.
- 사진/AI 비전/사진 검색: 음성 후반에 나온 고도화 아이디어지만 현재 ERP 필수 범위는 아니다.

## 권장 처리 순서

1. `monthly-profit-loss.ts`의 조정사유/메모 분리부터 고친다.

3. 냉동/생물 기준표를 확정해 장기재고 기준과 자동 분류를 마무리한다.
4. `ledger-cell-mapping-review-2026-06-28.md`의 앱 확인 필요 셀을 채운다.
