# 의뢰자 검수 체크리스트 (2026-06-28 최신)

- 작성일: 2026-06-28
- 대체 대상: `docs/meeting/client-review-checklist-2026-06-27.md` (6/27 이후 결정과 일부 불일치 → superseded)
- 기준: `implementation-work-order-2026-06-27.md`, `cross-document-gap-review-2026-06-28.md`의 2026-06-28 확정 정책

이 체크리스트는 6/27 이후 확정된 권한/라벨/노출 정책에 맞춰 다시 작성했다. 구버전 체크리스트의 `commit 가능`, 지점장 급여 입력, `품목별 이익률` 같은 표현은 검수 항목에서 제거했다.

## 1. 권한·민감정보 (가장 먼저 확인)

- [ ] 지점장 계정으로 로그인 시 **급여액·인건비 합계·개인별 급여**가 화면에 보이지 않는다.
- [ ] 지점장 계정 네트워크 응답(API/Server Action)에도 급여액·원가·마진·매출 차이 금액이 없다. (UI 숨김만으로 끝내지 않는다.)
- [ ] 지점장은 근무자를 **선택만** 하고, 급여 금액을 입력하지 않는다.
- [ ] 본사 계정은 급여 기준 등록/수정과 인건비 집계를 볼 수 있다.
- [ ] 재고 수량·장부 적용 단가는 **본사만** 수정한다. 지점장 수정 요청은 서버에서 거부된다.
- [ ] 원본 이카운트 단가(`sourceUnitPrice`)는 보존되고, 본사 적용 단가 보정은 사유와 함께 이력에 남는다.

## 2. 본사 홈 매출/마진 표시 (WO-14)

- [ ] 본사 홈에 `장부 매출`/`장부 이익률`(C5/C17 기준)과 `분석 매출`/`분석 이익률`(AE4/AE5 기준)이 구분되어 보인다.
- [ ] 본사 홈 마진율이 `100% - 현재 표시값`으로 반전 표시된다. (예: 80% → 20%)
- [ ] 반전 표시는 화면 표시 전용이며, 리포트·이상 신호의 원 계산값은 바뀌지 않는다.
- [ ] `AE4/AE5`, `C5/C17` 값과 의미가 확인 완료 기준으로 문서/테스트에 반영되어 있다.

## 3. 이카운트 업로드 (WO-01, WO-08, WO-09)

- [ ] 상세 화면에서 `COMMITTED`/`READY`/`MAPPING_REQUIRED` 같은 영어 raw 상태가 보이지 않는다.
- [ ] 상태 라벨이 `반영 가능`(READY), `반영됨`(COMMITTED), `매핑 필요`(MAPPING_REQUIRED), `취소됨`(VOIDED), `오류`(FAILED)로 통일되어 있다.
- [ ] 배치 목록과 상세 라인의 같은 상태가 같은 한글 라벨로 보인다.
- [ ] 업로드 시 지점명/품목명이 자동 등록 또는 자동 매핑되고, 같은 원본 키는 다음 업로드에서 자동 매핑된다.
- [ ] 신규 품목은 본사 확인 후 등록되며, 냉동/활어 분류는 기준표 도착 전까지 **수동 선택(`분류 확인 필요`)**으로 처리된다.
- [ ] `품목묭도`, `모두 적용` 같은 별도 UI는 남아 있지 않다.

## 4. 지점장 입력 7단계 (WO-03, WO-11, WO-18)

- [ ] 진입 직후 아무 값도 바꾸지 않고 이동하면 미저장 경고가 뜨지 않는다.
- [ ] 값을 바꾸고 저장하지 않은 채 이동하면 미저장 경고가 뜬다.
- [ ] 재고 필수 수량 미입력은 저장 시 field error로 안내되고, 단순 이동 경고로 포장되지 않는다.
- [ ] 비용 입력은 본사 표준 항목 선택이 기본이고, 예외는 `기타 + 메모`로 남긴다.
- [ ] 재고 화면 상단 `전날 재고 보기`에서 품목명·규격·수량·FIFO 기준일/lot만 보이고, 금액·단가·원가·마진은 보이지 않는다.
- [ ] 전날 장부 수정 링크는 제공되지 않는다.

## 5. 리포트·검토 (WO-04, WO-15, WO-16)

- [ ] 품목별 리포트 제목이 `품목별 판매 현황 (추정)`(또는 `품목별 매출`)으로 표시된다. (`품목별 이익률` 단독 강조 아님)
- [ ] 품목별 표에 추정 판매 수량이 보이고, `추정` 표현이 빠지지 않는다.
- [ ] 원가·마진 열이 있는 품목 리포트는 본사 권한에서만 보인다.
- [ ] 월별 손익계산서가 `매출 - 매입 원가 - 인건비 - 고정비 - 기타 비용 = 남은 금액` 구조로 보인다.
- [ ] xlsx 다운로드가 제공되고, `요약`/`기간조회_RAW`/`월별손익`/`재고현황`/`품목매출` 시트로 나뉜다.
- [ ] export 실행이 감사 로그에 남는다(필터·기간·지점·row 수·포맷).

## 6. 장기재고·알림 (WO-13)

- [ ] 본사 왼쪽 네비게이션에 `장기재고 기준일` 관리 메뉴가 있다.
- [ ] 본사가 기준일을 등록/수정하면 장기 재고 판단에 반영된다.
- [ ] 기준일이 코드 하드코딩이 아니라 설정/master로 관리된다.
- [ ] 기준이 없는 품목은 `기준 확인 필요`로 표시되어 잘못 알림되지 않는다.
- [ ] 지점장은 품목명·수량·경과일만, 본사는 지점·품목·수량·경과일·원가 영향·적용 기준일을 본다.
- [ ] 장기재고/아침 요약 알림은 **LINE으로만** 발송된다.

## 7. 변경 이력·작성자 (WO-05, WO-06)

- [ ] 변경 이력에서 변경 필드명이 한글로, 전/후 값과 사유가 함께 보인다.
- [ ] 장부 관련 이력에 장부 상세 링크가 있다.
- [ ] 작성자/수정자가 `name → email → id` 순서로 사람이 알아볼 수 있게 보인다.
- [ ] 최초 작성자와 본사 수정자가 구분된다.
- [ ] 이력 삭제 기능은 없다.

## 8. 사업자명 (WO-07)

- [ ] 로그인·사이드바·메타데이터의 화면 표시명이 `도원에스디`로 보인다.

## 9. 남은 의뢰자 확인 항목

- [ ] 냉동/활어 기준 품목표 (최종 용어가 `냉동/활어`인지 `냉동/생물`인지 포함). WO-13 자동 분류와 장기재고 기준 연결에 필요.

## 10. 주의해서 말할 부분

- `추정`이라고 표시된 값은 POS 확정 매출이 아니다.
- 마진율 반전 표시는 화면 표시 전용이며 저장값을 바꾸지 않는다.
- 지점장 화면에는 급여액·원가·마진·매출 차이 금액이 보이지 않는다.
- 이카운트 원본 단가·원본 품목명은 덮어쓰지 않는다.
