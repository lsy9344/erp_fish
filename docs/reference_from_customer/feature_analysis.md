# 고객 제공 엑셀 기반 기능분석 문서

작성일: 2026-05-27

## 1. 분석 대상

`docs/reference_from_customer` 폴더의 엑셀 파일 2개를 기준으로 분석했다.

| 파일 | 시트 수 | 주요 역할 |
| --- | ---: | --- |
| `total_sales.xlsx` | 14 | 전 지점 장기 매출 데이터베이스, 기간/지점별 분석, 월별 추이, 보고 양식, 직원 목록 |
| `장부-202605현대 (1).xlsx` | 33 | 현대점 2026년 5월 일일 장부, 이월 재고, 근무표, 일별 매출/재고/매입/손실 마감 |

## 2. 핵심 결론

현재 엑셀은 단순 매출 입력표가 아니라 `일일 장부 마감 -> 재고 이월 -> 월간 근무/영업 집계 -> 본사 분석 리포트`까지 이어지는 운영 시스템 역할을 한다. 웹 ERP로 전환할 때는 화면을 엑셀처럼 복제하기보다, 엑셀의 계산 구조를 다음 업무 기능으로 분리해야 한다.

1. 지점별 일일 마감 입력
2. 현금/카드/기타 매출 정산
3. 냉동/생물 상품 재고 입력 및 이월
4. 매입 내역 입력 또는 외부 매입 데이터 연동
5. 상품별 판매원가/판매금액/이익률 계산
6. 떨이/폐기 손실 기록
7. 근무자/근무일수/인당 생산성 집계
8. 본사 관리자용 일별/월별/기간별 분석
9. 이상치 알림 및 검토 메모 관리

엑셀 안에는 `#REF!`, `#DIV/0!`, `#VALUE!` 오류가 다수 남아 있다. 특히 평균재고 관련 참조가 깨져 있어, 신규 시스템에서는 수식 복원이 아니라 데이터 모델과 계산식을 백엔드 로직으로 재정의해야 한다.

## 3. 엑셀별 기능 해석

### 3.1 `total_sales.xlsx`

#### `입력` 시트

전 지점 일별 마감 결과를 누적하는 원장이다.

확인된 컬럼:

| 구분 | 컬럼 |
| --- | --- |
| 기준 정보 | 일자, 요일, 매장 |
| 매출/정산 | 매출, 현금, 카드, 매출이익, 이익률, 일지출, 영업이익 |
| 생산성 | 인당생산성, 근무인원 |
| 베스트 상품 | 최고매출품목, 매출액 |
| 인력 | 팀장 2칸, 팀원 11칸 |
| 검증 | 매출차액 |

데이터 특성:

| 항목 | 값 |
| --- | ---: |
| 데이터 행 수 | 2,772행 |
| 기간 | 2025-01-01 ~ 2026-01-31 |
| 주요 매장 | 불광, 못골, 안양, 인헌, 구로, 강서, 현대 |
| `현금`/`카드` 공란 | 각 877건 |
| `최고매출품목` 공란 | 969건 |
| `매출액` 공란 | 860건 |
| `매출차액` 공란 | 2,772건 전체 |

`현금`, `카드`, `최고매출품목`, `매출차액`은 최근 장부에서는 관리되고 있으나 장기 누적 원장에는 누락된 구간이 있다. 이관 시 필수값 처리 여부를 기간별로 달리 적용해야 한다.

#### 분석/보고 시트

`분석`, `보고 양식`, `보고양식1`, `매장 별(년도)`, `매장 별(달)` 시트는 `입력` 원장을 기준으로 `SUMIFS`, `AVERAGEIFS`, `IFERROR`를 사용해 지점별 성과를 계산한다.

주요 분석 항목:

| 기능 | 설명 |
| --- | --- |
| 기간 비교 | 과거 기간과 현재 기간을 지정해 매출, 매출이익, 이익률 증감을 비교 |
| 지점 비교 | 현대, 강서, 인헌, 불광, 구로, 안양, 못골 등 매장별 집계 |
| 인력 효율 | 평균 근무인원과 인당생산성 비교 |
| 재고 효율 | 평균재고, 평균매출, 매출대비 재고비율 계산 |
| 보고서 출력 | 본사용 보고 양식 형태로 매출액/이익률/생산성/재고비율 표시 |

주의점:

평균재고 관련 참조가 `입력!#REF!`로 깨져 있다. 시스템 전환 시 재고 데이터는 `입력` 시트의 단일 컬럼이 아니라, 일일 장부의 상품별 재고/이월 데이터에서 별도 집계해야 한다.

#### 월별 추이 시트

`매출`, `매출이익`, `이익률`, `인당생산성`, `평균 재고`, `Sheet3`는 월 단위 추이와 차트의 원천이다.

필요 기능:

| 기능 | 설명 |
| --- | --- |
| 월별 지표 집계 | 지점별 월 매출, 매출이익, 이익률, 생산성 |
| 월별 점유율 | 월별 매장 매출 점유율 차트 |
| 일자별 추이 | 특정 월의 일별 매출 추이 |
| 설날 비교 | 명절 기간 매출 비교 |

#### 직원이름 시트

직원 마스터 역할이다. 시트에는 "직원 추가될 때마다 아래에 추가로 적어주세요"라는 안내가 있다.

필요 기능:

| 기능 | 설명 |
| --- | --- |
| 직원 등록 | 이름, 소속 매장, 직책, 활성 여부 관리 |
| 근무 입력 참조 | 일일 장부 근무자 선택 목록으로 사용 |
| 생산성 계산 | 일별 근무자 수와 매출을 연결해 인당 생산성 산출 |

### 3.2 `장부-202605현대 (1).xlsx`

#### 시트 구조

| 시트 | 역할 |
| --- | --- |
| `근무표` | 1~31일 일일 장부를 모아 월간 근무일수와 영업자료를 조회 |
| `이월` | 전월 또는 전일 재고 이월 기준 |
| `1` ~ `31` | 일별 장부. 하루 단위 매출, 재고, 매입, 손실, 근무자 입력 |

2026년 5월 현대점 장부 기준으로 31개 일자 시트 중 매출이 입력된 일자는 22일이며, 입력은 1일부터 26일까지 분포한다.

확인된 5월 현대점 입력 요약:

| 항목 | 값 |
| --- | ---: |
| 매출 입력 일수 | 22일 |
| 입력된 총 매출 | 74,977,000 |
| 입력된 총 매입 | 54,041,000 |
| 입력된 총 비용 | 2,518,000 |
| 첫 입력일 | 2026-05-01 |
| 마지막 입력일 | 2026-05-26 |

이 수치는 제공 엑셀에서 읽은 현황 요약이며, 마감 확정 값으로 쓰려면 원장 이관 후 재계산 검증이 필요하다.

#### 일일 장부 입력 구조

일별 시트는 다음 영역으로 나뉜다.

| 영역 | 주요 셀/컬럼 | 기능 |
| --- | --- | --- |
| 헤더 | 날짜, 지점, 전달 사항 | 장부 기준일, 지점, 본사 검토 메모 |
| 요약 | 매출, 매입, 전일재고, 당일재고, 매출원가, 매출이익, 이익률, 영업이익 | 일마감 핵심 지표 |
| 결제 | 현금, 카드, 전일 이월금액 | 매출 정산 |
| 비용 | 비용내역, 금액, 비용합계 | 일 지출 관리 |
| 냉동 재고 | 품목, 규격, 단가, 재고량, 재고금액, 박스당 판매단가, 30%단가 | 냉동 상품 재고/단가 |
| 생물 재고 | 품목, 규격, 단가, 재고량, 재고금액, 박스당 판매단가, 30%단가 | 생물 상품 재고/단가 |
| 떨이/폐기 | 품목, 규격, 처리방법, 손실액, 이익률 | 손실 기록 |
| 매입 | 품목명(규격), 수량, 단가, 공급가액, 변환 품목규격 | 당일 매입 입력 |
| 관리자 분석 | 품목규격, 전재고, 매입, 판매원가, 이익률, 판매금액, 판매량 | 상품별 판매/원가/마진 계산 |
| 근무자 | 근무자 목록, 근무자 수 | 일별 근무 집계 |

#### 주요 계산식

| 지표 | 엑셀 계산 구조 | 시스템 전환 방향 |
| --- | --- | --- |
| 매출 | 현금 + 카드 + 전일 이월금액 + 비용성 조정값 | 결제수단별 매출 합계와 조정 항목을 분리 저장 |
| 매입 | 매입 임시 작성칸 또는 매입합계 참조 | 매입 전표/매입 라인 합계로 계산 |
| 전일재고 | 전일 또는 이월 시트의 당일재고 참조 | 전일 마감 재고 스냅샷 자동 이월 |
| 당일재고 | 냉동/생물 재고금액 합계 | 상품별 재고금액 합계 |
| 매출원가 | 매입 + 전일재고 - 당일재고 | 재고수불 기반으로 계산 |
| 매출이익 | 매출 - 매출원가 | 자동 계산 |
| 이익률 | 매출이익 / 매출 | 0원/미입력 예외 처리 |
| 영업이익 | 매출이익 - 비용합계 | 비용 입력 후 자동 계산 |
| 인당생산성 | 매출 / 근무자 수 | 근무자 마감 기준 자동 계산 |
| 상품 판매량 | 판매원가 / 매입단가 | 단가 기준 판매량 추정 또는 POS 연동 시 실판매량으로 대체 |
| 최고매출품목 | 상품별 판매금액 최대값 | 상품별 판매금액 정렬로 산출 |

#### 검증/차이 관리

일일 장부에는 `차이`와 `매출차액`이 별도로 존재한다.

| 항목 | 의미 |
| --- | --- |
| 매출 차이 | 요약 매출과 관리자모드 상품별 판매금액 합계 간 차이 |
| 이익률 차이 | 장부 이익률과 상품별 분석 이익률 간 차이 |
| 재고 비교 | 전재고 + 매입 - 판매원가와 실제 재고의 차이 |

시스템에서는 다음 검증이 필요하다.

| 검증 | 조건 |
| --- | --- |
| 매출 정합성 | 현금 + 카드 + 기타 매출/조정 = 총매출 |
| 원가 정합성 | 전일재고 + 매입 - 당일재고 = 매출원가 |
| 상품 분석 정합성 | 상품별 판매금액 합계와 총매출의 차이 허용범위 관리 |
| 재고 이월 정합성 | 전일 마감 재고가 다음 영업일 전일재고로 자동 반영 |
| 손실 반영 | 떨이/폐기 손실액이 이익률/매출차이 계산에 반영 |

#### 본사 전달 사항

일별 장부에는 본사가 남긴 검토 메모가 있다.

확인된 예:

| 일자 | 전달 사항 |
| ---: | --- |
| 1일 | 참소라, 고등어 30미 누락 수정 필요 |
| 4일 | 자반 재고 증가, 생합/황석어 판매단가 확인, 쭈꾸미 원가 이하 정리 |
| 5일 | 포크오징어M 재고 증가 |
| 6일 | 냉꽃게 잔량 정리 요청 |
| 8일 | 자반8손 과재고 |
| 15일 | 오징어M2 재고 증가 |

필요 기능:

| 기능 | 설명 |
| --- | --- |
| 본사 검토 메모 | 장부별 코멘트 작성 |
| 지점 확인 처리 | 지점장이 확인/조치 여부 체크 |
| 재검토 상태 | 미확인, 확인, 수정요청, 완료 상태 관리 |
| 알림 | 마감 후 코멘트가 있으면 지점장에게 알림 |

#### 손실 처리

일별 장부는 떨이/폐기 손실을 품목별로 기록한다. 제공 파일에서는 `폐기`, `떨이`, `페기`처럼 같은 의미의 값이 혼재한다.

필요 기능:

| 기능 | 설명 |
| --- | --- |
| 손실 유형 코드화 | 폐기, 떨이, 파손, 오입고, 신선도 저하 등 표준 코드 |
| 손실액 입력 | 정상 판매 대비 손실액을 음수 또는 양수 정책으로 통일 |
| 사유 필수화 | 손실 등록 시 사유/메모 필수 |
| 상품 연결 | 손실 품목을 상품 마스터와 연결 |
| 손실 리포트 | 품목별/매장별/기간별 손실 빈도와 금액 집계 |

## 4. 도출 기능 목록

### 4.1 권한 및 사용자

### 4.3 일일 장부 마감

| 기능 | 설명 |
| --- | --- |
| 장부 생성 | 매장+일자 기준으로 하루 장부 생성 |

## 6. 계산 로직 요구사항

| 계산 | 정의 |
| --- | --- |
| 총매출 | 현금 + 카드 + 기타매출 + 조정 매출 |
| 매출원가 | 전일재고 + 매입 - 당일재고 |
| 매출이익 | 총매출 - 매출원가 |
| 이익률 | 매출이익 / 총매출 |
| 영업이익 | 매출이익 - 비용합계 |
| 인당생산성 | 총매출 / 근무인원 |
| 재고금액 | 단가 * 재고량 |
| 30%단가 | 단가 / 0.7. 단, 실제 정책명은 확인 필요 |
| 상품별 판매금액 | 판매원가 / (1 - 이익률) 또는 실제 판매 데이터 |
| 매출차액 | 총매출 - 상품별 판매금액 합계 - 손실/조정 반영액 |
| 평균 판매단가 | `(판매단가1*판매량1 + 판매단가2*판매량2) / 총판매량` |

## 11. 개발 시 확인이 필요한 정책 질문

1. 장부의 `30%단가`는 고정 마진 30% 판매가인지, 참고 판매가인지 확인이 필요하다.
2. `전일 이월금액`이 총매출에 포함되는 정확한 회계 의미를 확인해야 한다.
3. `매출차액` 허용 기준을 금액 기준으로 볼지 비율 기준으로 볼지 정해야 한다.
4. 매입 입력 주체가 항상 본사인지, 일부 지점 입력을 허용하는지 정해야 한다.
5. 폐기/떨이 손실액을 음수로 저장할지, 양수 금액+손실유형으로 저장할지 정해야 한다.
6. 과거 `total_sales.xlsx`의 결제수단 공란 데이터를 이관 시 미입력으로 둘지 0으로 둘지 정해야 한다.
7. 상품별 판매량은 엑셀처럼 원가/단가로 추정할지, 향후 POS 판매 수량 연동을 전제로 할지 정해야 한다.
