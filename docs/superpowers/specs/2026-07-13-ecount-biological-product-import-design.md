# 이카운트 생물 품목코드 일괄 등록 설계

## 목표

`ESA009M (2).xlsx`의 `품목등록` 시트에 있는 이카운트 품목 284건을 ERP 품목마스터에 일괄 등록한다. 품목코드는 ERP 품목의 공식 외부 식별자로 보존하고, 이번 파일의 모든 품목은 `생물`로 등록한다.

## 입력 계약

- 파일 확장자: `.xlsx`
- 시트명: `품목등록`
- 헤더 행: 2행
- 필수 컬럼: `품목코드`, `품목명`, `품목구분`, `규격정보`, `사용`
- 선택 컬럼: `품목그룹1명`, `검색창내용`
- 이번 등록 규칙: `품목구분`은 원본 값과 관계없이 `생물`, `사용=YES`는 활성으로 저장
- 회사명 행과 마지막 일시 행, 빈 행은 무시한다.

## 데이터 모델

`Product`에 `externalCode` nullable 문자열 필드를 추가하고 unique 제약을 둔다. 코드가 없는 기존 품목은 계속 사용할 수 있으므로 nullable로 유지한다. 품목의 기존 중복 기준인 `name + category + spec`은 유지한다.

기존 `externalCode`가 있는 품목과 같은 코드가 들어오면 같은 품목으로 간주한다. 코드가 같은데 품목명 또는 규격이 다르면 충돌 오류로 처리한다. 코드가 없고 이름·구분·규격이 같은 기존 품목이 있으면 해당 품목에 코드를 보강한다.

## 등록 흐름

1. 본사 설정 권한 사용자가 품목마스터 화면에서 파일을 선택한다.
2. 서버 액션이 파일 형식과 용량을 검증하고 `품목등록` 시트를 파싱한다.
3. 모든 행을 검증해 미리보기 결과를 만든다. 결과에는 신규, 기존 코드 수정, 변경 없음, 오류 건수를 포함한다.
4. 오류·코드 중복·동일 코드의 속성 충돌이 하나라도 있으면 반영하지 않는다.
5. 오류가 없을 때 하나의 트랜잭션으로 신규 등록과 기존 품목 업데이트를 수행한다.
6. 각 변경은 기존 품목 감사 로그 형식으로 기록하고, 반영 후 품목마스터 경로를 재검증한다.

미리보기와 반영은 같은 파싱/검증 결과를 사용한다. 서버에 파일 내용을 장기 보관하지 않으며, 중복 업로드는 코드 기준으로 멱등 처리한다.

## 화면

품목마스터에 `품목코드` 열과 검색 필터를 추가한다. 기존 단건 추가/수정 UI는 유지한다. 업로드 결과에는 총 행 수, 신규 등록 수, 코드 보강 수, 변경 없음 수를 표시한다. 오류는 행 번호와 원인을 표시한다.

## 검증 및 테스트

- 정상 `ESA009M` 파일에서 284건을 생물로 파싱한다.
- 회사명/일시/빈 행을 제외한다.
- 코드가 같은데 품목 정보가 다른 행을 거부한다.
- 기존 코드 재업로드는 변경 없이 성공한다.
- 이름·규격이 같은 기존 품목에 코드를 보강한다.
- 필수 헤더 누락, 코드 누락, 품목명 누락, 규격 길이 초과를 거부한다.
- 품목마스터 화면의 코드 표시·검색과 업로드 결과를 E2E로 확인한다.

## 범위 제외

이번 작업에서는 `품목그룹1명`과 `검색창내용`을 별도 저장하지 않는다. 검색창내용을 외부 alias로 확장하거나 거래 장부 업로드에 코드 우선 매핑을 추가하는 작업은 별도 범위다.
