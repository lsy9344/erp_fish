# 본사 통합 리포트 차트 확장 설계

**상태:** 방향 승인, 작성 문서 검토 대기

**작성일:** 2026-07-17

**대상 사용자:** 본사 관리자, 총책임자
**대상 화면:** `/app/reports/overview`

## 1. 목표

본사 책임자가 여러 지점의 매출, 이익, 손실, 마감 상태를 짧은 시간 안에 파악하고 기존 상세 리포트에서 근거를 확인할 수 있는 통합 리포트 홈을 만든다.

성공 기준은 다음과 같다.

1. 선택 월의 전사 흐름, 지점 비교, 손익 원인, 조치 대상을 한 화면에서 순서대로 읽을 수 있다.
2. 실제값, 추정값, 누락값, 계산 불가를 서로 오해하지 않는다.
3. 차트와 표가 같은 서버 계산 결과를 사용한다.
4. 차트에서 기존 일별, 기간 비교, 월간, 재고, 장부 상세로 이동할 수 있다.
5. 새 데이터베이스 모델이나 차트 라이브러리를 추가하지 않는다.

## 2. 제품 원칙

이 설계는 `PRODUCT.md`의 다음 원칙을 따른다.

- 판단 순서: 전체 흐름 → 지점 비교 → 원인 → 조치 대상
- 예외 우선: 미마감, 손실, 급락, 계산 불가를 먼저 찾게 함
- 근거 추적: 집계값에서 장부와 정정 이력으로 이동
- 상태 정직성: 누락이나 계산 불가를 0으로 대체하지 않음
- 기존 문법 재사용: ERP Fish의 파란색, 밝은 표면, 얕은 경계, 간결한 상태 배지 유지

화면의 물리적 사용 장면은 “본사 책임자가 밝은 사무실의 데스크톱 모니터에서 아침 또는 월말 회의 전에 전체 지점 상황을 빠르게 검토하는 장면”이다. 따라서 기존 라이트 테마를 기본으로 사용하고 정보 밀도와 가독성을 우선한다.

## 3. 현재 구현 조사 결과

### 3.1 현재 리포트 구조

현재 제품에는 다음 본사 리포트가 있다.

- 일별 아침 회의: `/app/reports/daily`
- 기간별 지점 비교: `/app/reports/comparison`
- 전 지점 재고: `/app/reports/inventory`
- 월간 요약: `/app/reports/monthly`
- 본사 출고·지점 입고: `/app/reports/ecount-supply`
- 매출 검토: `/app/reports/sales-review`
- 품목 검토: `/app/reports/product-review`

사이드바의 리포트 진입점은 현재 일별 리포트로 연결된다. 새 통합 홈을 추가한 뒤 이 진입점을 `/app/reports/overview`로 바꾸고 기존 페이지는 상세 분석 화면으로 유지한다.

### 3.2 현재 차트

현재 차트는 모두 Recharts 기반 막대 차트다.

- 냉동·생물 추정 매출: `src/features/reports/components/product-category-margin-chart.tsx`
- 지점별 매출·이익률: `src/features/reports/components/store-daily-performance-chart.tsx`
- 품목별 추정 판매액·이익률: `src/features/reports/components/product-profitability-report.tsx`

공통 `ChartContainer`, `ChartTooltipContent`, `ChartLegendContent`는 `src/components/ui/chart.tsx`에 이미 있으며 `recharts` 3.8.0도 설치되어 있다. 새 시각화 의존성은 필요하지 않다.

### 3.3 사용할 수 있는 데이터

`DailyLedger`에는 지점·영업일별 총매출, 결제 금액, 근무인원과 마감 상태가 있다 (`prisma/schema.prisma:419`). 품목별 실제 POS 매출은 없으므로 품목·카테고리 매출은 재고 흐름과 판매가 계획을 사용한 추정값이다.

현재 코드에서 재사용할 수 있는 집계는 다음과 같다.

- 정정 반영 매출·FIFO 원가·이익: `src/features/reports/queries.ts`
- 월별 손익: `src/features/reports/monthly-profit-loss.ts`
- 월간 손실 유형 집계: `src/features/reports/queries.ts:2193`
- 월간 마감 상태 집계: `src/features/reports/queries.ts:2841`
- 재고와 FIFO lot: `src/features/reports/inventory-position-queries.ts`
- 이상 신호: `src/server/calculations/anomaly.ts`

## 4. 외부 시각화 연구 결론

차트는 표현하려는 관계에 맞춰 선택한다.

| 판단 목적 | 선택한 표현 | ERP Fish 적용 |
|---|---|---|
| 시간 변화 | 선 차트 | 일별 매출과 전월 같은 일 비교 |
| 범주 순위 | 0 기준 가로 막대 | 지점 성과 순위 |
| 단일 합계의 구성 | 도넛 | 손실 유형별 금액 구성 |
| 값이 만들어지는 과정 | 워터폴 | 매출에서 순이익까지의 흐름 |
| 상태 구성 비교 | 100% 누적 막대 | 본사 마감, 진행 중, 미입력, 휴무 |
| 이상 탐색 | 정렬된 목록 | 조치가 필요한 지점과 근거 |

원형·도넛 차트는 하나의 완전한 합계, 3~4개 범주, 음수가 없는 경우에만 사용한다. 지점 비교, 시계열, 손익은 원형으로 표현하지 않는다. 이 원칙은 [ONS 원형·도넛 차트 가이드](https://service-manual.ons.gov.uk/data-visualisation/chart-types/pie-and-doughnut-charts)와 [SAP Analytical Card 가이드](https://experience.sap.com/fiori-design-web/analytical-card/)에 부합한다.

막대는 0에서 시작하고 값순으로 정렬한다. 선 차트의 누락값은 0으로 연결하지 않는다. 근거는 [ONS 차트 선택 가이드](https://service-manual.ons.gov.uk/data-visualisation/chart-types/choosing-a-chart-type), [막대 차트 가이드](https://service-manual.ons.gov.uk/data-visualisation/chart-types/bar-charts), [선 차트 가이드](https://service-manual.ons.gov.uk/data-visualisation/chart-types/line-chart)를 따른다.

## 5. 승인된 정보 구조

### 5.1 화면 순서

1. 페이지 제목과 상세 리포트·내보내기 이동
2. 공통 필터와 데이터 품질 요약
3. 매출, 매출이익, 순이익, 손실, 조치 필요 요약 띠
4. 전사 매출 추세와 손실 유형 구성
5. 지점 성과 순위와 월 손익 흐름
6. 조치가 필요한 상황 목록
7. 같은 데이터의 표 보기와 기존 상세 리포트 링크

모든 숫자를 같은 크기의 카드로 반복하지 않는다. 핵심 숫자는 하나의 요약 띠로 묶고, 매출 추세를 가장 큰 시각 요소로 둔다.

### 5.2 공통 필터

공통 필터는 `조회 월 + 지점`으로 제한한다.

- 기본 월: 현재 달력월
- 기본 지점: 사용자가 접근할 수 있는 전체 지점
- 선택 월의 일별 흐름을 선 차트에 표시
- 조치 목록은 `오늘` 기준임을 제목에 명시
- 자유 기간 분석은 기존 `/app/reports/comparison`으로 연결

월 손익 정책이 달력월 기준이므로 임의 30일과 월 손익을 같은 범위로 보이는 UI는 만들지 않는다.

### 5.3 데이터 흐름

```text
URL의 month/storeId
  → 서버 페이지에서 requireReportAccess 및 지점 범위 확인
  → 통합 리포트 조회 함수
  → 기존 정정·매출·FIFO·손실·월 손익 계산 재사용
  → 차트용 읽기 전용 DTO
  → 차트와 표에 동일 DTO 전달
  → 기존 상세 리포트와 장부 근거로 이동
```

차트 컴포넌트는 집계나 업무 계산을 수행하지 않는다. 포맷과 보기 전환만 담당한다.

## 6. 1차 구현 차트

### 6.1 매출 추세 선 차트

**목적:** 선택 월의 매출 흐름과 전월 같은 일의 차이를 파악한다.

**데이터**

- 실제 총매출: 정정 반영된 `DailyLedger.totalSalesAmount`
- 비교값: 직전 달의 같은 일자
- 차원: 영업일, 지점
- 모집단: 검토 대기와 본사 마감 장부

**표현 규칙**

- 현재 월은 파란 실선, 전월은 중립색 점선
- 월 길이가 다르면 존재하는 같은 일까지만 비교
- 미입력·휴무·계산 불가는 선을 끊고 상태 건수를 표시
- 툴팁에 금액, 장부 상태, 데이터 기준 표시
- 차트 아래에 “최고 매출일”과 “직전 달 대비 변화” 한 문장 요약

### 6.2 손실 유형 구성 도넛

**목적:** 어떤 처리 유형에서 손실 금액이 발생했는지 한눈에 파악한다.

**선정 근거**

- 월간 손실은 이미 `MonthlyLossSummary.byType`으로 유형별 수량과 금액이 집계된다 (`src/features/reports/types.ts:205`, `src/features/reports/queries.ts:2193`).
- 기본 테스트 데이터의 유형은 폐기, 파손, 변질, 시식제공 네 개다 (`scripts/seed-test-data.mjs:77`).
- 손실액은 `판매가 계획 × 수량 - 실제 판매·회수액`으로 계산된다 (`src/features/losses/amount.ts:10`).

**집계 규칙**

- 분모: 판매가 계획 기준으로 금액을 계산할 수 있는 손실액 합계
- 조각: 금액 상위 3개 손실 유형 + 기타
- 유형이 4개 이하면 실제 유형을 그대로 표시
- 중앙: 계산 가능한 총 손실액
- 범례: 유형명, 금액, 비율

**데이터 품질 규칙**

- `usedPlannedPrice=false`인 항목을 0원 조각으로 넣지 않는다.
- `금액 계산 가능 건수 / 전체 손실 건수`를 함께 표시한다.
- 계산 기준이 없는 항목 수와 상세 이동 링크를 제공한다.
- 제목 또는 설명에 `판매가 계획 기준`임을 표시한다.

### 6.3 지점 성과 순위 가로 막대

**목적:** 성과가 좋은 지점과 확인이 필요한 지점을 빠르게 찾는다.

**지표 전환**

- 실제 총매출
- 매출이익
- 이익률
- 판매가 계획 기준 손실액

한 번에 하나의 지표만 표시한다. 값순으로 정렬하고 막대는 0에서 시작한다. 음수 값이 가능한 지표는 0 기준선을 가운데 두고 양·음을 분리한다.

계산할 수 없는 지점은 0 막대로 표시하지 않고 차트 아래 제외 건수와 이유를 표시한다. 막대를 선택하면 기존 지점 비교 또는 장부 상세로 이동한다.

### 6.4 월 손익 워터폴

**목적:** 매출에서 어떤 원가와 비용이 빠져 순이익이 되는지 설명한다.

**단계**

1. 매출
2. FIFO 매출원가
3. 매출이익
4. 인건비
5. 지점 귀속 고정비·기타 비용
6. 본사 조정
7. 순이익

전체 지점 조회에서는 `storeId=null`인 전사 공통 비용을 별도 단계로 표시하며 개별 지점에 배분하지 않는다. 특정 지점 조회에서는 해당 지점에 귀속된 비용만 사용한다.

FIFO 원가가 불완전하거나 매출이익을 계산할 수 없으면 `(grossProfit ?? 0)`과 같은 값으로 워터폴을 그리지 않는다. 차트를 숨기고 계산 불가 원인과 근거 이동 링크를 보여준다.

### 6.5 월간 마감 상태 100% 누적 막대

**목적:** 선택 월의 지점·일 단위 입력과 마감 진행 상태를 파악한다.

기존 상태 수를 다음 네 그룹으로 표시한다.

- 본사 마감: `closedCount`
- 진행 중: `reviewCount + inProgressCount`
- 미입력: `missingDayCount`
- 휴무: `holidayCount`

분모는 선택 범위의 전체 지점·일 수다. 비율과 건수를 함께 표시하며 미입력 구간에서 해당 일자 목록으로 이동할 수 있다.

### 6.6 조치가 필요한 상황 목록

차트가 아니라 우선순위가 있는 목록으로 유지한다.

- 이익률 급락
- 손실 증가
- 마감 지연·미입력
- 재고 차이
- 정정 미반영
- 계산 기준 누락

각 행은 문제, 대상 지점·품목군, 수치 근거, 상세 링크를 제공한다. 색상 외에 아이콘·상태 라벨·문장을 함께 사용한다.

## 7. 2차 후보

### 7.1 FIFO 재고 노화 누적 막대

`LedgerInventoryFifoLot.sourceBusinessDate`와 `remainingAmount`를 사용해 분류별 기준일 이내·초과 재고를 지점별로 비교한다. 원천일이 없는 lot과 기준 미설정 분류는 별도 상태로 표시한다.

### 7.2 지점×일 데이터 완성도 히트맵

장부 상태, 재고 입력, FIFO 근거, 판매가 계획, 계산 가능 여부를 지점×일 셀로 표시한다. 지표 자체보다 리포트 신뢰도를 확인하는 품질 화면이다.

### 7.3 재고회전율·재고일수

- 회전율 = 기간 FIFO 매출원가 ÷ 평균 일말 재고금액
- 재고일수 = 평균 재고금액 ÷ FIFO 매출원가 × 계산 가능 영업일 수

FIFO 근거와 일말 재고가 충분한 지점만 계산한다.

## 8. 제외 범위

- 지도: 위치 기반 분석 데이터와 업무 목적이 없음
- 예측 차트: 검증된 목표, 충분한 계절성 데이터, 예측 정확도 기준이 없음
- 맞춤형 위젯 배치: 단일 사용자 요구에 비해 복잡도가 큼
- 지점별 여러 원형 차트: 조각 비교가 어려움
- 품목별 확정 매출 차트: 실제 POS 품목 매출이 없음
- 새 차트 라이브러리와 데이터베이스 모델

## 9. 상태·오류 처리

| 상태 | 화면 처리 |
|---|---|
| 로딩 | 기존 shadcn `Skeleton`으로 최종 레이아웃 형태 유지 |

| 미입력 | 0으로 대체하지 않고 누락 상태와 건수 표시 |
| 계산 불가 | 차트에서 제외하거나 전체 차트를 숨기고 원인 표시 |
| 일부만 계산 가능 | `계산 가능 건수 / 전체 건수` 표시 |
| 추정값 | 제목, 범례, 툴팁, 표에 동일한 기준 라벨 표시 |
| 정정 반영 | 정정 반영값을 기본 사용하고 원본·정정 근거 연결 |
| 잘못된 월·지점 | 기존 쿼리 정규화 방식으로 안전한 기본값 또는 빈 결과 반환 |
| 권한 없음 | 업무 쿼리 실행 전 서버 권한 검사로 차단 |

## 10. 권한과 데이터 경계

- 페이지와 조회 함수는 `requireReportAccess()`를 사용한다.
- 모든 지점 집계는 `getHeadquartersStoreScope()` 결과 안에서만 수행한다.
- 조회 권한과 `EXPORT_CREATE` 권한을 분리한다.
- 지점장은 매출원가, 매출이익, 순이익, 인건비, 재고금액 같은 본사 민감 지표를 응답으로 받지 않는다.
- UI 숨김에 의존하지 않고 서버 응답 범위에서 차단한다.
- 차트와 export는 동일한 계산 기준을 사용하되 권한 검사는 각각 수행한다.

## 11. 접근성

- 차트마다 한 문장 핵심 요약을 제공한다.
- 차트와 동일한 데이터의 표 보기를 제공한다.
- Recharts `accessibilityLayer`와 shadcn `ChartContainer`를 사용한다.
- 색상만으로 실제·추정·위험을 구분하지 않고 실선·점선, 라벨, 아이콘을 함께 사용한다.
- 키보드 포커스와 터치에서 툴팁과 상세 이동을 사용할 수 있어야 한다.
- 금액과 퍼센트 단위를 축·범례·툴팁에 표시한다.
- 텍스트 대비는 WCAG 2.2 AA 4.5:1, 의미 있는 경계와 차트 요소는 가능한 3:1을 목표로 한다.

근거는 [W3C 복잡한 이미지 안내](https://www.w3.org/WAI/tutorials/images/complex/), [WCAG 색상 사용](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color), [WCAG 텍스트 대비](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum), [WCAG 비텍스트 대비](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast.html)를 따른다.

## 12. 검증 전략

### 단위 테스트

- 선택 월과 전월 같은 일 정렬
- 누락일을 0으로 바꾸지 않는 변환
- 손실 유형 금액 정렬과 `상위 3개 + 기타`
- 판매가 기준 없는 손실 항목의 별도 집계
- 마감 상태 네 그룹 합계와 전체 지점·일 수 일치
- 전체·지점별 워터폴 합계와 전사 공통 비용 처리
- FIFO 원가 불완전 시 워터폴 차단
- 차트 DTO와 표 DTO 합계 일치
- 정정 반영값과 데이터 품질 상태 유지

### E2E 테스트

- 본사 사용자가 통합 리포트 홈을 연다.
- 월·지점 필터가 모든 차트와 표에 동일하게 적용된다.
- 조회 권한 사용자는 차트를 보지만 export 권한이 없으면 다운로드하지 못한다.
- 지점장은 통합 리포트 홈에 접근하지 못한다.
- 차트에서 기존 상세 리포트와 장부로 이동한다.
- 추정·계산 불가·미입력 라벨이 보인다.
- 좁은 화면에서 가로 넘침 없이 핵심 값과 표 전환을 사용할 수 있다.

### 실행 명령

구현 계획에서 정확한 테스트 파일을 확정한 뒤 최소한 다음 검증을 실행한다.

```bash
pnpm typecheck
pnpm test:unit:file tests/unit/hq-report-overview.test.mjs
pnpm test:e2e -- tests/e2e/hq-reports.spec.ts
git diff --check
```
