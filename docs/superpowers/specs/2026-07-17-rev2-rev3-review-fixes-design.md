# rev 2·3 구현 검토 보완 설계

## 목표

`1aae219..a56b68c` 구현 검토에서 확인된 의미 불일치, 좁은 화면 레이아웃, 차트 접근성, 표시 조건, 포맷 문제를 기존 데이터 정책과 계산식을 바꾸지 않고 보완한다.

## 범위

1. 결제 차액 설명을 `총매출 - 결제 합계 - 지출 합계` 의미로 통일한다.
2. 4단계 지출 합계는 읽기 전용으로 유지하고 `원` 단위를 표시한다.
3. 대시보드 마진 셀의 실제·예상·경보·미달 금액이 열 안에서 줄바꿈되게 한다.
4. 지점 차트는 좁은 화면에서 문서 전체가 아니라 차트 영역만 가로 스크롤되게 하고, 막대 폭과 라벨 공간을 유지한다.
5. 지점 차트에 접근 가능한 이름·설명을 제공하고, 매출·실제 마진·예상 마진·1.5%p 경고를 담은 동등 데이터 표를 스크린리더에 제공한다.
6. 품목 추정 요약 4개는 차트가 표시될 때만 렌더한다.
7. 판매순위 표에 `판매수량 상위 10개` 표제와 검색 입력의 native `search` 타입을 적용한다.
8. 변경 파일의 Prettier 오류를 정리한다.
9. Task 2 운영 데이터는 읽기 전용으로 재확인할 수 있을 때 정확한 지점·장부·품목 ID와 이월 상태를 검증 기록에 남긴다. 고객 데이터는 수정하지 않는다.

## 구현 방식

### 결제·검토 문구

공통 계산 함수와 저장 의미는 유지한다. 검토 단계의 경고 라벨, 정상 설명, metric 라벨만 지출 합계를 포함하도록 바꾼다. 지출 합계 input은 기존 KRW formatter로 `2,000원`처럼 표시한다.

### 대시보드 마진

공통 셀 클리핑 정책에 예외 추상화를 추가하지 않는다. `grossMarginRate` 셀만 줄바꿈을 허용하고 미달 금액의 `whitespace-nowrap`을 제거한다. 열 리사이즈 정책은 유지한다.

### 지점 차트

Recharts 계산과 정렬은 유지한다. 차트 컨테이너에 최소 폭을 주고 바깥 래퍼에 `overflow-x-auto`를 적용해 390px에서도 막대 영역이 줄어들지 않게 한다. 차트의 접근 가능한 이름·설명과 스크린리더 전용 표는 같은 `chartData`를 사용해 실제 화면과 값이 어긋나지 않게 한다.

### 품목 리포트

요약 조건을 `showChart`로 바꾼다. 판매순위 검색과 상위 10개 계산은 그대로 두고 표제와 `type="search"`만 보완한다.

## 테스트

변경 전 실패를 확인한 뒤 최소 구현한다.

- 단위/source 계약: 결제 검토 문구, 요약의 `showChart` 조건, 차트 접근성 표 계약.
- E2E: 지출 합계 `원` 표시, 대시보드 마진 셀 비클리핑, 390px 차트 로컬 스크롤과 유효 막대 폭, 차트 동등 데이터, 표 보기 시 요약 숨김.
