# [요구사항정의서] 매장 관리 및 ERP 자동화 시스템 구축 의뢰

## 1. 프로젝트 개요
* **목적**: 원드라이브 엑셀 기반 수동 정산 프로세스를 웹 기반 맞춤형 시스템으로 전환하여 데이터 오류를 방지하고, 대표의 검토 시간을 단축하며, 생선 유통 특성에 맞는 재고 관리체계를 정립함.
* **대상 지점**: 약 7~8개 매장 (지점장 운영 체제)

---

## 2. 현행 시스템 및 문제점
* **수동 정산의 한계**: 지점장들이 퇴근 전 원드라이브 공유 엑셀에 당일 매출, 이익률을 수동 입력함. 수식 깨짐 우려로 항목 추가 등 확장성이 매우 떨어짐.
* **모니터링 리소스 과다**: 대표가 매일 아침 모든 지점의 엑셀 시트를 일일이 확인하며 영업 분석 및 오류 수정을 진행하고 있어 피로도가 높고 중요 포인트를 놓침.
* **매출 및 정산 불일치**: 장부상 기록된 매출과 카드/현금 정산 결과 간 오차(로스)가 발생하나 사후 추적이 어려움.
* **재고 관리 난항**: 유통 ERP(이카운트)를 사용 중이나, 생선 매장 특성상 박스당 마릿수가 상이하고 신선도 폐기(로스)가 많아 실재고와 장부상 재고가 일치하지 않음. 지점장들이 평균치로 대충 기록하는 경향이 있음.

---

## 3. 시스템 도입 요구 기능 정의

### ① 지점장 전용 웹 포탈 (입력 시스템)
* **기능**: 지점장별 개별 계정(ID/PW) 부여, 모바일/태블릿/PC 웹브라우저에서 최적화된 화면으로 당일 정산 데이터 입력.
* **효과**: 수식이나 포맷이 깨질 위험 원천 차단.

### ② 마감 데이터 제어 및 수정 권한 제한
* **기능**: 당일 마감 장부 입력 완료 시, 지점장이 데이터를 임의로 조작하거나 사후 수정할 수 없도록 마감 락(Lock) 기능 처리.
* **효과**: 정산 장부의 무결성 확보 및 투명한 관리.

### ③ 조건별 자동 리포트 및 문제 매장 알람
* **기능**: 매출 오차 발생, 이익률 급감 등 본사가 지정한 '특이사항 기준'에 부합하는 매장만 필터링하여 하나의 대시보드/보고서 형태로 표출.
* **효과**: 대표가 모든 매장을 전수조사할 필요 없이 문제 매장만 집중 모니터링 가능.

### ④ 재고 조정 입력 및 차이 사유 필수 기록
* **기능**: 실재고와 전산 장부 재고가 다를 때 강제로 맞추는 '재고 조정' 메뉴를 신설하고, 조정 시 **차이 사유(예: 폐기, 유통 로스, 오입고 등)**를 필수로 선택/입력하도록 제한.
* **효과**: 자주 손실이 발생하는 상품군을 추적하고 원인 파악 용이.

### ⑤ 텔레그램 연동 자동 메시지 발송
* **기능**: 매일 마감 정산이 완료되면 시스템이 데이터를 자동 분석하여, 해당 지점장의 텔레그램으로 당일 결과 및 주의사항(로스 내역 등)을 메시지로 자동 전송.
* **효과**: 지점별 피드백 자동화 및 관리 효율성 극대화.

### ⑥ [확장 기능/미확정] 입고일 추적 및 이카운트(eCount) 연동
* **기능**: 매장에 남은 재고가 며칠 전 입고분인지 신선도를 추적하고, 기존 이카운트 ERP의 매입 데이터를 API 또는 파일 형태로 연동하여 입력 리소스를 줄이는 방안. 단, **이카운트 연동은 통화 당시 확정된 범위가 아니며**, 추후 필요 여부와 방식(API/파일)을 별도 검토해야 함. (※ 별도 추가 공수 필요)

### ⑦ [추후 검토] AI 기반 진열 이미지 식별
* **기능**: 매장 진열 사진을 업로드하면 AI가 특정 어종(예: 병어)을 인식하고 올바르게 진열되었는지 자동 판별. (※ AI 학습 데이터 구축 비용 별도)

## 4. 개발 대안 비교
1. **맞춤형 ERP 웹 시스템 구축 (권장)**: 요구 기능(권한 제어, 사유 필수 기록, 확장성 등)을 가장 완벽하게 구현 가능하며 장기적인 유지보수에 유리함.
2. **MS 자동화 프로그램(Power Automate 등) 활용**: 기존 원드라이브 엑셀 데이터를 모아서 보고서로 추출하는 방식. 단, 지점장의 장부 조작 가능성 및 엑셀 기반의 근본적인 한계는 존재함.
3. **클로드(Claude) 등 AI 활용**: 단편적인 수식 계산이나 보조 역할은 가능하나, 매일 누적되는 데이터 처리 및 권한 제어가 포함된 통합 자동화 시스템 구축에는 비용 및 기능적 한계가 있음.

---

## 5. 개발 일정 및 예산 (가견적)
* **예상 구축 비용**: 약 **150만 원** (상세 스펙 확정 시 최종 조율)
* **예상 개발 기간**: 약 **2주 ~ 3주** 소요 예정
* **향후 일정**: 제안 내용 검토 후 다음 주 월요일 또는 화요일까지 진행 여부 확정.
