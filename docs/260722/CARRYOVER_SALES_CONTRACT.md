# 이월 매출 계산 계약

- 확정일: 2026-07-22
- 확정 방식: 운영 사용자 확인
- 선택 계약: **A**

## 포함 관계

| 항목 | 장부 마감 매출 | 이월 매출 | 다음 영업일 총매출·결제 |
| --- | ---: | ---: | --- |
| 의미 | 장부 마감 시점에 기록된 매출 | 마감 뒤 추가로 발생한 매출 | 전일 이월은 포함하지 않음 |
| 영업 성과 반영 | 포함 | 포함 | 해당 영업일 입력값만 사용 |
| 마감 정산 반영 | 포함 | 제외 | 현금·카드·기타도 전일 이월을 포함하지 않음 |

```text
영업 매출 합계 = totalSalesAmount + carryoverSalesAmount
마감 정산 차액 = totalSalesAmount - cashAmount - cardAmount
                 - otherPaymentAmount - expenseTotal
```

## 2영업일 확인 예시

| 영업일 | 장부 마감 매출 | 전일 이월 반영 | 당일 이월 매출 | 영업 매출 합계 | 결제 정산 기준 |
| --- | ---: | ---: | ---: | ---: | ---: |
| D | 100,000원 | 0원 | 10,000원 | 110,000원 | 100,000원 |
| D+1 | 120,000원 | **포함하지 않음** | 5,000원 | 125,000원 | 120,000원 |

- D의 이월 매출 10,000원은 D의 영업 성과에만 더한다.
- D+1의 `totalSalesAmount`, 현금, 카드, 기타 결제에는 D의 이월 매출을 다시 넣지 않는다.
- 따라서 전일 이월 차감이 필요한 계약 B 공식을 사용하지 않는다.
