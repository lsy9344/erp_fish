# 실사용 후 요청사항 검토 및 작업 계획

작성일: 2026-07-02

## 1. 한 줄 요약

이번 요청은 지점과 본사가 실제로 장부를 쓰는 순서를 현장 흐름에 맞게 바꾸고, 매입 입력 화면을 더 짧고 편하게 만들며, 본사 홈에서 주요 항목을 눌렀을 때 관련 상세 화면으로 바로 이동하게 고치는 작업입니다.

## 2. 요청사항 정리

| 번호 | 의뢰자 요청 | 정리한 의미 | 우선순위 |
|---:|---|---|---|
| 1 | 본사 및 지점 아이디에 고정된 단계 변경 | 매입을 1단계로 옮기되, 계산 안전성을 위해 최종 순서는 `매입 > 손실/폐기 > 재고 > 비용 > 근무/인건비 > 매출/결제 > 검토/제출`로 구성한다. | P0 |
| 2 | 매입 화면 카드가 많아 스크롤 압박이 심할 것 같음 | 매입 항목이 여러 개일 때 한 항목이 너무 길게 보이지 않도록 화면을 압축한다. 입력 편의와 실수 방지를 함께 본다. | P0 |
| 3 | 본사계정에서 매출/비용/매입 등을 눌러도 해당 위치로 이동이 안 됨 | (2026-07-02 확인) 지점장 입력 화면 상단 "현재 단계" 네비는 이미 각 단계 클릭 이동을 지원한다. 실제 개선 대상은 **본사 장부 상세 탭 순서**를 지점 입력 순서와 동일하게 맞추는 것이다. | P1 |

> **정정(2026-07-02):** 최초 요청 문구는 "본사 홈(관제판) 카드/항목 클릭 이동"으로 읽혔으나, 의뢰자 확인 결과 요지는 (1) 지점장 입력 7단계 순서 일치와 (2) **본사 장부 상세 탭 순서**를 같은 흐름으로 맞추는 것이다. 관제판 표에는 매출/마진율/손실 값 셀만 있어 비용·매입·재고·근무 열이 없으므로, 새 열 추가 없이 탭 순서 정렬로 처리한다.

## 3. 현재 상태 검토

### 3.1 단계 순서

현재 지점 장부의 단계 표시는 아래 순서로 잡혀 있습니다.

`매출/결제 > 비용 > 매입 > 손실/폐기 > 재고 > 근무/인건비 > 검토/제출`

의뢰자 요청 순서는 아래와 다릅니다.

`매입 > 재고 > 손실/폐기 > 비용 > 근무/인건비 > 매출/결제 > 검토/제출`

단계 라벨만 바꾸면 안 됩니다. 각 화면의 `다음 단계로` 버튼, 검토 화면의 누락 항목 링크, 본사 장부 상세 탭 순서도 함께 바뀌어야 사용자가 같은 흐름으로 느낍니다.

추가로 확인한 결과, 현재 재고 계산은 `손실/폐기`가 먼저 검토된 뒤 `재고`를 저장하는 흐름을 전제로 합니다. 서버에서도 손실/폐기 단계를 먼저 저장하지 않으면 재고 저장을 막고 있습니다. 이유는 판매 수량과 추정 매출이 `전일재고 + 당일매입 - 손실수량 - 당일재고`로 계산되기 때문입니다.

따라서 요청 순서를 그대로 `매입 > 재고 > 손실/폐기`로 바꾸는 것은 안전하지 않습니다.

가장 효율적인 최종안은 아래 순서입니다.

`매입 > 손실/폐기 > 재고 > 비용 > 근무/인건비 > 매출/결제 > 검토/제출`

이 방식은 의뢰자의 핵심 요청인 "매입을 먼저"는 반영하면서, 현재 계산 구조가 요구하는 `손실/폐기 -> 재고` 흐름은 유지합니다. 그래서 계산 정책을 새로 만들지 않고, 화면 순서와 이동 링크 중심으로 고칠 수 있습니다.

### 3.2 매입 화면

현재 매입 화면은 품목 1개당 카드 1개가 세로로 쌓이는 구조입니다. 한 카드 안에 품목 선택, 원문명, 구분, 규격, 참고 단가, 매입 단가, 수량, 오늘 팔 가격, 매입금액이 모두 들어갑니다.

품목이 많아지면 스크롤이 길어지는 것이 맞습니다. 특히 이카운트 업로드 후 여러 품목이 한 번에 들어오면 지점장이 실제로 입력해야 하는 값은 많지 않은데 화면은 길게 느껴질 수 있습니다.

### 3.3 본사 홈 항목 이동

현재 본사 홈/관제판은 요약과 지점별 상태를 먼저 보는 구조입니다. 지점 행이나 `상세 보기`를 통해 장부 상세로 들어가는 흐름은 있지만, 사용자가 `매출`, `비용`, `매입` 같은 항목명을 직접 누르면 각 상세 화면으로 이동하는 컨셉은 아직 충분히 적용되어 있지 않습니다.

즉, 원래 구현은 "홈은 요약, 상세는 별도 진입"에 가깝습니다. 다만 실제 사용에서는 숫자나 항목명을 누르면 바로 근거 화면으로 들어가기를 기대하므로, 이 부분은 사용성 개선 대상으로 보는 것이 맞습니다.

## 4. 작업 계획

### 4.1 단계 순서 변경

1. 최종 단계는 아래 순서로 구성합니다.
   - 1단계: 매입
   - 2단계: 손실/폐기
   - 3단계: 재고
   - 4단계: 비용
   - 5단계: 근무/인건비
   - 6단계: 매출/결제
   - 7단계: 검토/제출

2. 이 순서를 최종안으로 잡는 이유는 아래와 같습니다.
   - 매입을 먼저 입력해야 손실/폐기와 재고가 오늘 들어온 물량을 기준으로 계산됩니다.
   - 손실/폐기를 재고보다 먼저 둬야 판매 수량과 추정 매출 계산이 맞습니다.
   - 기존 서버 검증, FIFO 갱신, 재고 저장 규칙을 크게 바꾸지 않아 가장 빠르고 안전합니다.
   - `재고 -> 손실/폐기`로 억지 변경하는 것보다 작업량과 회귀 위험이 작습니다.

3. 작성자 표시명은 1단계 매입 화면 또는 장부 공통 상단에서 받도록 옮깁니다.
   - 현재는 매출/결제 단계에서 받지만, 매출/결제가 6단계로 이동하므로 그대로 두면 앞 단계에서 작성자 표시명이 `미입력`으로 보입니다.
   - 최초 작성자 보존 정책은 유지하고, 입력 위치만 앞쪽으로 옮깁니다.

4. 지점 장부의 기본 진입 화면을 매입 단계로 바꿉니다.
   - 현재처럼 장부 메뉴를 눌렀을 때 매출부터 시작하지 않고, 매입부터 시작하게 합니다.

5. 각 단계의 `다음 단계로` 버튼을 최종 순서에 맞춥니다.
   - 매입 -> 손실/폐기
   - 손실/폐기 -> 재고
   - 재고 -> 비용
   - 비용 -> 근무/인건비
   - 근무/인건비 -> 매출/결제
   - 매출/결제 -> 검토/제출

6. 검토/제출 화면의 단계 요약과 누락 항목 안내도 새 순서로 정렬합니다.

7. 본사 장부 상세 탭도 같은 순서로 정리합니다.
   - 매입, 손실, 재고, 비용, 근무, 매출 순서로 맞춥니다.

8. `재고 -> 손실/폐기` 원안은 보류합니다.
   - 이 원안을 적용하려면 손실 입력 전 재고 임시 저장, 손실 저장 후 재고/FIFO 재계산, 재고 재확인 상태 표시가 추가로 필요합니다.
   - 이번 요청은 사용성 개선이 목적이므로, 계산 정책 변경까지 확장하지 않는 것이 가장 효율적입니다.

### 4.2 매입 화면 간소화

1. 매입 항목 카드를 더 짧게 만듭니다.
   - 기본 화면에는 실제로 자주 확인하는 값만 먼저 보입니다.
   - 예: 품목, 규격, 매입 단가, 수량, 오늘 팔 가격, 매입금액

2. 보조 정보는 접거나 약하게 표시합니다.
   - 원문명, 구분, 참고 단가, 참조 정보는 필요할 때만 보이게 하거나 한 줄 요약으로 줄입니다.

3. 여러 품목을 빠르게 입력할 수 있게 합니다.
   - 품목 행을 표 형태에 가깝게 압축하는 방안을 우선 검토합니다.
   - 모바일에서는 표가 좁아질 수 있으므로 카드 형태를 유지하되, PC에서는 더 촘촘하게 보이게 합니다.

4. 지점장이 수정할 수 없는 값은 더 작게 보이게 합니다.
   - 잠긴 값은 큰 입력칸처럼 보이지 않게 해서 "입력해야 하는 칸"으로 오해하지 않게 합니다.

### 4.3 본사 항목 클릭 시 상세 이동 (정정된 범위)

의뢰자 확인 결과, 이 항목은 아래 두 가지로 정리합니다.

1. 지점장 입력 화면 상단 "현재 단계" 네비는 이미 각 단계(매출/비용/매입/재고/손실/근무) 클릭 이동을 지원합니다. 별도 구현이 필요 없고, 단계 순서만 4.1 최종안으로 맞춥니다.

2. 본사 장부 상세 탭 순서를 지점 입력 순서와 동일하게 정렬합니다(4.1의 7번과 동일 작업).
   - 탭 순서: 매입 -> 손실 -> 재고 -> 비용 -> 근무 -> 매출/결제

3. 관제판(본사 홈) 표에는 매출·마진율·손실 값 셀만 있고 비용·매입·재고·근무 열이 없습니다.
   - 손실 셀은 이미 `?tab=losses` 바로가기가 있습니다. 관제판 행 클릭은 기본 탭(매입)으로 이동합니다.
   - 새 열 추가나 관제판 재설계는 이번 범위에서 제외합니다(요청 요지가 아님).

## 5. 확인 필요한 부분

| 확인할 것 | 왜 필요한가 | 제안 |
|---|---|---|
| 매입 화면을 표 형태로 압축해도 되는지 | 표는 PC에서 빠르지만 모바일에서는 좁을 수 있습니다. | PC는 압축형, 모바일은 간단 카드형으로 나누는 것을 추천합니다. |
| 본사 홈에서 클릭 대상이 지점별 표인지, 상단 요약 카드까지 포함인지 | 요청 문구만 보면 홈 전체 항목처럼 보일 수 있습니다. | 1차는 지점별 표의 매출/비용/매입 항목을 상세로 연결하고, 상단 요약 카드는 별도 리포트 연결 여부를 확인하는 것을 추천합니다. |
| "손실/폐기"에 "떨이" 표현을 같이 둘지 | 기존 화면에는 손실/폐기/떨이 표현이 섞여 있습니다. | 단계명은 요청대로 `손실/폐기`로 두고, 화면 내부 설명에만 떨이를 남기는 것을 추천합니다. |

## 6. 권장 실행 순서

1. 단계 순서와 다음 단계 링크를 최종안으로 고칩니다.
2. 작성자 표시명 입력 위치를 1단계 매입 또는 공통 상단으로 옮깁니다.
3. 본사 홈/관제판의 매출, 비용, 매입 등 항목을 관련 상세 화면으로 연결합니다.
4. 매입 화면을 압축형으로 개선합니다.
5. 지점 계정과 본사 계정에서 같은 날짜 장부를 열어 실제 클릭 흐름을 확인합니다.
6. 모바일 화면에서 매입 항목이 깨지거나 너무 좁아지지 않는지 확인합니다.

## 7. 완료 기준

1. 지점 장부에서 `매입 > 손실/폐기 > 재고 > 비용 > 근무/인건비 > 매출/결제 > 검토/제출` 순서대로 보입니다.
2. `다음 단계로` 버튼을 누르면 같은 순서대로 이동합니다.
3. 본사 장부 상세에서도 탭 순서가 같은 흐름과 맞습니다.
4. 본사 홈/관제판에서 `매출/비용/매입/...` 항목을 누르면 해당 상세 화면으로 이동합니다.
5. 매입 품목이 여러 개 있어도 한 화면에서 더 많은 항목을 볼 수 있습니다.
6. 손실/폐기, 재고, FIFO, 추정 매출, 재고 차이 계산이 기존 테스트 기준과 맞습니다.
7. 작성자 표시명이 앞 단계부터 `미입력`으로 방치되지 않습니다.

## 8. 의뢰자 전달용 짧은 브리핑

요청 주신 3가지는 실제 입력 순서와 화면 사용성을 맞추는 작업으로 정리했습니다.

장부 단계는 최종적으로 `매입 > 손실/폐기 > 재고 > 비용 > 근무/인건비 > 매출/결제 > 검토/제출` 순서로 구성하는 것이 가장 효율적입니다.

요청하신 것처럼 매입은 맨 앞으로 옮기되, 재고는 손실/폐기 뒤에 두겠습니다. 현재 판매 수량과 추정 매출이 `전일재고 + 당일매입 - 손실수량 - 당일재고` 기준으로 계산되기 때문에, 손실/폐기를 먼저 검토한 뒤 재고를 입력해야 계산이 가장 안정적입니다.

작성자 표시명은 기존처럼 매출/결제 단계에서 받으면 너무 늦어지므로, 1단계 매입 또는 장부 공통 상단에서 먼저 입력하도록 위치를 조정하겠습니다.

매입 화면은 품목이 많아질수록 카드가 길어져 스크롤이 많아지는 구조라, PC에서는 표처럼 더 촘촘하게 보고 입력할 수 있게 줄이고, 모바일에서는 깨지지 않는 간단 카드형을 유지하는 방향으로 개선하겠습니다.

본사 홈에서 `매출/비용/매입` 등을 눌렀을 때 해당 상세 화면으로 바로 이동하지 않는 부분은 현재 홈이 요약 관제판 중심으로 구성되어 있어서 생긴 사용성 문제로 보입니다. 실사용 흐름에 맞게 각 항목을 클릭하면 해당 장부 상세의 관련 화면으로 바로 이동하도록 개선하겠습니다.

본사 장부 상세 화면의 탭 순서도 지점과 동일하게 맞추겠습니다.
