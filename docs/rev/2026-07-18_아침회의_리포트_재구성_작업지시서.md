# 아침 회의 리포트 재구성 Implementation Plan

> **구현 기준:** `docs/rev/2026-07-18_아침회의_리포트_재구성_계획.md`를 구현 가능한 작업 단위로 구체화한 문서다. 별도 지시가 없으면 `매장 매출 포지션`은 사분면이 아니라 **권한 범위 내 매출 순위 + 매출 비중**으로 구현한다.

**Goal:** `/app/reports/daily`를 `지점별 매출·이익률 → 매출 분석 → 품목별 판매 현황 → 직원 근태 현황 → 마감·이상 신호 현황` 순서로 재구성하고, 정정 반영 매출 분석 3종과 기존 장부 기반 근태 현황을 추가한다.

**Architecture:** 기존 `getHqDailyMeetingReport()`의 권한 범위, 장부 조회, 정정 overlay, FIFO 재고금액, 기존 차트·표를 재사용한다. 선택일과 직전 달력일 장부를 같은 조회 흐름에서 읽고 서버에서 계산을 끝낸 DTO만 UI에 전달한다. 신규 DB 모델, 마이그레이션, API route, 상태 저장소, 차트 의존성은 추가하지 않는다.

**Tech Stack:** Next.js 15 App Router, React 19, TypeScript, Prisma 6, shadcn/ui, Tailwind CSS 4, Recharts 3(기설치), Node test runner, Playwright

## 1. 확정 정책

### 1.1 화면 순서

1. 페이지 제목 / 리포트 이동 / 날짜 / Excel·CSV
2. `지점별 매출·이익률`
3. `매출 분석`
   - 전일 대비 매출액 증감률
   - 재고비율
   - 매장 매출 포지션
4. `품목별 판매 현황`
5. `직원 근태 현황`
6. `마감·이상 신호 현황`
   - 기존 상태 요약 카드 5개
   - 기존 `DailyMeetingReportTable`

데스크톱과 모바일의 섹션 순서는 동일해야 한다. 페이지 상단의 리포트 이동, 오늘·어제, 날짜 입력, Excel·CSV 동작은 변경하지 않는다.

### 1.2 전일 대비 매출

```text
증감액 = 선택일 정정 반영 매출 - 전일 정정 반영 매출
증감률 = 증감액 / 전일 정정 반영 매출
```

- 전일은 직전 **달력일**이다. 영업일 캘린더를 만들지 않는다.
- 선택일과 전일 모두 기존 정정 overlay를 거친 `totalSales` metric을 사용한다.
- 기본 순서는 계산 가능한 행의 `abs(증감률)` 내림차순, 동률은 지점명 오름차순이다. 계산 불가 행은 뒤에 둔다.
- 전일 장부 미입력, 전일 휴무, 전일 매출 0원, 선택일 매출 계산 불가는 `0%`가 아니라 `계산 불가`와 원인을 표시한다.

### 1.3 재고비율

```text
재고비율 = 선택일 FIFO 기준 재고금액 / 선택일 정정 반영 매출
```

- 기존 장부의 FIFO `inventoryAmount`와 기존 정정 overlay 계산 흐름을 재사용한다.
- 클라이언트에서 `수량 × 단가`를 다시 계산하지 않는다.
- 재고 항목이 없거나 한 품목이라도 재고금액을 확정할 수 없으면 지점 전체 재고비율을 `계산 불가`로 표시한다. 부분합은 표시하지 않는다.
- 선택일 매출이 0원이거나 계산 불가이면 비율도 `계산 불가`다.
- `재고금액 / 매출액 / 재고비율`을 함께 표시한다.

### 1.4 매장 매출 포지션

순위 대상은 권한 범위 안의 활성 지점 중 선택일 장부가 있고 휴무가 아니며 정정 반영 매출을 계산할 수 있는 지점이다. 유효하게 입력된 0원 매출은 순위에는 포함하되, 전체 합계가 0원이면 비중과 평균 대비 값은 계산 불가다.

```text
매출 비중 = 지점 정정 반영 매출 / 순위 대상 지점 매출 합계
평균 대비 = (지점 정정 반영 매출 - 순위 대상 평균 매출) / 순위 대상 평균 매출
```

- 기본 순서는 매출 내림차순, 동률은 `Intl.Collator("ko-KR")` 기준 지점명 오름차순이다.
- `순위 / 지점 / 매출액 / 전체 비중 / 전체 평균 대비`를 표시한다.
- 비중 막대는 CSS로 그린다. 새 차트나 시각화 의존성을 추가하지 않는다.
- 미입력과 휴무 지점은 순위에서 제외하고 지점명과 제외 사유를 별도로 표시한다.
- 순위·비중·평균의 모집단은 반드시 동일한 권한 범위와 동일한 제외 규칙을 사용한다.

### 1.5 직원 근태

근태 원본은 선택일 `DailyLedger.workerCount`와 `LedgerLaborItem`이다. 급여·인건비 금액은 사용하지 않는다.

| 상태        | 판정                                               |
| ----------- | -------------------------------------------------- |
| 정상        | 직원 행이 있고 지각·조퇴·특이사항 메모가 모두 없음 |
| 지각        | `lateMemo`의 trim 결과가 비어 있지 않음            |
| 조퇴        | `earlyLeaveMemo`의 trim 결과가 비어 있지 않음      |
| 특이사항    | `specialMemo`의 trim 결과가 비어 있지 않음         |
| 직원 미연결 | `workerName`은 있으나 `employeeId`가 없음          |
| 명단 부족   | 정정 반영 `workerCount`가 직원 행 수보다 큼        |

- 직원 한 명에게 여러 상태가 있으면 모든 배지를 표시한다.
- `직원 미연결`은 상세 상태로 표시하되 별도 요약 카드는 만들지 않는다.
- 요약값은 `총 근무자 / 지각 / 조퇴 / 특이사항 / 명단 부족`이다.
- `총 근무자`는 지점별로 `workerCount ?? 직원 행 수`를 합산한다.
- `명단 부족`은 `max(workerCount - 직원 행 수, 0)`의 인원 합계다.
- 명단이 부족한 지점은 상세 목록에 지점별 한 행을 추가하고 직원 칸에 `명단 미입력 N명`을 표시한다.
- 직원 행이 없고 `workerCount`도 없으면 해당 지점은 근태 미입력으로 표시한다.
- 결근, 휴가, 정확한 출퇴근 시각은 추정하지 않는다.

### 1.6 보안·데이터 경계

- `requireReportAccess()`와 `getHeadquartersStoreScope()`를 우회하거나 클라이언트 필터로 대체하지 않는다.
- 쿼리는 scope의 `storeIds`만 사용한다. 전일 데이터도 같은 범위로 제한한다.
- 근태 DTO에 `amount`, `employeeId`, `createdById`, `updatedById`를 넣지 않는다.
- 미입력·휴무·계산 불가를 숫자 0으로 바꿔 순위, 평균, 비율에 포함하지 않는다.
- 기존 일별 export에는 새 매출 분석·근태 컬럼을 추가하지 않는다.
