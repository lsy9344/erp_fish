# 본사 인건비·손실 입력·판매한 가격 이월 구현 결과

- 구현 기준: `docs/rev/2026-07-22_본사인건비_손실입력_판매가격이월_작업지시서.md`
- 구현일: 2026-07-22
- 검토 반영일: 2026-07-22 (4차)

## 구현 결과

### 본사 인건비 현황

- `/app/reports/labor` 읽기 전용 화면을 추가했다.
- `REPORT_VIEW` 권한과 `getHeadquartersStoreScope()` 지점 범위를 쿼리 전에 적용했다.
- 월, 지점, 장부 상태 필터와 전체/지점/근무자 상세 합계를 추가했다.
- `employeeId`가 없는 자유 입력 근무자도 `workerName` 스냅샷으로 표시한다.
- 본사 사이드바와 리포트 상단 메뉴에 인건비 진입점을 추가했다.
- 권한 밖 `storeId`는 전체 승인 지점으로 확장하지 않고 **빈 결과 + 안내 메시지**로 fail-closed 처리한다.
- 권한 밖 요청 시 DTO의 `selectedStoreId`는 `null`이다 (`requestedStoreId`만 요청값을 보존).

### 손실 수량 소수 둘째 자리

- 손실 전용 두 자리 클라이언트 파서와 서버 검증을 추가했다.
- 허용 상한은 `9,999,999,999.99`이며 저장 경계에서 두 자리로 정규화한다.
- 지점장 손실 저장과 본사 손실 수정에 같은 정책을 적용했다.
- 떨이 실제 판매 금액의 정수 정책과 매입 수량의 소수 첫째 자리 정책은 유지했다.

### 지점장 손실 품목 제한

- 기존 재고 계산 결과의 `전일 이월 + 당일 매입 - 저장 손실` 값이 0보다 큰 active 품목만 신규 선택 목록에 표시한다.
- 신규 행과 품목 변경 행은 서버에서도 같은 가용 품목 집합으로 검증한다.
- 기존 `id + productId` 행은 재고 소진 또는 품목 비활성 후에도 저장 스냅샷으로 표시하고 수량/사유 수정 또는 삭제할 수 있다.
- 본사 장부 수정 화면의 전체 활성 품목 옵션은 변경하지 않았다.
- 가용 품목 계산은 이미 로드한 ledger `{ id, storeId, closingDate }`만 받고, 매입/손실/재고/이월에서 **수량 필드만** 조회한다.
- 전일 수량 원본 선택은 `resolveInventoryPreviousQuantitySource`로 재고 이월·손실 가용 경로가 공유한다.

### 판매한 가격 이월

- 공용 `InventoryStepData` / HQ·감사 스냅샷은 **당일 exact-date** `plannedUnitPrice`만 유지한다.
- 전일 fallback은 지점장 응답 경계(`toStoreManagerInventoryStepDataInTx`)에서만 적용한다.
- 지점장 DTO에 `plannedUnitPriceSource: CURRENT | CARRYOVER | null`을 두어 당일 저장값과 입력 제안값을 구분한다.
- 충돌 응답은 `buildInventoryConflictServerValues`로 `2500(당일)` / `2000(이월)`을 `serverValues`에 포함한다.
- 현재 영업일 값이 없으면 최근 `IN_REVIEW` 또는 `HEADQUARTERS_CLOSED` 장부 한 건의 같은 품목 값을 초깃값으로 사용한다.
- fallback 조회는 읽기 전용이며, 기존 재고 저장 upsert에서만 당일 행을 만든다.
- 당일 완료 판정과 본사 리포트의 exact-date 계약은 변경하지 않았다.
