### 1.2 별도 지시가 없을 때 적용할 기본값

1. 같은 지점·영업일·품목의 오늘 팔 가격은 한 개라는 서버/DB 규칙을 유지하고 UI 행만 동기화한다.
2. 재고 근거가 없는 손실 저장 차단은 유지한다. 2026-07-16 데이터에서 품목 ID 또는 이월 누락이 확인될 때만 해당 상류 데이터를 고친다.
3. 근거 없는 품목의 매입단가는 직접 추가하는 최초 입력 때만 지점장이 작성한다. 저장 후 내부 단가를 지점장 DTO로 다시 노출하지 않는다. 재수정은 기존 본사 수정 경로를 사용한다.
4. `0원` 매입단가는 기존 KRW 검증과 맞춰 허용한다. 빈 단가는 저장을 막는다.
5. 6단계의 `총매출` 저장 의미는 바꾸지 않는다. 차액만 `총매출 - 결제 합계 - 지출 합계`로 계산한다.
6. 대시보드의 경보 기준 마진은 예상마진이 아니다. 실제/예상 마진과 별도 줄로 표시한다.
7. 지점 차트는 현재처럼 입력 중·검토 대기·본사 마감 장부를 모두 포함한다. 따라서 화면 용어는 `확정 매출` 대신 정확한 `장부 입력 매출`을 사용한다. 본사 마감 장부만 표시하는 상태 필터는 추가하지 않는다.
8. 지점 차트의 막대는 항상 매출액이며 버튼은 정렬 기준만 바꾼다. 실제/예상 마진 차이가 정확히 1.5%p 이상이면 막대를 `destructive` 색으로 표시한다.
9. 품목 상위 기준은 사진 문맥에 맞춰 판매수량 내림차순이다. 검색 중에는 전체 일치 결과를 보여주고 검색이 비면 상위 10개로 돌아간다.
10. 품목 그래프와 위 4개 추정 요약은 사진의 일별 리포트에서만 숨긴다. 공용 컴포넌트를 쓰는 매출 검토·품목 검토의 차트는 유지한다.
11. 판매수량은 POS 실판매량이 아니라 재고 흐름 추정값이다. 컬럼명에서는 `추정`을 빼되 표 아래 근거 문구는 유지한다.
12. 재고·손실 정정값을 예상마진과 품목순위에 완전히 재적용하는 일은 이번 사진 요청 범위에서 제외한다. 현재 쿼리의 혼합 기준은 알려진 후속 결함으로 기록한다.

2026-07-16 테스트지점의 `참소라 / 중`에 대해 다음 값을 한 표로 남긴다.

```sql
SELECT p.id, p.name, p.category, p.spec, p."isActive"
FROM "Product" p
WHERE p.name = '참소라' AND p.spec = '중';

SELECT s.name AS store_name, d.id AS ledger_id, d."closingDate",
       p.id AS product_id, p.name, p.spec,
       lp.quantity AS purchase_quantity,
       li."currentQuantity", li.quantity AS saved_quantity,
       li."carryoverSource", li."carryoverStatus"
FROM "DailyLedger" d
JOIN "Store" s ON s.id = d."storeId"
LEFT JOIN "LedgerPurchaseItem" lp ON lp."dailyLedgerId" = d.id
LEFT JOIN "Product" p ON p.id = lp."productId"
LEFT JOIN "LedgerInventoryItem" li
  ON li."dailyLedgerId" = d.id AND li."productId" = p.id
WHERE (d."closingDate" AT TIME ZONE 'Asia/Seoul')::date = DATE '2026-07-16'
  AND p.name = '참소라' AND p.spec = '중';

SELECT s.name AS store_name, o."yearMonth", o."productId",
       o."productName", o."productSpec", o.quantity
FROM "InventoryOpeningSnapshot" o
JOIN "Store" s ON s.id = o."storeId"
WHERE o."yearMonth" = '2026-07'
  AND o."productName" = '참소라' AND o."productSpec" = '중';
```
