## Decisions

### D-001 UI 시스템
- **결정:** shadcn/ui + Tailwind CSS
- **근거:** 개발팀 확정 선택
- **영향:** DESIGN.md 토큰은 shadcn 기본값을 참조/확장. EXPERIENCE.md는 동작 델타만 명시.

### D-002 시각적 방향
- **결정:** 깔끔하고 현대적인 SaaS 느낌
- **근거:** 사용자 확인
- **영향:** 밀도 높은 데이터 화면에서도 여백과 계층을 유지하는 설계 방향

### D-003 다크 모드
- **결정:** 필요함 (라이트/다크 모두 지원)
- **근거:** 야간 매장 환경 고려
- **영향:** DESIGN.md에 light/dark 색상 토큰 쌍 정의 필요

### D-004 재고 입력 품목 수
- **결정:** 한 지점 하루 장부 기준 30개 이상 (냉동+생물 합산)
- **근거:** 사용자 확인
- **영향:** 모바일에서 30+ 행 테이블 입력 → 핵심 UX 도전 과제. 가상 스크롤, 전일 이월 프리필, 빠른 편집 패턴 설계 필요.

### D-005 관제판 레이아웃
- **결정:** 테이블형, 일반 모니터 환경
- **근거:** 사용자 확인
- **영향:** 지점당 1행, 밀도 높은 컬럼 구성 가능. 가로 스크롤 최소화 설계.

### D-010 네비게이션 구조
- **결정:** 데스크탑 좌측 사이드바 (shadcn 표준) + 모바일 하단 탭 바 (장부/재고/손실/코멘트)
- **근거:** 사용자 승인

### D-011 관제판 드릴다운
- **결정:** 관제판에서 지점 행 클릭 → 해당 지점 일일 장부 상세 + 품목 재고 테이블 표시
- **근거:** 사용자 명시 요구사항
- **영향:** 장부 상세 화면에 InventorySnapshot 기반 품목별 테이블 필수 포함.
  슬라이드오버(Sheet) 또는 전용 페이지로 진입.

### D-009 색상 테마
- **결정:** Coastal SaaS (#2) — 슬레이트 + 스카이 블루
- **Light:** bg #f9fafb / surface #fff / primary #2563eb / accent #0ea5e9 / border #e5e7eb
- **Dark:** bg #0a0a0a / surface #111 / primary #60a5fa / accent #38bdf8 / border #222
- **공통:** danger #ef4444(light)/#f87171(dark) / warning #f59e0b/#fbbf24 / success #22c55e/#6ee7b7
- **근거:** 사용자 선택

### D-007 재고 입력 방식
- **결정:** 전일 이월 프리필 → 변경 품목만 수정 (A타입)
- **근거:** 사용자 확인

### D-012 실시간 검색 및 정렬 (관제판)
- **결정:** 지점명 실시간 검색 및 지표 컬럼 정렬 기능(▲▼ 인디케이터) 제공
- **근거:** 대시보드 내 탐색 편의성 극대화 및 지점 확장성 확보

### D-013 매출차액 시각적 대비 개선
- **결정:** 매출차액 ₩0은 흐린 회색(중립), 차액 발생 시 굵은 빨간색/주황색으로 구분 표시
- **근거:** 정량적 편차 확인 효율 증대

### D-014 재고 탭 뱃지 확장
- **결정:** 상세 페이지 내 재고 탭(냉동/생물) 우측에 경고/수정 뱃지(예: ⚠ 1) 표시
- **근거:** 탭 전환 없이 미결 상태의 품목 인지 유도

### D-015 테이블 행 클릭 기반 정정 연동
- **결정:** 마감된 품목 재고 테이블의 각 행 클릭 시 우측 정정 패널의 타겟과 원본값이 자동으로 바인딩 및 포커싱 처리
- **근거:** 수동 입력 시 오타 및 선택 실수 최소화

### D-016 모바일 테이블 헤더 고정 (Sticky Header)
- **결정:** 모바일 재고 입력 시 테이블 스크롤에도 헤더(thead th)를 상단에 고정
- **근거:** 장시간 스크롤 시 열 항목 혼동 예방

### D-017 모바일 대화형 경고 툴팁 및 터치 타겟 확장
- **결정:** 모바일 환경에서 ⚠ 터치 시 말풍선 안내 툴팁 노출, 버튼 및 수량 입력 필드 크기 44px 이상 확장
- **근거:** 모바일 조작 제한(Hover 불가, 젖은 손 터치 등) 대응 및 모바일 전용 Validation 제공
- **영향:** 재고 입력 화면은 전일 재고가 자동 채워진 상태로 시작. 변경된 행만 강조 표시.

### D-008 이상 신호 알림
- **결정:** 관제판 화면 내 표시만으로 충분 (푸시 알림 불필요, 1차)
- **근거:** 사용자 확인
- **영향:** 이상 신호는 관제판 행의 시각적 강조로만 처리.

### D-006 폼팩터
- **결정:** 멀티 서피스
  - 지점장: 모바일(390px+) + 태블릿 + PC
  - 본사: PC/데스크탑 주력
- **근거:** PRD §5 비기능 요구사항 + 고객 요구사항
- **영향:** 지점장 장부 입력 흐름은 모바일 퍼스트 설계. 본사 관제판은 데스크탑 최적화.

### D-018 최신 PRD 동기화 상태
- **결정:** DESIGN.md와 EXPERIENCE.md 상태를 `draft`로 조정하고, 2026-06-11 최신 PRD 및 G6 체크리스트 기준으로 동기화한다.
- **근거:** PRD가 `draft` 상태이며 G1~G6 구현 게이트와 OQ가 남아 있다. S7은 MVP-S01~S03만 구현 스토리 추출 가능하다고 승인한다.
- **영향:** UX 스파인은 최신 기준과 맞지만, OQ가 열린 계산·FIFO·민감 지표·알림·직원/급여·특수기간 기능은 확정 UI로 취급하지 않는다.

### D-019 릴리스 버킷과 게이트 표기
- **결정:** IA와 Key Flow는 MVP, Extension A/B/C/D, blocked pending OQ, approved backlog candidate, approved discovery only, contract/ops only를 명시한다.
- **근거:** PRD §0.2, §8 CAP 구현 순서와 약속 원장.
- **영향:** 매입 업로드, FIFO, 전체 재고, 상품 분석, 직원/급여, 고정비, 특수기간, 알림은 기본 MVP 화면처럼 보이지 않으며, 각 게이트가 닫히기 전에는 disabled/gated 상태로 표시한다.

### D-020 권한 프로파일과 민감 지표 차단
- **결정:** `본사 사용자`를 대표/소유자, 본사 관리자, 본사 스텝, 업로드 담당자, 마감 담당자, 설정 관리자, 조회 전용 본사로 나눠 UX 동작을 정의한다. 지점장 경로는 원가·이익·마진율·재고금액·원가 lot 근거·본사 고정비·타 지점 비교를 기본 차단한다.
- **근거:** PRD §4.1, CAP-13, OQ-10A/OQ-10B.
- **영향:** 기존 Flow 2의 지점장 마진율/재고금액 요약은 제거한다. 차단은 UI 숨김이 아니라 서버 응답, export, 캐시, 알림 템플릿까지 포함한다.

### D-021 계산, FIFO, 가격 신뢰 상태
- **결정:** MVP 기본 계산값, FIFO 기준 계산값, `기준 확인 필요`를 분리한다. 가격 신뢰 상태는 `approved`, `manual_override`, `pending_review`, `basis_missing`으로 표시한다.
- **근거:** PRD §4.3, §4.5, CAP-7, OQ-7/OQ-17.
- **영향:** LotTracePanel과 FIFO 근거 조회는 Extension B 게이트 뒤로 이동한다. 정책 전에는 0원이나 확정 원가처럼 표시하지 않는다.

### D-022 마감 사전점검 패턴
- **결정:** 단순 ClosingDialog 대신 ClosePreflight를 기준 패턴으로 삼는다.
- **근거:** PRD §4.5 마감 가능성 표와 CAP-15.
- **영향:** 권한, 이미 마감, 숫자 오류, 업로드 대기/실패, 미확정 정정, 필수 누락, 이월 공백, 가격 기준 없음, 이상 신호, 휴무일을 차단/경고/예외 가능 상태로 나눠 보여준다.

### D-023 업로드 배치 상태 모델
- **결정:** 이카운트 업로드는 `preview`, `committed`, `voided`, `reprocessed`, `failed` 상태를 가진다.
- **근거:** PRD CAP-6 업로드 상태 모델.
- **영향:** 업로드 flow는 헤더 매핑, preview 편집, 중복 후보 선택, commit, void/reprocess 감사 이벤트를 포함한다. CAP-6은 OQ-15와 파일 계약 전까지 gated Extension B로 둔다.

### D-024 내비게이션 및 목업 참조 정리
- **결정:** 모바일 하단 탭은 `장부` / `재고` / `손실` 3개로 둔다. `특이사항`은 7단계 입력의 근무인원/특이사항 단계에 포함한다. 관제판 드릴다운은 Sheet가 아니라 `/app/ledgers/[ledgerId]` 페이지 라우트가 기준이다.
- **근거:** EXPERIENCE.md 현행 동작과 PRD FR-5, FR-15.
- **영향:** D-010의 `코멘트` 탭과 D-011의 Sheet 가능성은 최신 기준으로 대체된다. 존재하지 않는 improved/v2 목업 링크는 EXPERIENCE.md frontmatter에서 제거한다.

### D-025 알림 결정 범위 보정
- **결정:** MVP 이상 신호 알림은 관제판 화면 내 표시가 기준이다. LINE/텔레그램 자동 알림은 Extension D CAP-11이며 OQ-13/OQ-16과 템플릿 권한 정책 전까지 차단한다.
- **근거:** PRD §6, §8.6, OQ-13/OQ-16.
- **영향:** D-008은 MVP 범위에는 유지되지만, 외부 알림을 영구 제외로 해석하지 않는다.

### D-026 관제판 검색/정렬과 리사이징 범위
- **결정:** 실시간 검색/임의 컬럼 정렬은 별도 승인 전 필수 동작으로 보지 않는다. 대시보드 그리드 리사이징은 Extension A CAP-18로 두고, 핵심 컬럼 최소 폭과 reset 동작을 EXPERIENCE.md에 정의한다.
- **근거:** PRD FR-15, CAP-18.
- **영향:** D-012는 필수 v1 결정이 아니라 deferred/optional 결정으로 대체된다.
