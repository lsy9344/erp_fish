# ERP Fish — Experience Spine

> 멀티 서피스 반응형 웹. shadcn/ui + Tailwind CSS. 시각적 정체성은 `DESIGN.md` 참조. 스파인이 목업·와이어프레임과 충돌하면 스파인이 이긴다.

## Foundation

멀티 서피스 반응형 웹. shadcn/ui on Next.js + Tailwind CSS.

사용자 역할은 두 가지다: **본사 사용자**(전체 지점 조회·입력·수정·마감·정정)와 **지점장**(자기 지점 일일 장부 입력). 역할 분기는 서버에서 강제하며, 클라이언트 분기에 의존하지 않는다.

본사 사용자의 주 서피스는 데스크탑이다. 지점장의 주 서피스는 모바일(390px+)·태블릿이며 PC도 지원한다. 장부 입력 핵심 흐름은 모바일 퍼스트로 설계한다.

`DESIGN.md`가 시각적 정체성의 단일 출처다. 색상 토큰, 배지 스타일, 신호 칩, MetricCard, 재고 행 강조는 `DESIGN.md.Components`를 참조한다.

### 내비게이션 구조

**데스크탑 (본사 / 지점장 PC):** 본사는 흰색 좌측 사이드바 + 옅은 회색 본문 캔버스. 활성 메뉴는 primary blue 배경으로 표시한다. 항목은 현재 앱 라우트 구조를 유지하되 추가 범위에서는 홈, 장부, 매입/재고, 리포트, 인사, 손익, 기준정보, 알림 설정, 변경 이력, 설정으로 묶는다.

**모바일·태블릿 (지점장 주 서피스):** 하단 탭 바 56px + safe-area-inset. 탭: `장부` / `재고` / `손실`. 탭 이동은 현재 입력 단계를 유지한다.

**모달 스택:** 1단계 이하. Dialog 위에 Dialog 금지. 현 구현의 본사 장부 상세는 Sheet가 아니라 페이지 라우트다. 마감 확인과 정정 기록 입력은 같은 상세 페이지 안에서 처리한다.

## Component Patterns

동작 기준. 시각적 스펙은 `DESIGN.md.Components` 또는 shadcn 기본값 참조.

| 컴포넌트                                   | 사용처                     | 동작 규칙                                                                                                                                                                                                            |
| ------------------------------------------ | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **관제판 행 (BranchRow)**                  | 관제판                     | 행 전체 클릭 → 장부 상세 페이지로 이동. 이상 신호 있는 행은 `{DESIGN.md.dashboard-row.anomaly}` 처리. 현재 구현은 날짜/정렬/필터 링크 상태를 보존하며, 실시간 검색·컬럼 정렬·호버 플로팅 힌트는 v1 구현 범위 밖이다. |
| **MetricCard**                             | 관제판, 리포트, 장부 상세  | 핵심 수치 요약 카드. 흰색 카드, 얇은 border, 작은 shadow, `tabular-nums`. 상태형 변형은 `default`, `success`, `warning`, `danger`, `muted`.                                                                          |
| **품목 재고 테이블 (InventoryTable)**      | 장부 상세 / 지점 재고 입력 | 냉동/생물 탭 전환. 품목·규격·전일재고·매입·손실·당일재고·재고금액·수량·상태 컬럼 표시. 수정된 행은 `{DESIGN.md.inventory-row-modified}` 강조. 모바일/좁은 화면에서는 가로 스크롤과 sticky table header를 사용한다.   |
| **단계형 입력 폼 (StepForm)**              | 지점장 장부 입력           | 7단계: 매출/결제 → 비용 → 매입 → 재고 → 손실/폐기 → 근무인원 → 검토/제출. 상단 Step Indicator. 단계 간 이동 자유(저장된 데이터 유지). 이전/다음 버튼 + 하단 탭 직접 이동.                                            |
| **재고 행 인라인 편집 (InventoryRowEdit)** | 재고 입력                  | 전일 이월값 프리필. 모바일 터치 최적화 규격(`cell-input-mobile`, `add-row-mobile` 최소 44px 이상 터치 규격 확보). 변경 즉시 `{DESIGN.md.inventory-row-modified}` 표시.                                               |
| **이상 신호 칩 (SignalChip)**              | 관제판 행, 장부 상세 헤더  | 아이콘(삼각 경고 또는 원) + 짧은 레이블. 경고 등급(amber), 심각 등급(red). 툴팁: 구체적 수치 표시. 복수 칩 허용.                                                                                                     |
| **마감 확인 다이얼로그 (ClosingDialog)**   | 본사 장부 상세             | shadcn `Dialog`. 제목: "장부를 마감합니다". 안내: "마감 후 지점장은 원본을 수정할 수 없습니다." 버튼: 취소 / 마감 확정.                                                                                              |
| **정정 기록 패널 (CorrectionPanel)**       | 본사 마감된 장부 상세      | 원본 잠금 시 대상 항목 선택, 원본값 확인, 정정값과 사유 입력. 저장 후 "정정 반영" 레이블과 원본/정정값 근거를 표시한다.                                                                                              |
| **장부 상태 배지 (StatusBadge)**           | 관제판, 장부 상세 헤더     | 5종: 미입력·입력중·검토대기·본사마감·휴무. `{DESIGN.md.badge-*}` 스타일 적용. 항상 텍스트만, 아이콘 없음.                                                                                                            |
| **검토/제출 요약 (ReviewSummary)**         | 장부 입력 마지막 단계      | 핵심 계산값 요약. 누락 항목·이상 신호 후보 목록. "제출" 버튼은 이상 후보 있어도 활성화—경고는 차단이 아닌 확인 목적.                                                                                                 |
| **매핑 실패 검수 (ProductMappingReview)**  | 매핑 실패 검수             | 원문 품목명, 제안 품목, 분석용 품목/규격, 상태, 저장 액션. 실패 행은 `매핑 실패` 배지와 함께 표시한다.                                                                                                                |
| **이카운트 업로드 (EcountImportUploader)** | 매입 업로드                | 파일 선택, 검증 결과, 행별 실패 사유, 업로드 확정/취소. 확정 전에는 장부 반영 상태와 구분한다.                                                                                                                        |
| **매입 잔량 근거 패널 (LotTracePanel)**    | 재고 금액 상세             | 매입일, 단가, 원수량, 잔량, 반영 금액, 업로드/수기 출처, 기간 필터를 보여준다.                                                                                                                                          |
| **민감 지표 제한 (SensitiveMetricGuard)**  | 지점장 장부/요약           | 지점장 권한에서 매출원가, 매출이익, 인당생산성, 영업이익이 표시되지 않도록 빈 자리 없이 대체 레이아웃을 제공한다. 보안 경계는 서버다.                                                                                 |
| **일괄 마감 다이얼로그 (BulkCloseDialog)** | 본사 관제판/마감           | 대상 장부 목록, 위험 알림, 명시 확인, 실행 결과를 표시한다.                                                                                                                                                            |
| **근무자 선택 (EmployeeWorkSelector)**     | 장부 근무 단계             | 직원 검색/선택, 여러 지점 근무 기록, 근무자 수 자동 산출을 제공한다.                                                                                                                                                    |
| **리사이징 데이터 테이블 (ResizableDataTable)** | 관제판/전체 재고/분석 | 컬럼 리사이징, 가로 스크롤, 필요 시 고정 컬럼을 제공한다. 숨김 컬럼은 권한 제어가 아니라 표시 편의일 뿐이다.                                                                                                          |

## Interaction Primitives

**터치 (모바일 · 태블릿):**

- 터치 타겟 최소 44×44px. 재고 행 편집 셀 포함.
- 재고 테이블: 셀 탭 → 인라인 편집 진입. 완료는 키패드 닫기 또는 다음 셀 탭.
- 단계 이동: 하단 "다음" 버튼 또는 하단 탭 탭.
- 스와이프: v1에서 도입 없음. 뒤로 가기는 헤더 뒤로 버튼.
- 숫자 입력 필드: `inputmode="numeric"` 또는 `"decimal"`. 키패드 자동 수치 모드.
- **경고 메시지 확인:** 경고 마커(⚠) 터치 시 비정상 수치에 대한 상세 검증 가이드 툴팁 노출, 빈 화면 터치 시 자동 닫힘.

**마우스 · 키보드 (데스크탑):**

- 관제판 행: 클릭 → 장부 상세. 호버 시 행 강조(`bg-muted/50`).
- 관제판 헤더: 현재는 쿼리 기반 문제 우선순/지점명순 정렬 버튼을 제공한다. 추가 범위에서 컬럼 리사이징을 제공할 수 있으며, 리사이징은 데이터 확인을 방해하면 안 된다.
- 재고 테이블: 셀 클릭 → 인라인 편집. `Tab`으로 다음 편집 가능 셀 이동. `Enter`로 편집 확정.
- 본사 업로드 매입 행: 클릭 → 업로드/원문 근거 확인. 지점장 권한에서는 편집 진입 없음.
- 재고 금액/전일 재고 값: 본사 권한 클릭 → `LotTracePanel`로 FIFO 매입 잔량 근거 확인.
- 이카운트 업로드: 파일 선택 → 검증 → 실패 행 검수 → 확정. 검증 실패 행이 있으면 확정 전 `매핑 실패` 상태를 먼저 처리한다.
- 단계형 폼: `Tab`/`Shift+Tab` 필드 순서 이동. 버튼 엔터.
- `Esc`: 열린 Dialog/Sheet 닫기, 인라인 편집 취소.
- 단축키 v1 없음. 커맨드 팔레트 v1 없음.

**금지 패턴:**

- 호버 전용 어포던스(모바일에서 동작 불가).
- 무한 스크롤(재고 테이블은 가상화 또는 30~50행 페이지).
- 드래그 재정렬 v1 없음.
- 모달 2단계 이상 스택.
- 브라우저 push 알림 없음. 추가 알림 범위는 LINE Messaging API 또는 텔레그램 발송 로그 화면으로 다룬다.
- **테이블 헤더 스크롤 분리:** 모바일 재고 테이블은 스크롤 시 반드시 Sticky Header로 상단에 고정한다.

## Accessibility Floor

동작 기준. 시각적 명도 대비는 `DESIGN.md`(shadcn WCAG AA 기본값 상속; primary/warning 오버라이드 대비 검증 필요).

- **WCAG 2.2 AA** 반응형 웹 전 서피스.
- 이상 신호는 아이콘 + 레이블 병행. 색상 단독 전달 금지.
- 장부 상태 배지는 색상 + 텍스트. 스크린 리더 aria-label: "현대점 장부 상태: 검토 대기".
- `Tab` 순서는 읽기 순서와 일치. `Esc`는 항상 최상위 모달/팝오버 닫기.
- 재고 테이블: 컬럼 헤더 `scope="col"`. 수정된 행 `aria-label` 추가: "수정됨".
- 단계형 폼 스텝 인디케이터: `aria-current="step"` 활성 단계 표시.
- 폼 오류: `aria-describedby`로 오류 메시지 연결. 포커스 첫 번째 오류 필드로 이동.
- 모달: `aria-modal="true"`, 포커스 트랩, 닫힌 후 트리거로 포커스 반환.

## Key Flows

### Flow 1 — 본사 김관리자가 아침 회의 전 문제 지점을 찾는다 (UJ-1)

1. 김관리자는 매일 07:40 PC에서 ERP Fish에 접속한다.
2. 로그인 즉시 관제판이 뜬다. 오늘 날짜 기준 전체 지점 7행이 테이블로 펼쳐진다. Skeleton이 1초 이하로 사라지고 실데이터가 채워진다.
3. 강서점 행 좌측에 빨간 보더가 보인다. `SignalChip` 2개: "매출차액 ₩48K 초과"(red), "이익률 21.4% ↓"(amber).
4. 김관리자는 강서점 행을 클릭한다. 강서점 당일 장부 상세 페이지가 열린다.
5. 헤더에 이상 신호 칩 반복. 스크롤하면 품목 재고 테이블이 나온다. 냉동/생물 탭. 특이 품목(꽃게 당일재고 급증) 한 눈에 확인.
6. **클라이맥스:** 김관리자는 상세 페이지에서 매입 누락 항목을 직접 추가 입력하고 "마감" 버튼을 누른다. `ClosingDialog`가 뜨고 확인한다. 관제판으로 돌아오면 강서점 행 상태 배지가 `검토대기 → 본사마감`으로 바뀐다. 이상 신호 칩은 해소된 것만 사라진다.
7. 회의 시작 전 3분, 김관리자는 7개 지점 모두 확인 완료. 열어봐야 할 지점을 미리 결정했다.

실패: 상세 페이지 로딩 실패 → 오류 상태 또는 재시도 안내를 표시한다.

---

### Flow 2 — 현대 지점장이 하루 장부를 입력한다 (UJ-2)

1. 현대 지점장은 퇴근 전 폰으로 ERP Fish를 연다. 오늘 장부 화면이 바로 뜬다. "오늘 장부: 2026-05-28, 현대점 · 입력 중" 헤더.
2. 단계 1 — 매출/결제: 총매출, 현금, 카드, 기타를 입력. 숫자 키패드 자동. 매출차액 실시간 계산 표시.
3. 단계 2 — 비용: 비용 항목 선택(코드 드롭다운) + 금액. 행 추가 버튼.
4. 단계 3 — 매입: 품목 마스터에서 선택 + 수량·단가. 합계 자동.
5. 단계 4 — 재고: **전일 이월 프리필** 상태로 열린다. "전일 재고를 불러왔습니다. 변경된 품목만 수정하세요." 안내 배너. 냉동 탭에서 꽃게·광어 수량 수정(수정된 행 파란 강조). 생물 탭 전환.
6. 단계 5 — 손실: 꽃게 일부 폐기. 품목 선택, 손실 유형 "폐기" 선택, 금액·사유 입력(필수).
7. 단계 6 — 근무자: 직원 목록에서 근무자를 선택한다. 특이사항 메모 선택 입력. 근무자 수는 선택 인원 기준으로 계산된다.
8. 단계 7 — 검토/제출: 지점장 권한에서는 현장 입력값, 매출 마진율, 재고 금액 중심의 요약을 본다. 매출원가, 매출이익, 인당생산성, 영업이익은 표시하지 않는다.
9. **클라이맥스:** "제출" 버튼. Toast: "장부를 제출했습니다." 상태 배지 `입력중 → 검토대기`. 지점장 화면에 "오늘 장부 제출 완료. 본사 검토를 기다리고 있습니다." 표시. 폰을 주머니에 넣는다.

실패: 단계 4에서 전일 재고 이월 실패(전일 마감 없음) → 알림 배너: "전일 장부가 마감되지 않아 자동 이월이 불가합니다. 직접 입력하거나 본사에 문의하세요." 수동 입력 허용.

---

### Flow 3 — 본사 이스태프가 누락 매입을 보완하고 마감한다 (UJ-3)

1. 이스태프는 못골점 장부가 `검토대기` 상태임을 관제판에서 확인한다.
2. 못골점 행 클릭 → 장부 상세 페이지. 변경 이력에서 지점장이 매입 입력을 빠트린 것 확인.
3. 매입 섹션 하단 "매입 추가" 버튼. 품목 마스터에서 광어 선택, 수량·단가 입력. "저장" → Toast "저장됐습니다." 본사 수정으로 이력 기록.
4. 계산값 재확인: 매출원가, 이익률 정상 범위. 이상 신호 없음.
5. **클라이맥스:** "마감" 버튼 → `ClosingDialog` 확인 → 마감 완료. 상태 배지 `검토대기 → 본사마감`. 이력에 "이스태프 · 본사마감" 타임스탬프 남음.

---

### Flow 4 — 본사 김관리자가 마감 후 오류를 정정 기록으로 반영한다 (UJ-4)

1. 회의 준비 중 현대점 매출차액 오류 발견. 이미 `본사마감`.
2. 관제판 현대점 클릭 → 장부 상세 페이지. 모든 원본 입력 필드 disabled. "마감된 장부입니다. 정정 기록을 추가할 수 있습니다." 안내.
3. `CorrectionPanel`에서 대상 항목 선택, 원본값 확인, 정정값 입력, 사유 필수.
4. 저장 → 장부 상세에 "정정 반영값" 레이블 표시. 총매출 수치 옆 "정정 반영" 태그. 원본값 토글로 확인 가능.
5. **클라이맥스:** 관제판 숫자는 정정 반영값으로 갱신. 리포트도 동일. 원본 감사 추적 유지. 회의 자료에는 최신 운영 숫자가 나온다.

---

### Flow 5 — 본사 이스태프가 이카운트 매입을 업로드하고 매핑 실패를 해결한다 (UJ-5)

1. 이스태프는 사이드바 `매입 업로드`에 들어간다.
2. 이카운트 엑셀 파일을 선택한다. 화면은 업로드 전 검증 상태를 보여준다.
3. 시스템은 품목명, 규격, 단가, 수량, 지점, 일자를 검증한다. 매핑 실패 3건이 `매핑 실패` 배지로 표시된다.
4. 이스태프는 실패 행을 눌러 `매핑 실패 검수`로 이동하고 원문 품목명을 분석용 품목/규격에 연결한다.
5. 업로드 결과 화면으로 돌아오면 실패 건이 사라지고 `업로드 확정` 버튼이 활성화된다.
6. **클라이맥스:** 확정 후 해당 장부의 매입 단계에는 `업로드 반영` 읽기 전용 행이 표시된다. 지점장은 이 행을 수정할 수 없고, 본사는 업로드 이력과 감사 로그를 확인할 수 있다.

### Flow 6 — 본사가 재고 금액의 FIFO 근거와 전체 지점 재고를 확인한다 (UJ-6)

1. 김관리자는 `전체 재고` 화면에서 꽃게를 검색한다.
2. 전체 합계와 지점별 잔량이 한 표에 표시된다. 일부 항목은 `확인 필요` 배지를 갖고 있다.
3. 김관리자는 현대점 재고 금액을 클릭한다.
4. `LotTracePanel`이 열리고 최근 1개월 매입일, 단가, 원수량, 잔량, 반영 금액이 표시된다.
5. **클라이맥스:** 김관리자는 특정 단가의 잔량이 오래 남은 것을 확인하고 발주 판단을 조정한다. 계산 불가 항목은 정상값처럼 숨겨지지 않는다.

### Flow 7 — 본사가 알림 조건과 월 손익을 운영한다 (UJ-7)

1. 본사 사용자는 `고정비`에서 매장별 월세와 세금 항목을 입력한다.
2. `월 손익계산서`에서 일일 비용과 월 고정비가 분리된 손익을 확인한다.
3. `알림 설정`에서 적자 지점, 목표 마진율 미달, 장기 체화 재고 조건과 발송 채널을 설정한다.
4. **클라이맥스:** 다음 예약 실행에서 LINE 또는 텔레그램으로 요약이 발송된다. 실패하면 `발송 실패` 상태와 재시도 로그가 남는다.
