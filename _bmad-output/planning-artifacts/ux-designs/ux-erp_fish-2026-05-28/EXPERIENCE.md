# ERP Fish — Experience Spine

> 멀티 서피스 반응형 웹. shadcn/ui + Tailwind CSS. 시각적 정체성은 `DESIGN.md` 참조. 스파인이 목업·와이어프레임과 충돌하면 스파인이 이긴다.

## Foundation

멀티 서피스 반응형 웹. shadcn/ui on Next.js + Tailwind CSS.

현재 기준 소스인 PRD는 `draft` 상태다. 이 UX 스파인은 최신 PRD와 `mvp-story-extraction-checklist.md`에 맞춘 정렬본이며, PRD 게이트와 Open Question이 닫히기 전까지 확정되지 않은 계산·알림·FIFO·민감 지표 노출 정책을 최종 동작처럼 표현하지 않는다.

사용자 역할은 설명상 **본사 사용자**와 **지점장**으로 묶지만, 화면 표시와 저장/변경 권한은 다음 권한 프로파일로 세분화한다: 대표/소유자, 본사 관리자, 본사 스텝, 업로드 담당자, 마감 담당자, 설정 관리자, 조회 전용 본사, 지점장. 역할 분기는 서버에서 강제하며, 클라이언트 분기에 의존하지 않는다.

| 권한 프로파일 | UX 기준 |
| --- | --- |
| 대표/소유자 | 전체 조회, 민감 지표 조회, 권한 위임, 계약/운영 설정 확인. 파괴적 변경은 확인 요약과 감사 로그를 요구한다. |
| 본사 관리자 | 전체 지점 조회, 직접 입력/수정, 마감, 정정, 리포트 조회. 권한/계정 변경은 설정 관리자 권한이 함께 있을 때만 노출한다. |
| 본사 스텝 | 전체 지점 조회, 누락 데이터 보완, 리포트 확인. 마감·권한 변경·민감 설정 변경은 숨기거나 비활성화하고 이유를 표시한다. |
| 업로드 담당자 | 이카운트 업로드 preview/commit/void/reprocess. 장부 직접 수정 권한과 구분한다. |
| 마감 담당자 | 개별 마감, 일괄 마감 dry run, 마감 실행. preflight 결과와 사유 입력을 통과해야 한다. |
| 설정 관리자 | 지점, 사용자, 권한, 코드, 이상 신호 기준값 관리. 변경 전/후 값과 사유 입력 UI를 제공한다. |
| 조회 전용 본사 | 허용된 지점/기간 조회와 리포트 조회. 입력/수정/마감/정정/API mutation은 차단 화면 또는 비활성 상태로 처리한다. |
| 지점장 | 연결된 지점의 장부 입력/조회. 타 지점, 민감 지표, 본사 전용 리포트, 본사 설정 mutation은 목록·상세·export·알림 템플릿·캐시 응답에서 차단한다. |

스토리 추출 기준은 `mvp-story-extraction-checklist.md`를 따른다. `MVP-S01~MVP-S03`만 구현 스토리로 추출 가능하며, `MVP-S04~MVP-S10`은 discovery 또는 정책 확정 작업으로 유지한다.

본사 사용자의 주 서피스는 데스크탑이다. 지점장의 주 서피스는 모바일(390px+)·태블릿이며 PC도 지원한다. 장부 입력 핵심 흐름은 모바일 퍼스트로 설계한다.

`DESIGN.md`가 시각적 정체성의 단일 출처다. 색상 토큰, 배지 스타일, 신호 칩, MetricCard, 재고 행 강조는 `DESIGN.md.Components`를 참조한다.

### 내비게이션 구조

**데스크탑 (본사 / 지점장 PC):** 본사는 흰색 좌측 사이드바 + 옅은 회색 본문 캔버스. 활성 메뉴는 primary blue 배경으로 표시한다. 항목은 현재 앱 라우트 구조를 유지하되 홈, 장부, 리포트, 기준정보, 감사 로그를 MVP 기본 묶음으로 둔다. 매입/재고, 인사, 손익, 알림 설정, 운영 로그는 릴리스 버킷과 권한 프로파일에 따라 숨김 또는 비활성 상태로 표시한다.

**모바일·태블릿 (지점장 주 서피스):** 하단 탭 바 56px + safe-area-inset. 탭: `장부` / `재고` / `손실`. `특이사항`은 별도 탭이 아니라 7단계 입력 흐름의 근무인원/특이사항 단계에 포함한다. 탭 이동은 현재 입력 단계를 유지하되 저장되지 않은 변경사항이 있으면 저장, 취소, 계속 편집 중 하나를 선택하게 한다.

**모달 스택:** 1단계 이하. Dialog 위에 Dialog 금지. 현 구현의 본사 장부 상세는 Sheet가 아니라 페이지 라우트다. 마감 확인과 정정 기록 입력은 같은 상세 페이지 안에서 처리한다.

**게이트 표기:** `blocked pending OQ`, `approved backlog candidate`, `approved discovery only`, `contract/ops only`인 항목은 기본 업무 내비게이션에서 완성 기능처럼 보이지 않는다. 표시가 필요하면 배지와 짧은 설명으로 왜 아직 실행할 수 없는지 알려준다.

## Component Patterns

동작 기준. 시각적 스펙은 `DESIGN.md.Components` 또는 shadcn 기본값 참조.

| 컴포넌트 | 사용처 | 동작 규칙 |
| --- | --- | --- |
| **관제판 행 (BranchRow)** | 관제판 | 행 전체 클릭 → 장부 상세 페이지로 이동. 이상 신호 있는 행은 `{components.dashboard-row.anomaly}` 또는 `{components.dashboard-row.anomaly-warning}` 처리. 매출차액은 OQ-1 전까지 임계값 “초과”가 아니라 부호 있는 차이값과 `기준 확인 필요`로 표시한다. |

| **PermissionActionGuard** | 전역 액션 | 권한 프로파일별로 숨김, 비활성, 차단 화면을 구분한다. 저장/변경 요청은 서버 차단이 최종 기준이며, UI는 “권한 없음” 대신 필요한 권한과 감사 기준을 짧게 설명한다. |
| **AuditEventTimeline / AuditSearch** | 장부 상세, 감사 로그 | 생성·수정·마감·정정·업로드·기준값 변경·권한 변경·일괄 마감 이벤트를 운영자가 읽을 수 있는 문장으로 표시한다. 지점, 일자, 사용자, 이벤트 유형, 대상 필드 기준 검색과 export를 제공한다. |
| **품목 재고 테이블 (InventoryTable)** | 장부 상세 / 지점 재고 입력 | 냉동/생물 탭 전환. 품목·규격·전일재고 후보·확정 이월 기준·매입·손실·당일재고·수량·상태 컬럼 표시. 재고금액과 원가 파생값은 지점장에게 숨기고, 본사 화면에서도 MVP 기본 계산값/FIFO 기준값/`기준 확인 필요`를 구분한다. 모바일/좁은 화면에서는 가로 스크롤과 sticky table header를 사용한다. |
| **단계형 입력 폼 (StepForm)** | 지점장 장부 입력 | 7단계: 매출/결제 → 비용 → 매입 → 재고 → 손실/폐기/떨이 → 근무인원/특이사항 → 검토/제출. 단계 이동 시 미저장 변경이 있으면 저장, 취소, 계속 편집 중 하나를 고른다. 모바일 네트워크 오류는 잃을 수 있는 입력값과 재시도 방법을 보여준다. |
| **작성자 표시값 (AuthorDisplayName)** | StepForm 1단계, 장부 상세 | 1단계에 입력한 작성자 이름을 같은 장부 저장 흐름 전체에 유지한다. 화면 표시용 작성자 이름은 인증 계정, 감사 로그 실행자, 실제 입력자와 구분한다. |
| **재고 행 인라인 편집 (InventoryRowEdit)** | 재고 입력 | 전일재고 후보를 프리필하되 확정값이 아니면 `검토 필요`, 전일 장부 누락은 `이월 공백`, 정정 영향은 `이월 재확인 필요`로 표시한다. 변경 즉시 `{components.inventory-row-modified}` 강조. |
| **이상 신호 칩 (SignalChip)** | 관제판 행, 장부 상세 헤더 | 아이콘(삼각 경고 또는 원) + 짧은 레이블. 경고 등급(amber), 심각 등급(red). 툴팁: 구체적 수치와 기준 출처. 복수 칩 허용. OQ가 열린 기준은 확정 임계값처럼 말하지 않는다. |
| **마감 사전점검 (ClosePreflight)** | 본사 장부 상세 | 마감 전 권한, 이미 마감, 숫자 오류, 업로드 실패/대기 배치, 미확정 정정, 필수 누락, 이월 공백, 가격 기준 없음, 이상 신호, 휴무일을 표로 요약한다. 차단 조건은 실행 불가, 예외 가능 조건은 사유 입력 후 개별 마감만 허용한다. |
| **정정 기록 패널 (CorrectionPanel)** | 본사 마감된 장부 상세 | 원본 잠금 시 대상 항목 선택, 원본값, 직전 정정 반영값, 새 정정값, 사유 입력. 영향 범위 식별 후 후속 장부와 리포트에 `정정 반영 재확인`을 표시하고, 본사 확정 후 기본 운영 숫자로 반영한다. |
| **장부 상태 배지 (StatusBadge)** | 관제판, 장부 상세 헤더 | 5종: 미입력·입력중·검토대기·본사마감·휴무. `{components.badge-*}` 스타일 적용. 항상 텍스트만, 아이콘 없음. |
| **검토/제출 요약 (ReviewSummary)** | 장부 입력 마지막 단계 | 필수 누락, 단계별 완료 여부, 이상 후보, 이월 공백, 가격 기준 상태를 한 번에 보여준다. 지점장에게는 원가·이익·재고금액·마진율 같은 민감 지표를 표시하지 않는다. |
| **SaveConflictDialog** | 단계형 폼, 장부 상세 | edit token 충돌 시 내 입력값, 서버 최신값, 마지막 수정자, 수정 시각을 나란히 보여준다. 같은 섹션 같은 필드 충돌은 사용자가 다시 저장할 값을 선택해야 한다. |
| **매핑 실패 검수 (ProductMappingReview)** | 매핑 실패 검수 | Extension B 게이트. 원문 품목명, 제안 품목, 분석용 품목/규격, 상태, 저장 액션. 실패 행은 `매핑 실패` 배지와 함께 표시한다. |
| **이카운트 업로드 (EcountImportUploader)** | 매입 업로드 | Extension B 게이트. `.xlsx` 선택 → 헤더 매핑 확인 → preview → 중복/매핑/필수 컬럼/숫자 오류 검수 → commit. preview 중 단가·수량·품목 매핑을 수정해도 원본 source identity와 preview edit version을 보존한다. |
| **UploadBatchState** | 매입 업로드 | 배치 상태는 `preview`, `committed`, `voided`, `reprocessed`, `failed`. void/reprocess는 사유와 작성자, 시각을 감사 로그에 남긴다. 마감 후 직접 commit/void는 허용하지 않고 정정 기록 흐름으로 처리한다. |
| **매입 잔량 근거 패널 (LotTracePanel)** | 재고 금액 상세 | Extension B, OQ-7/OQ-17 이후. 매입일, 단가, 원수량, 잔량, 반영 금액, 업로드/수기 출처, 기간 필터를 보여준다. 정책 전에는 `기준 확인 필요` 또는 MVP 기본 계산값으로 표시한다. |
| **민감 지표 제한 (SensitiveMetricGuard)** | 지점장 화면/API/export/캐시/알림 | 지점장 권한에서 매출원가, 매출이익, 영업이익, 이익률/마진율, 인당생산성, 재고금액, 원가 lot 근거, 본사 고정비, 타 지점 비교를 기본 차단한다. 보안 경계는 서버 응답과 mutation 권한이다. |
| **일괄 마감 다이얼로그 (BulkCloseDialog)** | 본사 관제판/마감 | Extension A 게이트. dry run으로 대상 지점/일자, 상태, 검증 오류, 이상 신호, 누락 장부, 휴무일, 이미 마감된 장부를 요약한다. 일부 실패해도 성공 건을 자동 rollback하지 않고, 실패한 적격 장부만 재시도한다. |
| **근무자 선택 (EmployeeWorkSelector)** | 장부 근무 단계 | Extension C, OQ-12 이후. MVP는 근무인원 숫자와 특이사항 메모만 다룬다. 직원 검색/선택, 여러 지점 근무 기록, 월 근무일수 집계는 gated 기능이다. |
| **ReportContractPanel** | 리포트 | 리포트별 필수 컬럼, 필터, 기본 정렬, export 컬럼 차이, chart 여부, 정정 반영값, 미마감/휴무/미입력 처리, 가격 신뢰 상태를 명시한다. |
| **AlertConfigPanel** | 알림 설정 | Extension D, OQ-13/OQ-16 이후. 수신자, 채널, 발송 시간, 조건, 임계값, 중복 억제 기간, test send, 실패/재시도, 템플릿 권한 필터를 설정한다. LINE은 LINE Notify가 아니라 Official Account/Messaging API 기준이다. |
| **리사이징 데이터 테이블 (ResizableDataTable)** | 관제판/전체 재고/분석 | 데스크탑 전용. 컬럼 리사이징, 가로 스크롤, 필요 시 고정 컬럼. 핵심 컬럼(지점명, 장부 상태, 마감 상태, 이상 신호, 최신 반영 시각)은 숨김 불가이며 최소 폭 아래로 줄일 수 없다. 숨김 컬럼은 권한 제어가 아니라 표시 편의일 뿐이다. |

## State Patterns

| 상태 | 서피스 | 처리 |
| --- | --- | --- |
| 데이터 로딩 | 관제판, 장부 상세 | shadcn `Skeleton` 행/블록. 관제판은 10개 내외 지점과 이상 신호를 3초 목표로 표시하고, 부분 실패 시 로드된 행과 실패 안내를 분리한다. |
| 장부 미입력 | 관제판 행 | `badge-empty`. 클릭 시 빈 장부 상세 진입(본사는 직접 입력 가능). |
| 장부 입력중 | 관제판 행 | `badge-draft`. 지점장·허용된 본사 프로파일 모두 수정 가능. |
| 검토 대기 | 관제판 행 | `badge-review`. 지점장 또는 본사가 제출한 상태. 원본 잠금 아님. |
| 본사 마감 | 관제판 행, 장부 상세 | `badge-closed`. 원본 입력 필드 전체 잠금. 본사 정정 기록만 추가 가능. |
| 휴무일 | 관제판 행, 리포트 | `badge-holiday`. 미입력과 시각적으로 구분하며 집계에서 별도 상태로 다룬다. |
| 지연 제출 / 미제출 | 관제판, 아침 회의 리포트 | 회의 기준 시각(기본 08:00) 이후 제출된 장부는 `지연 제출`, 기준 시각까지 없는 장부는 `미제출`로 표시한다. |
| 이상 신호 — 경고 | 관제판 행, 장부 상세 헤더 | `SignalChip` amber + 툴팁 수치. 마감 차단 아님. 기준이 OQ에 걸린 경우 “기준 확인 필요”를 함께 표시한다. |
| 이상 신호 — 심각 | 관제판 행, 장부 상세 헤더 | `SignalChip` red + 툴팁 수치. 마감 차단 아님. |
| 재고 행 — 전일재고 후보 | 재고 테이블 | 직전 저장 장부의 당일재고를 후보로 표시한다. 확정 이월 기준과 다르면 `검토 필요`. |
| 재고 행 — 이월 공백 | 재고 테이블, 마감 사전점검 | 전일 장부 누락 또는 월초 스냅샷 차이. `badge-carryover-gap`; 개별 마감은 사유 입력 필요, 일괄 마감은 차단. |
| 재고 행 — 이월 재확인 필요 | 재고 테이블, 리포트 | 늦은 마감이나 정정 영향으로 후속 장부를 다시 봐야 하는 상태. `badge-recheck-needed`. |
| 재고 행 — 수정됨 | 재고 테이블 | `{components.inventory-row-modified}` 강조. |
| 가격 신뢰 — approved | 마감 요약, 리포트 | 승인 단가 또는 유효한 마스터 단가. 기본 표시. |
| 가격 신뢰 — manual_override | 마감 요약, 리포트 | 사람이 단가를 바꾼 상태. `수동 단가 포함` 배지와 건수 집계. |
| 가격 신뢰 — pending_review | 마감 요약, 업로드 preview | 검토 전 단가. 마감 전 확인 필요, 일괄 마감 차단. |
| 가격 신뢰 — basis_missing | 마감 요약, 리포트 | 기준 단가 없음. `기준 확인 필요`; 0원으로 대체하지 않음. |
| 데이터 부족 | 기간 리포트 | 집계 대상 영업일, 재고 스냅샷, 비교 기간이 부족한 상태. `badge-data-insufficient`; 0과 구분. |
| 계산 불가 | 계산/리포트/export | 0으로 나누기, 서버 계산 실패, 정책 미확정. `badge-calculation-unavailable`; 원인 표시. |
| 폼 유효성 오류 | 단계형 폼 각 단계 | 필드 하단 인라인 오류 메시지. 제출 시 첫 번째 오류 필드로 포커스 이동. |
| 미저장 변경 | 단계형 폼 | 단계 이동 전 저장, 취소, 계속 편집을 선택하게 한다. |
| 부분 저장 | 단계형 폼 | 입력 손실 방지용 상태. 다른 사용자의 확정 저장을 덮어쓸 수 없음. |
| 저장 성공 | 단계형 폼 | shadcn `Toast`: "저장됐습니다." 2초 자동 닫힘. |
| 저장 실패 / 네트워크 오류 | 단계형 폼 | 재시도 버튼, 잃을 수 있는 입력값, 마지막 저장 시각을 표시한다. |
| 저장 충돌 | 단계형 폼, 장부 상세 | `SaveConflictDialog`. 내 입력값과 서버 최신값, 마지막 수정자, 시각을 비교해 해결한다. |
| 본사 수정 중 | 지점장 장부 | 본사 강제 수정 중인 섹션은 최신값 재확인 후 저장 가능. |
| 오래된 세션 | 전역 | 저장/변경 요청 차단 후 최신 장부 다시 불러오기와 재로그인 안내. |
| 권한 없음 | 메뉴, 상세, export | 직접 URL 접근 시 권한 차단 화면. 권한 프로파일과 필요한 권한을 설명하고 빈 에러 페이지로 두지 않는다. |
| 민감 지표 차단 | 지점장 화면/API/export/알림 | 원가·이익·마진율·재고금액·원가 lot 근거·타 지점 비교를 서버 응답부터 차단한다. 빈 자리 없이 비민감 요약으로 대체한다. |
| 정정 반영 대기 | 마감된 장부 상세 | 정정 기록이 추가됐지만 영향 범위 확정 전. |
| 정정 영향 식별 | 장부 상세, 리포트 | 영향받는 날짜 범위와 후속 장부 목록을 산정 중. |
| 정정 반영 재확인 | 관제판, 리포트, 후속 장부 | 원본 기준값과 정정 반영 후보값을 함께 노출. 본사 확정 전 기본 운영 숫자로 자동 확정하지 않는다. |
| 정정 반영 확정 | 관제판, 리포트 | 권한 있는 본사 프로파일이 영향 범위를 확인한 뒤 정정 반영값을 기본 운영 숫자로 사용한다. |
| 정정 무효화 | 정정 기록 | 삭제 대신 새 이벤트로 바로잡는다. |
| 업로드 preview | 매입 업로드 | 파싱·매핑 검증 후 commit 대기. 아직 장부 숫자에 영향 없음. |
| 업로드 committed | 매입 단계/장부 상세 | 매입 라인이 장부에 반영됨. 출처 배지와 읽기 전용 상태 표시. |
| 업로드 voided | 매입 업로드 | commit된 업로드를 무효화. 원본 이력 보존, 사유 필수. |
| 업로드 reprocessed | 매입 업로드 | 같은 원본 또는 수정 파일 재처리. 이전 배치와 연결해 추적. |
| 업로드 failed / 매핑 실패 | 매입 업로드/품목 검수 | 실패 사유를 사용자와 감사 로그에 표시. 장부 라인을 남기지 않음. |
| 발송 실패 | 알림 로그 | 실패 사유, 재시도 횟수, 다음 시도 여부. 민감한 메시지 본문 전체는 장기 보존하지 않는다. |
| 일괄 마감 위험 | 일괄 마감 다이얼로그 | dry run 결과, 차단/경고/예외 가능 항목, 부분 실패, 재시도 대상을 요약한다. |

## Interaction Primitives

**터치 (모바일 · 태블릿):**

- 터치 타겟 최소 44×44px. 재고 행 편집 셀 포함.
- 재고 테이블: 셀 탭 → 인라인 편집 진입. 완료는 키패드 닫기 또는 다음 셀 탭.
- 단계 이동: 하단 "다음" 버튼 또는 하단 탭 탭. 미저장 변경이 있으면 저장, 취소, 계속 편집을 먼저 고른다.
- 스와이프: v1에서 도입 없음. 뒤로 가기는 헤더 뒤로 버튼.
- 숫자 입력 필드: `inputmode="numeric"` 또는 `"decimal"`. 키패드 자동 수치 모드.
- **경고 메시지 확인:** 경고 마커(⚠) 터치 시 비정상 수치에 대한 상세 검증 가이드 툴팁 노출, 빈 화면 터치 시 자동 닫힘.
- 모바일 네트워크 오류: 마지막 저장 시각, 저장되지 않았을 수 있는 단계/필드, 재시도 버튼을 같은 화면 안에 표시한다.

**마우스 · 키보드 (데스크탑):**

- 관제판 행: 클릭 → 장부 상세. 호버 시 행 강조(`bg-muted/50`).
- 관제판 헤더: 기본 정렬은 이상 신호 있음, 미제출/지연, 미마감, 지점명 순서다. 실시간 검색/임의 컬럼 정렬은 별도 승인 전 필수 동작으로 보지 않는다.
- 대시보드 리사이징: 데스크탑에서만 제공한다. 지점명 최소 160px/최대 280px, 장부 상태 최소 96px, 마감 상태 최소 96px, 이상 신호 최소 160px, 최신 반영 시각 최소 132px 아래로 줄일 수 없다. 숫자 컬럼은 최소 112px, 금액 컬럼은 최소 128px. `기본값으로 되돌리기`를 제공한다.
- 재고 테이블: 셀 클릭 → 인라인 편집. `Tab`으로 다음 편집 가능 셀 이동. `Enter`로 편집 확정.
- 본사 업로드 매입 행: 클릭 → 업로드/원문 근거 확인. 지점장 권한에서는 편집 진입 없음.
- 재고 금액/전일 재고 값: 본사 권한 클릭 → Extension B와 FIFO 정책 확정 후 `LotTracePanel`로 근거 확인. 정책 전에는 `기준 확인 필요`와 현재 사용 중인 계산 기준을 표시한다.
- 이카운트 업로드: 파일 선택 → 헤더 매핑 → preview → 실패 행 검수/중복 선택 → commit. 검증 실패 행이 있으면 확정 전 `매핑 실패` 상태를 먼저 처리한다.
- 단계형 폼: `Tab`/`Shift+Tab` 필드 순서 이동. 버튼 엔터.
- `Esc`: 열린 Dialog/Sheet 닫기, 인라인 편집 취소.
- 리사이징 핸들: 포커스 가능해야 하며 좌/우 화살표로 8px 단위 조정, `Shift`+화살표로 24px 단위 조정한다.
- 단축키 v1 없음. 커맨드 팔레트 v1 없음.

**금지 패턴:**

- 호버 전용 어포던스(모바일에서 동작 불가).
- 무한 스크롤(재고 테이블은 가상화 또는 30~50행 페이지).
- 드래그 재정렬 v1 없음.
- 모달 2단계 이상 스택.
- 브라우저 push 알림 없음. 추가 알림 범위는 LINE Official Account/Messaging API 또는 텔레그램 발송 로그 화면으로 다룬다.
- 확정되지 않은 OQ 값을 화면에서 최종 계산값, 임계값, 마진율로 표시하지 않는다.
- 지점장 권한에서 민감 지표를 UI에서만 숨기고 API/export/cache/알림 템플릿에 남기는 패턴 금지.
- **테이블 헤더 스크롤 분리:** 모바일 재고 테이블은 스크롤 시 반드시 Sticky Header로 상단에 고정한다.

## Accessibility Floor

동작 기준. 시각적 명도 대비는 `DESIGN.md`(shadcn WCAG AA 기본값 상속; primary/warning 오버라이드 대비 검증 필요).

- **WCAG 2.2 AA** 반응형 웹 전 서피스.
- 이상 신호는 아이콘 + 레이블 병행. 색상 단독 전달 금지.
- 장부 상태 배지는 색상 + 텍스트. 스크린 리더 aria-label: "현대점 장부 상태: 검토 대기".
- `Tab` 순서는 읽기 순서와 일치. `Esc`는 항상 최상위 모달/팝오버 닫기.
- 재고 테이블: 컬럼 헤더 `scope="col"`. 수정된 행 `aria-label` 추가: "수정됨".
- 단계형 폼 스텝 인디케이터: `aria-current="step"` 활성 단계 표시.
- 폼 오류: `aria-describedby`로 오류 메시지 연결. 포커스 첫 번째 오류 필드로 이동.
- 모달: `aria-modal="true"`, 포커스 트랩, 닫힌 후 트리거로 포커스 반환.
- 리사이징 핸들: 키보드 포커스 가능, `role="separator"`, 현재 폭과 최소/최대 폭을 보조기기에 전달한다.
- 저장 충돌 다이얼로그: 내 입력값과 서버 최신값을 컬럼 헤더로 구분하고, 마지막 수정자/수정 시각을 스크린 리더가 읽을 수 있게 연결한다.
- 민감 지표 차단: 지점장 화면에서 숨긴 값은 aria-label, data attribute, export, 캐시 응답, 알림 템플릿에도 남기지 않는다.
- `기준 확인 필요`, `데이터 부족`, `계산 불가`는 색상만이 아니라 텍스트 배지와 설명으로 전달한다.

**모바일 재고 테이블:** 30개 이상 행은 가상화(windowed list) 또는 50행 단위 페이지로 처리. 편집 중 키패드가 올라와도 현재 셀이 가려지지 않도록 스크롤 보정. 지점장 모바일에서는 품목, 규격, 전일재고 후보, 매입/손실, 당일재고, 상태를 우선하고 재고금액·원가 근거는 표시하지 않는다.

**태블릿(모바일 landscape):** 재고 테이블 가로 스크롤 허용. 지점장 권한의 주요 컬럼은 품목·전일재고 후보·당일재고·상태다. 본사 권한에서는 재고금액과 가격 신뢰 상태를 추가로 볼 수 있다.

**본사 관제판 성능:** 10개 내외 지점 기준 지점 목록과 이상 신호 표시 목표는 3초 이내다. 3초를 넘기면 Skeleton을 유지하지 않고 부분 로드 상태, 마지막 갱신 시각, 재시도 액션을 보여준다.

## Key Flows

### Flow 1 — 본사 김관리자가 아침 회의 전 문제 지점을 찾는다 (UJ-1)

1. 김관리자는 매일 07:40 PC에서 ERP Fish에 접속한다.
2. 로그인 즉시 관제판이 뜬다. 오늘 날짜 기준 전체 지점 행이 테이블로 펼쳐진다. 3초 안에 지점 목록과 이상 신호가 채워지고, 실패한 행은 재시도 안내로 분리된다.
3. 강서점 행 좌측에 빨간 보더가 보인다. `SignalChip` 2개: "매출차액 +₩48K · 기준 확인 필요"(amber), "이익률 21.4% ↓"(amber). 매출차액 임계값은 OQ-1 전까지 확정 초과로 말하지 않는다.
4. 김관리자는 강서점 행을 클릭한다. 강서점 당일 장부 상세 페이지가 열린다.
5. 헤더에 이상 신호 칩 반복. 스크롤하면 품목 재고 테이블이 나온다. 냉동/생물 탭. 특이 품목(꽃게 당일재고 급증) 한 눈에 확인.
6. **클라이맥스:** 김관리자는 상세 페이지에서 매입 누락 항목을 직접 추가 입력하고 "마감" 버튼을 누른다. `ClosePreflight`가 필수 누락, 이월 공백, 가격 기준, 업로드 대기, 이상 신호를 요약한다. 차단 조건이 없고 필요한 사유를 입력하면 마감된다.
7. 회의 시작 전 3분, 김관리자는 7개 지점 모두 확인 완료. 열어봐야 할 지점을 미리 결정했다.

실패: 상세 페이지 로딩 실패 → 오류 상태 또는 재시도 안내를 표시한다.

---

### Flow 2 — 현대 지점장이 하루 장부를 입력한다 (UJ-2)

1. 현대 지점장은 퇴근 전 폰으로 ERP Fish를 연다. 오늘 장부 화면이 바로 뜬다. "오늘 장부: 2026-05-28, 현대점 · 입력 중" 헤더. 1단계에서 입력한 작성자 이름은 7단계 전체 흐름에 유지된다.
2. 단계 1 — 매출/결제: 총매출, 현금, 카드, 기타를 입력. 숫자 키패드 자동. 매출차액 실시간 계산 표시.
3. 단계 2 — 비용: 비용 항목 선택(코드 드롭다운) + 금액. 행 추가 버튼.
4. 단계 3 — 매입: 품목 마스터에서 선택 + 수량·단가. 합계 자동.
5. 단계 4 — 재고: **전일재고 후보 프리필** 상태로 열린다. 확정 이월 기준과 같으면 "전일 재고를 불러왔습니다. 변경된 품목만 수정하세요."가 보이고, 차이가 있으면 `검토 필요` 또는 `이월 공백` 배지가 붙는다. 냉동 탭에서 꽃게·광어 수량 수정(수정된 행 파란 강조). 생물 탭 전환.
6. 단계 5 — 손실: 꽃게 일부 폐기. 품목 선택, 손실 유형 "폐기" 선택, 금액·사유 입력(필수).
7. 단계 6 — 근무인원/특이사항: MVP에서는 근무인원 숫자와 특이사항 메모를 입력한다. 직원별 선택과 월 근무일수 집계는 Extension C/OQ-12 이후 기능이다.
8. 단계 7 — 검토/제출: 지점장 권한에서는 현장 입력값, 필수 누락, 비민감 검증 메시지, 이월/저장 상태를 본다. 매출원가, 매출이익, 영업이익, 이익률/마진율, 인당생산성, 재고금액, 원가 lot 근거는 표시하지 않는다.
9. **클라이맥스:** "제출" 버튼. Toast: "장부를 제출했습니다." 상태 배지 `입력중 → 검토대기`. 지점장 화면에 "오늘 장부 제출 완료. 본사 검토를 기다리고 있습니다." 표시. 폰을 주머니에 넣는다.

실패: 단계 4에서 전일 장부가 본사 마감 전이면 자동 이월 실패로 보지 않고 전일재고 후보를 `검토 필요`로 표시한다. 전일 장부 자체가 없으면 `이월 공백`으로 표시하고 수동 입력과 본사 확인 안내를 제공한다.

---

### Flow 3 — 본사 이스태프가 누락 매입을 보완하고 마감한다 (UJ-3)

1. 이스태프는 못골점 장부가 `검토대기` 상태임을 관제판에서 확인한다.
2. 못골점 행 클릭 → 장부 상세 페이지. 변경 이력에서 지점장이 매입 입력을 빠트린 것 확인.
3. 매입 섹션 하단 "매입 추가" 버튼. 품목 마스터에서 광어 선택, 수량·단가 입력. "저장" → Toast "저장됐습니다." 본사 수정으로 이력 기록.
4. 계산값 재확인: MVP 기본 계산값과 가격 신뢰 상태를 확인한다. FIFO 기준값이 필요한 항목은 정책 전까지 `기준 확인 필요`로 둔다.
5. **클라이맥스:** "마감" 버튼 → `ClosePreflight` 확인 → 마감 완료. 상태 배지 `검토대기 → 본사마감`. 이력에 "이스태프 · 본사마감" 타임스탬프와 검증 요약이 남음.

---

### Flow 4 — 본사 김관리자가 마감 후 오류를 정정 기록으로 반영한다 (UJ-4)

1. 회의 준비 중 현대점 매출차액 오류 발견. 이미 `본사마감`.
2. 관제판 현대점 클릭 → 장부 상세 페이지. 모든 원본 입력 필드 disabled. "마감된 장부입니다. 정정 기록을 추가할 수 있습니다." 안내.
3. `CorrectionPanel`에서 대상 항목 선택, 원본값 확인, 정정값 입력, 사유 필수.
4. 저장 → 상태가 `정정 반영 대기`가 되고, 시스템은 영향받는 날짜와 후속 장부를 `정정 영향 식별`로 산정한다.
5. **클라이맥스:** 영향 범위가 확인되면 후속 장부와 리포트에 `정정 반영 재확인`이 보인다. 권한 있는 본사 프로파일이 확정하면 관제판과 리포트의 기본 숫자가 `정정 반영 확정` 값으로 바뀐다. 원본 감사 추적은 유지된다.

---

### Flow 5 — 본사 마감 담당자가 일괄 마감 dry run을 검토한다 (Extension A / CAP-15)

1. 마감 담당자는 관제판에서 오늘 대상 지점들을 선택하고 `일괄 마감 dry run`을 실행한다.
2. 결과 표에는 대상 지점/일자, 현재 상태, 검증 오류, 이상 신호, 누락 장부, 휴무일, 이미 마감된 장부가 구분된다.
3. 숫자 형식 오류, 업로드 대기 배치, 미확정 정정은 차단으로 표시된다. 이상 신호와 휴무일은 경고로 표시된다.
4. 검증 오류나 이월 공백이 있는 장부는 일괄 마감에서 차단된다. 사유 입력 예외는 개별 마감에서만 허용된다.
5. **클라이맥스:** 실행 후 성공/실패 건수와 실패 사유가 표시된다. 성공한 장부는 자동 rollback하지 않고, 실패한 적격 장부만 재시도 대상으로 남는다.

---

### Gated Extension Flows

아래 흐름은 최신 PRD의 승인 추가 구현 범위다. 각 Flow는 해당 OQ와 승인 산출물이 닫히기 전에는 구현 완료 화면처럼 취급하지 않는다.

### Flow 6 — 본사 이스태프가 이카운트 매입을 업로드하고 매핑 실패를 해결한다 (Extension B / CAP-6)

1. 이스태프는 사이드바 `매입 업로드`에 들어간다.
2. 이카운트 `.xlsx` 파일을 선택한다. 헤더 매핑 화면에서 매입일자, 지점/출고처, 원문 품목명, 규격, 수량, 단가, 매입금액이 논리 필드에 연결됐는지 확인한다.
3. preview 결과에서 품목 매핑 실패, 중복 가능성, 필수 컬럼 누락, 숫자 형식 오류가 행 단위로 표시된다.
4. 이스태프는 실패 행을 눌러 원문 품목명을 분석용 품목/규격에 연결하고, 필요한 경우 preview 안에서 단가·수량·품목 매핑을 수정한다.
5. 업로드 결과 화면으로 돌아오면 실패 건이 사라지고 `commit` 버튼이 활성화된다. 중복 후보는 덮어쓰기, 추가, 취소 중 하나를 선택한다.
6. **클라이맥스:** commit 후 해당 장부의 매입 단계에는 `업로드 반영` 읽기 전용 행이 표시된다. void/reprocess가 필요하면 새 이벤트로 처리하고 원본 업로드 이력과 감사 로그를 보존한다.

---

### Flow 7 — 본사가 재고 금액의 FIFO 근거와 전체 지점 재고를 확인한다 (Extension B / CAP-7, CAP-8)

1. 김관리자는 `전체 재고` 화면에서 꽃게를 검색한다.
2. 전체 합계와 지점별 잔량이 한 표에 표시된다. FIFO 정책이 닫히지 않은 항목은 `기준 확인 필요` 배지를 갖고 있다.
3. 김관리자는 현대점 재고 금액을 클릭한다.
4. `LotTracePanel`이 열리고 최근 1개월 매입일, 단가, 원수량, 잔량, 반영 금액이 표시된다.
5. **클라이맥스:** 김관리자는 특정 단가의 잔량이 오래 남은 것을 확인하고 발주 판단을 조정한다. 계산 불가 항목은 정상값처럼 숨겨지지 않는다.

---

### Flow 8 — 본사가 알림 조건과 월 손익을 운영한다 (Extension D / CAP-10, CAP-11)

1. 본사 사용자는 `고정비`에서 매장별 월세와 세금 항목을 입력한다.
2. `월 손익계산서`에서 일일 비용과 월 고정비가 분리된 손익을 확인한다. 지점장 권한과 공유 링크/export에서는 본사 전용 손익 지표가 차단된다.
3. `알림 설정`에서 적자 지점, 목표 마진율 미달, 장기 체화 재고 조건, 발송 채널, 발송 시간, 중복 억제 기간, 수신자 권한 템플릿을 설정한다.
4. 운영자는 test send로 수신자와 메시지 형식을 확인한다. LINE은 LINE Official Account와 Messaging API 기준이다.
5. **클라이맥스:** 다음 예약 실행에서 LINE 또는 텔레그램으로 요약이 발송된다. 실패하면 `발송 실패` 상태와 재시도 로그가 남고, 로그에는 수신자·템플릿 id·주요 이벤트 id만 남긴다.
