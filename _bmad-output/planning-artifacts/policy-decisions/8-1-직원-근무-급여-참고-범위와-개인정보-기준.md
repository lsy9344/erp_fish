## 기존 MVP 근무 데이터 전환 원칙

현재 MVP는 직원별 근무 기록을 저장하지 않는다. 장부에는 `DailyLedger.workerCount` 숫자와 `DailyLedger.workMemo` 메모가 저장되며, 현재 근무 입력 UI도 근무인원 textbox와 특이사항 textarea를 다룬다.

| 항목 | 결정 |
| --- | --- |
| 기존 `workerCount` 자동 분해 | 금지. 기존 숫자를 특정 직원별 `LedgerWorker` 기록으로 자동 분해하지 않는다. |
| 기존 `workMemo` 자동 파싱 | 금지. 자유 메모에서 직원명이나 근태 사유를 추출해 구조화하지 않는다. |
| CAP-9 도입 전 장부 표시 | 직원별 집계 불가로 표시한다. 필요하면 본사 수동 보정 또는 별도 이관 상태를 남긴다. |
| 과거 월 급여 참고 export | CAP-9 도입 전 기간은 직원별 근무일수 칸을 비우거나 "직원별 집계 불가"로 표시한다. |
| 본사 수동 보정 | 본사가 별도 근거를 확인해 수동 입력한 경우에만 수동 보정 표시와 감사 로그를 남긴다. |

후속 CAP-9 구현 story는 기존 MVP 장부를 직원별 근무 기록으로 조작하지 않고, CAP-9 도입 이후 입력된 직원별 선택 기록만 직원별 월 근무일수 집계에 사용해야 한다.

## 권한, Export, 감사 로그 기준

### 권한 기준

| 대상 | 기준 |
| --- | --- |
| 직원 정보 생성/수정/비활성 | 본사 권한 프로파일로 제한한다. 지점장은 직원 마스터를 생성하거나 퇴사/비활성 처리할 수 없다. |
| 지점장 직원 선택 | 자기 지점 근무 입력에 필요한 직원 표시명, 중복 구분값, 근무 가능 상태만 조회할 수 있다. |
| 타 지점 전용 직원 | 지점장 기본 응답에 포함하지 않는다. 여러 지점 근무 가능 직원은 승인된 배정 범위 안에서만 표시한다. |
| 급여 차액 | 지점장에게 기본 노출하지 않는다. 본사 급여 참고 권한자만 조회/수정한다. |
| 특수 근태 메모 | field-level 권한을 둔다. 지점장은 자기 입력에 필요한 제한 범위만 볼 수 있고 본사 전용 민감 메모는 보지 않는다. |
| 입사/퇴사 정보 | 본사 기본 조회, 지점장은 선택 가능 여부에 필요한 최소 상태만 표시한다. |

### 참고 범위 구현 story 생성 가능 조건

1. 위 승인 테이블이 완료되어야 한다.
2. 구현 story가 실제 지급 확정, 세무/노무 판단, 자동 지급을 포함하지 않아야 한다.
3. 직원/급여 개인정보 보존표가 acceptance criteria에 연결되어야 한다.
4. 지점장 응답, 본사 export, audit log, cache 경계 테스트가 포함되어야 한다.
5. 기존 `workerCount` 자동 분해 금지가 구현 story의 명시적 회귀 테스트 또는 검증 항목으로 유지되어야 한다.

## 최종 정책 메모

Story 8.1의 산출물은 CAP-1/CAP-9를 "직원/근무/급여 참고" 범위로 제한한다. 이 문서만으로 실제 급여 지급 확정 기능을 구현할 수 없다. 급여 지급 확정, 자동 지급, 법정 임금/세무/노무 판단이 필요하다는 결론이 나오면 별도 PRD 또는 change proposal을 먼저 작성해야 한다.
