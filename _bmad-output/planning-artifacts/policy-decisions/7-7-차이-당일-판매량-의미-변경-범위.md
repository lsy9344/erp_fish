## 현재 의미

현재 재고 입력 테이블의 `차이`는 기준재고와 당일재고 사이 수량 차이를 표시하는 UI 용어다. 운영자가 입력한 당일재고가 기준재고와 다르면 그 차이를 보여주고, 필요하면 조정 사유를 남긴다.

현재 계산식은 다음 의미를 유지한다.

| 항목 | 현재 의미 |
| --- | --- |
| `differenceQuantity` | `afterQuantity - beforeQuantity` |
| `differenceAmount` | `afterAmount - beforeAmount` |
| `beforeQuantity` | 조정 전 기준 수량 |
| `afterQuantity` | 조정 후 당일 수량 |
| 재고 조정 사유 | `differenceQuantity`가 발생한 이유를 운영자가 설명하는 값 |
| 이상 신호 | `differenceQuantity`가 threshold를 넘으면 `inventory-difference-exceeded` 후보가 된다. |

`salesDifference`/`매출 차이`는 매출 지표다. Story 7.7은 재고 입력 테이블의 `차이` 문구만 다루며, `salesDifference` label, export column, 계산식, anomaly 기준을 변경하지 않는다.

## 당일 판매량 의미 결정

MVP-S10/CAP-17의 `당일 판매량`은 실제 판매 수량이 아니라 "기준재고와 당일재고의 차이"를 현장 사용자가 이해하기 쉽게 부르는 표시 용어로 결정한다.

단, 이 표현은 실제 POS 판매량처럼 오해될 수 있으므로 다음 copy guardrail을 둔다.

1. 첫 적용 화면은 `당일 판매량(현재 차이 수량)` 같은 과도기 표기를 우선 검토한다.
2. 컬럼 폭이 좁아 `당일 판매량`만 표시해야 하면 tooltip/help text를 함께 제공한다.
3. 도움말은 "전일재고, 매입, 손실을 반영한 기준재고와 입력한 당일재고의 차이입니다. 실제 POS 판매 수량과 다를 수 있습니다."로 둔다.
4. 본사 내부 계산 용어는 `differenceQuantity`/`재고 차이 수량`을 유지하고, 화면 표시 용어만 `당일 판매량`으로 매핑한다.

## 예시 재고 행

| 필드 | 값 |
| --- | ---: |
| 전일재고 | 100 |
| 매입 | 20 |
| 손실 | 5 |
| 기준재고 | 115 |
| 당일재고 | 92 |
| 현재 차이 수량 | -23 |
| 조정 사유 | 영업 중 판매 및 실사 차이 확인 필요 |

### 단순 라벨 변경 결과

단순 라벨 변경이면 표시 숫자는 현재 `차이`와 동일하다.

| 표시 항목 | 값 | 설명 |
| --- | ---: | --- |
| 현재 `차이` | -23 | `differenceQuantity = afterQuantity - beforeQuantity` |
| 변경 후 `당일 판매량(현재 차이 수량)` | -23 | 계산식과 저장값은 바꾸지 않고 라벨만 바꾼다. |

이 경우 `-23`은 실제 POS 판매 23개를 확정한다는 뜻이 아니다. 기준재고 115와 당일재고 92의 차이를 현장 친화적으로 표시한 값이다.

### 계산 의미 변경 결과

계산 의미 변경으로 `당일 판매량`을 실제 판매 수량으로 정의하려면 이 예시만으로는 계산할 수 없다. 최소한 다음 반영 순서와 source of truth가 필요하다.

| 반영 후보 | 정책 필요 항목 |
| --- | --- |
| 판매 | POS/수기 매출 수량과 재고 품목 매핑 기준 |
| 손실 | 손실, 폐기, 떨이 수량을 판매량에서 제외할지 여부 |
| 조정 | 실사 조정을 판매량으로 볼지, 별도 재고 보정으로 볼지 여부 |
| 반품 | 당일 판매량에서 차감할지, 재고 입고로 볼지 여부 |
| 이월 | 전일재고 후보와 확정 이월 기준 |

계산 의미 변경은 OQ-14 승인만으로 구현하지 않는다. 리포트, 이상 신호, 재고 이월, 손실/조정 반영 기준을 포함한 별도 계산 정책과 구현 story로 승격한다.

## 계산 의미 변경 범위

계산 의미 변경으로 승인하려면 별도 정책 산출물에서 아래 범위를 먼저 닫아야 한다.

### 재고 흐름

전일재고, 매입, 손실/폐기/떨이, 실제 판매, 당일재고, 조정의 관계를 새 식으로 정의한다. 예: `실제 판매 수량 = 전일재고 + 매입 - 손실 - 폐기/떨이 - 당일재고 +/- 조정`처럼 후보 식을 만들 수 있으나, 반품과 실사 조정 처리 정책 없이는 승인하지 않는다.

### 리포트

월간 재고 흐름, 지점별 기간 비교, 일별 아침 회의 리포트가 새 의미를 어느 컬럼명으로 표시할지 정한다. 기존 `재고 차이`와 새 `실제 판매 수량`이 동시에 필요한지 결정한다.

### 이상 신호

`inventory-difference-exceeded`가 계속 재고 실사 차이를 뜻하는지, 실제 판매량 이상 신호를 별도로 만들지 결정한다. 실제 판매량 이상 신호를 만들면 threshold, 사유, 경고 copy, 본사/지점장 노출 범위를 새로 정한다.

### 손실/조정

손실 저장 후 기존 재고 조정의 기준 수량과 차이를 재계산하는 현재 흐름을 유지할지 변경할지 정의한다. 손실과 조정이 실제 판매량에 중복 반영되지 않도록 순서를 확정해야 한다.

### 민감 필드

금액 기반 `differenceAmount`, `inventoryAmount`, 원가/이익 역추정 가능 값은 Story 7.6 지점장 차단 매트릭스를 따른다. `당일 판매량` 라벨이 지점장에게 금액 기반 차이나 원가 근거를 새로 노출하는 근거가 될 수 없다.

## 최종 정책 메모

Story 7.7의 권장 결론은 "단순 라벨 변경"이다. `당일 판매량`은 실제 판매 수량 계산을 새로 도입하는 용어가 아니라, 현재 재고 테이블의 `차이` 수량을 현장 친화적으로 표시하기 위한 용어다. 이 결정으로 CAP-17은 UI/copy/export-label story로 승격할 수 있지만, 계산 의미 변경은 별도 정책 승인 전까지 구현하지 않는다.
