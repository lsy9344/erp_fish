# ERP Fish 기능 추가/수정/보완 작업 지시서

## 1. 작업 목적

현재 ERP Fish는 1차 MVP 범위인 지점 일일 장부 입력, 본사 관제, 본사 마감/정정, 기본 리포트까지 구현된 상태다.

이번 작업은 2026-06-08 의뢰자 미팅 결과와 의뢰자 제공 엑셀 파일을 기준으로, MVP에서 빠졌거나 얇게 구현된 업무 기능을 추가/수정/보완하기 위한 것이다.

작업 시 다음 원칙을 지킨다.

- 기존 PRD와 구현된 기능을 무시하지 않는다.
- 의뢰자 제공 엑셀에 실제로 있던 업무는 중요한 근거로 본다.
- 새 기능과 기존 보완 기능을 구분한다.
- 기능을 한 번에 모두 만들기보다, 데이터 구조가 흔들리는 부분부터 먼저 잡는다.

- 인당생산성, 최고매출품목, 최고매출액
- 근무인원, 팀장, 팀원
- 매출차액

## 4. 작업 범위 A: 엑셀에는 있었지만 MVP에서 빠지거나 축소된 기능

이 범위는 의뢰자 입장에서는 "기존 엑셀에 있던 업무"로 볼 수 있다. 완전 신규라기보다, 웹 ERP에서 정식 기능으로 복원하거나 고도화하는 성격이다.

### A-1. 직원 마스터와 월간 근무일수 집계

현재 MVP는 근무인원과 특이사항 중심이다. 엑셀에는 직원 목록과 월간 근무표가 있다.

구현 요구:

- 직원 등록/수정/비활성 관리
- 직원 소속 지점 관리
- 일일 장부에서 근무자 선택
- 일자별 근무자 저장
- 월간 직원별 근무일수 자동 집계
- 지점별/직원별 근무 현황 조회

검수 조건:

- 지점장이 일일 장부에서 근무자를 선택할 수 있다.
- 같은 직원이 여러 지점에서 근무한 기록도 저장할 수 있다.
- 본사는 월별로 직원별 근무일수를 볼 수 있다.

### A-2. 특수기간 비교 리포트

`total_sales.xlsx`에는 `설날매출비교` 시트가 있다.

구현 요구:

- 설날 등 특수기간 템플릿 관리
- D-day 기준 비교
- 연도별 매출, 마진, 근무인원 비교
- 지점별 특수기간 리포트

검수 조건:

- 본사가 특수기간과 기준일을 등록할 수 있다.
- 본사는 선택한 특수기간의 연도별 지점 실적을 볼 수 있다.

### A-3. 엑셀 보고 양식 수준의 리포트 고도화

기본 리포트는 구현되어 있으나, 엑셀에는 대조 기간/현재 기간 비교, 연도별 비교, 월별 추이, 보고 양식이 더 넓게 존재한다.

구현 요구:

- 대조 기간과 현재 기간 비교
- 연도별 지점 비교
- 월별 매출/이익률/생산성 추이
- 보고용 화면 또는 출력 양식
- 기존 엑셀 항목과 신규 리포트 항목 매칭

검수 조건:

- 본사가 기간 두 개를 선택해 지점별 증감을 볼 수 있다.
- 본사가 연도별/월별 지점 실적을 볼 수 있다.
- 숫자 기준은 기존 서버 계산 로직을 사용한다.

### A-4. 상품별 관리자 분석

엑셀 일자 시트에는 상품별 관리자 분석 영역이 있다.

구현 요구:

- 품목규격 기준 분석
- 전재고, 매입, 판매원가 계산
- 상품별 이익률 계산
- 판매금액과 판매량 계산
- 최고매출품목 자동 산출
- 품목별 판매/마진 분석 화면

검수 조건:

- 본사는 장부별 상품 분석을 볼 수 있다.
- 최고매출품목이 자동 계산된다.
- 계산 기준이 불확실한 항목은 임의로 확정하지 않고 "기준 확인 필요"로 표시한다.

### A-5. 품목명/규격 정규화

엑셀에는 `꽃게 [사게]` 같은 품목명을 분석용 품목규격으로 바꾸는 수식이 있다.

구현 요구:

- 원문 품목명 저장
- 품목명/규격 분리
- 품목 별칭 관리
- 매입 품목과 재고 품목 매핑
- 매핑 실패 검수 화면

검수 조건:

- 본사는 매핑 실패 품목을 확인하고 직접 연결할 수 있다.
- 과거 장부 원문 품목명은 보존된다.
- 이 기능은 매입 엑셀 업로드와 연결 가능해야 한다.

### A-6. 이카운트 엑셀 업로드 기반 매입 자동 생성

이 기능은 추가견적 항목으로 분류하지 않는다. 우리가 무료 제공하는 기능으로 처리한다. 단, 실제 구현 작업에는 포함한다.

구현 요구:

- 이카운트 엑셀 업로드
- 업로드 파일 검증
- 품목명, 규격, 단가, 수량 파싱
- 품목 매핑
- 본사 업로드 데이터를 장부 매입으로 자동 생성
- 지점 화면에는 읽기 전용으로 노출
- 업로드 이력과 감사 로그 저장

검수 조건:

- 본사는 이카운트 엑셀을 업로드할 수 있다.
- 업로드 실패 사유가 사용자에게 보인다.
- 업로드된 매입은 해당 지점/일자 장부에 반영된다.
- 지점장은 본사 업로드 매입을 임의 수정할 수 없다.

## 5. 작업 범위 B: 회의에서 새로 확정되거나 크게 확장된 기능

이 범위는 기존 MVP보다 명확히 커지는 기능이다. 추가견적 대상이다.

### B-1. FIFO 재고 금액 계산과 매입 잔량 이력

이 기능은 적용한다. 다만 추가견적 항목에는 포함하지 않는다.

구현 요구:

- 매입 라인별 잔여 수량 관리
- FIFO 방식으로 재고 금액 계산
- 매입일, 단가, 수량, 잔량 저장
- 전일 재고 또는 재고 금액 클릭 시 근거 팝업
- 기본 1개월 조회와 기간 필터
- 손실, 정정, 재고 조정과 연결

검수 조건:

- 같은 품목의 단가가 여러 번 바뀌어도 잔량별 원가가 추적된다.
- 본사는 특정 재고 금액의 매입 근거를 볼 수 있다.
- FIFO 계산이 불가능한 데이터는 조용히 넘어가지 않고 확인 필요로 표시된다.

### B-2. 본사 통합 전체 재고 뷰

구현 요구:

- 전체 지점 품목별 재고 집계
- 지점별 잔량 비교
- 전체 합계
- 냉동/생물/품목/기간 필터
- 본사 발주 판단용 화면

검수 조건:

- 본사는 특정 품목이 각 지점에 얼마나 남았는지 한 화면에서 본다.
- 본사는 전체 지점 합산 수량을 볼 수 있다.

### B-3. 인사/급여 정산

구현 요구:

- 직원 입사일 관리
- 일당제 근무 기록
- 지각/조퇴/특수 상황 메모
- 급여 차액 수동 입력
- 여러 지점 근무 통합
- 개인별 월급 정산 페이지

검수 조건:

- 본사는 직원별 월 근무 내역을 볼 수 있다.
- 본사는 급여 차액을 수동으로 입력할 수 있다.
- 여러 지점에서 근무한 직원의 월 정산이 합산된다.

### B-4. 본사 고정비 입력과 월 손익계산서

구현 요구:

- 매장별 월 고정비 입력
- 고정비 항목 관리
- 일일 비용과 월 고정비 구분
- 월 손익계산서 계산
- 매장별 손익 비교

검수 조건:

- 본사는 매장별 월세, 세금 등 고정비를 입력할 수 있다.
- 월 손익계산서에는 일일 비용과 월 고정비가 구분된다.

### B-5. 라인 또는 텔레그램 자동 알림

LINE 알림은 기술적으로 가능하다. 단, `LINE Notify`가 아니라 `LINE Messaging API`와 `LINE Official Account`를 사용해야 한다. LINE Notify는 2025-03-31 종료되었으므로 사용하지 않는다.

구현 요구:

- LINE Messaging API 또는 텔레그램 API 연동
- LINE 사용 시 LINE Official Account 생성 및 Messaging API 활성화
- LINE channel access token 관리
- LINE 대상 `userId`, `groupId`, `roomId` 수집 및 저장
- 발송 대상과 채널 설정
- 정기 스케줄러
- 적자 지점, 목표 마진율 미달, 장기 체화 재고 조건 계산
- 발송 실패/재시도/로그
- 운영 토큰 보안 관리

검수 조건:

- 지정 시간에 요약 알림이 발송된다.
- 발송 실패가 로그에 남는다.
- 본사는 알림 조건을 확인할 수 있다.
- LINE 사용 시 알림 대상은 Official Account 친구이거나, 알림용 그룹/room에 Official Account가 참여 중이어야 한다.

참고:

- LINE Notify 종료 공지: `https://developers.line.biz/en/news/2025/04/01/line-notify/`
- LINE Messaging API 시작 문서: `https://developers.line.biz/en/docs/messaging-api/getting-started/`
- LINE 메시지 발송 문서: `https://developers.line.biz/en/docs/messaging-api/sending-messages/`
- LINE Messaging API 요금 문서: `https://developers.line.biz/en/docs/messaging-api/pricing/`

### B-6. AI 기반 경영 분석 확장성

이번 구현에서 AI 분석 기능은 작업하지 않는다. 에이전트는 AI 챗봇, 자연어 질의, 자동 분석 화면, AI 요약 기능을 구현하면 안 된다.

현재 허용되는 범위는 새 기능을 만들 때 날짜, 지점, 직원, 품목, 금액 같은 기본 데이터를 구조적으로 저장해 두는 것뿐이다. 이것은 향후 확장을 막지 않기 위한 데이터 정리이지, AI 기능 구현이 아니다.

구현 요구:

- 주요 변경/입력 이력을 구조화해서 저장
- 분석에 필요한 날짜, 지점, 직원, 품목, 금액 기준을 명확히 유지
- 자연어 AI 분석은 별도 고도화 범위로 분리

검수 조건:

- 새 기능이 중요한 운영 데이터를 텍스트 메모에만 묻어두지 않는다.
- 향후 분석 가능한 구조로 저장한다.
- AI 화면, AI API 호출, AI 프롬프트, AI 분석 결과 저장 기능이 추가되지 않는다.

## 6. 기존 기능 보완/정책 변경

다음 항목은 완전 신규 기능은 아니지만 기존 화면과 정책을 바꿔야 한다.

### C-1. 지점장 민감 회계 지표 숨김

지점장 화면에서는 다음 지표를 숨긴다.

- 매출원가
- 매출이익
- 인당생산성
- 영업이익

지점장 화면에는 현장 입력값, 매출 마진율, 재고 금액 중심으로 보여준다.

### C-2. 희망 판매가 기준 손실액 계산

손실액은 단순 매입원가 기준이 아니라, 지점장이 영업 전에 설정한 희망 판매가와 실제 처리 금액의 차이로 계산하는 정책을 검토한다.

필요 데이터:

- 품목별 희망 판매가
- 실제 판매/떨이/폐기 처리 금액
- 손실 사유

### C-3. 본사 일괄 마감 또는 검토 생략 토글

본사가 지점 장부를 개별 검토하지 않고 바로 마감할 수 있는 옵션이다.

주의:

- 권한은 본사만 가진다.
- 감사 로그가 남아야 한다.
- 일괄 마감 전에 위험 알림을 표시해야 한다.

### C-4. 작성자 이름 세션 캐싱

1단계에서 작성자 이름을 한 번 입력하면 7단계 전체 저장에 같은 작성자명이 따라가야 한다.

### C-5. 재고 테이블 문구 변경

기존 `차이` 항목을 `당일 판매량`으로 바꾸는 요청이다.

주의:

- 단순 라벨 변경인지 계산 의미 변경인지 먼저 확인한다.
- 계산 의미가 바뀌면 리포트와 이상 신호도 함께 확인한다.

### C-6. 대시보드 그리드 리사이징

본사 대시보드 테이블의 컬럼 또는 영역 크기를 사용자가 조절할 수 있게 하는 기능이다.

## 7. 권장 구현 순서

1. 품목명/규격 정규화
2. 이카운트 업로드
3. FIFO 재고 금액 계산
4. 매입 잔량 이력 팝업
5. 본사 통합 전체 재고 뷰
6. 지점장 민감 지표 숨김 및 매입 수정 제한
7. 직원 마스터와 월간 근무일수 집계
8. 리포트 고도화와 특수기간 비교
9. 본사 고정비와 월 손익계산서
10. 인사/급여 정산
11. 라인/텔레그램 알림

## 8. 개발 시 주의사항

- 기존 장부 원본값을 덮어쓰지 않는다.
- 본사 마감 후 원본 수정은 계속 막아야 한다.
- 본사 수정, 정정, 업로드, 일괄 마감은 모두 감사 로그를 남긴다.
- 지점장 권한 제한은 UI가 아니라 서버에서 강제한다.
- 엑셀 수식을 그대로 복사하지 말고 서버 계산 로직으로 재정의한다.
- 계산 기준이 불명확한 항목은 임의로 확정하지 않는다.
- 외부 연동 기능은 실패/재시도/로그를 반드시 포함한다.
- AI 분석 기능은 이번 작업 범위가 아니다. AI 관련 구현을 추가하지 않는다.
