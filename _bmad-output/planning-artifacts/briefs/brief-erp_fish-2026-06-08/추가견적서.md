# ERP Fish 회의 결과 정리

## 1. 이번 회의에서 정리된 방향

이번 회의에서는 기존에 만든 ERP를 실제 매장 운영 방식에 더 잘 맞게 보완해야 한다는 점이 확인되었습니다.

처음 개발 범위는 지점장이 하루 장부를 입력하고, 본사가 전체 지점 현황을 보고, 필요한 경우 수정/마감하는 데 초점을 맞췄습니다.

그런데 의뢰자께서 주신 기존 엑셀 파일을 다시 보면, 엑셀 안에는 단순 장부보다 더 많은 업무가 들어 있었습니다.

- 직원별 근무표
- 상품별 분석
- 설날 같은 특수기간 비교
- 매장별 월/연도 비교
- 품목명과 규격 정리
- 최고매출품목 계산

즉, 이번 요청은 완전히 새롭게 생긴 기능만 있는 것이 아니라, 기존 엑셀에 있던 운영 업무를 웹 ERP 안에서 더 제대로 쓰고 싶다는 요청으로 볼 수 있습니다.

## 2. 기존 기능에서 보완해야 할 부분

다음 항목은 기존 ERP 흐름과 연결되어 있지만, 실제 운영에 맞추려면 보완이 필요합니다.

### 직원 관리와 근무표

기존 엑셀에는 직원 이름 목록과 월간 근무표가 있습니다.

ERP에서도 직원 이름을 관리하고, 하루 장부에서 근무자를 선택하면 월별 근무일수가 자동으로 집계되도록 보완할 수 있습니다.

### 상품별 분석

기존 엑셀에는 품목별로 전재고, 매입, 판매원가, 이익률, 판매금액, 판매량을 계산하는 영역이 있습니다.

ERP에서도 본사가 상품별 판매와 마진을 더 자세히 볼 수 있도록 보완할 수 있습니다.

### 품목명과 규격 정리

엑셀에는 `꽃게 [사게]`처럼 입력된 품목명을 분석용 이름으로 바꾸는 수식이 있습니다.

ERP에서도 품목명과 규격을 정리하고, 같은 상품이 다르게 입력되어도 하나로 묶어 볼 수 있게 해야 합니다.

### 리포트 고도화

현재 ERP에도 기본 리포트는 있지만, 기존 엑셀에는 더 다양한 비교표가 있습니다.

- 기간 비교
- 월별 비교
- 연도별 비교
- 설날 같은 특수기간 비교

이 부분은 본사 보고용 화면으로 더 확장할 수 있습니다.

## 3. 새로 추가되는 기능

다음 항목은 기존 1차 개발 범위를 넘어서는 기능입니다. 별도 추가 산정이 필요한 부분입니다.

### FIFO 재고 계산

수산물은 같은 품목이라도 매입 단가가 자주 바뀝니다.

그래서 재고 금액을 단순 평균으로 계산하기보다, 먼저 들어온 매입분부터 먼저 빠지는 방식으로 계산해야 합니다. 이것을 FIFO, 즉 선입선출 방식이라고 합니다.

이 기능은 적용하되, 추가 견적에는 포함하지 않는 방향으로 정리했습니다.

### 재고 매입 이력 확인

본사가 재고 금액이나 전일 재고를 눌렀을 때, 해당 재고가 언제 얼마에 들어온 물건인지 확인할 수 있는 기능입니다.

예를 들면 "5월 3일에 몇 개를 얼마 단가로 샀고, 지금 몇 개가 남아 있다"는 식으로 볼 수 있습니다.

### 본사 전체 재고 보기

본사가 모든 지점의 재고를 한 화면에서 볼 수 있는 기능입니다.

예를 들면 꽃게가 각 매장에 몇 개씩 남아 있고, 전체 지점 합계가 몇 개인지 바로 확인할 수 있습니다.

이 기능은 본사에서 매입 발주를 계획할 때 도움이 됩니다.

### 인사와 급여 정산

직원별 입사일, 근무일수, 지각/조퇴 같은 특이사항을 관리하고, 필요하면 급여 차액을 수동으로 입력하는 기능입니다.

여러 지점에서 교대 근무한 직원도 한 달 기준으로 합산할 수 있습니다.

### 월 손익계산서

현재 장부에는 매장 현장에서 쓰는 일일 비용이 들어갑니다.

추가로 본사가 월세, 세금 같은 고정비를 입력하면 매장별 월 손익을 더 정확히 볼 수 있습니다.

### 라인 또는 텔레그램 알림

매일 아침 정해진 시간에 본사나 담당자에게 주요 내용을 자동으로 보내는 기능입니다.

라인 알림도 API로 구현 가능합니다. 다만 예전 방식인 LINE Notify는 종료되었기 때문에, 라인을 사용하려면 LINE 공식 계정과 Messaging API를 사용해야 합니다.

예를 들면 다음 내용을 보낼 수 있습니다.

- 적자가 난 지점
- 목표 마진율보다 낮은 지점
- 오래 남아 있는 재고

라인을 쓰는 경우에는 알림 받을 사람이 공식 계정을 친구 추가하거나, 알림용 단체방에 공식 계정이 들어가 있어야 합니다. 요금과 월 발송 가능 수는 공식 계정의 국가와 플랜에 따라 확인이 필요합니다.

### AI 분석 확장

향후에는 ERP에 쌓인 데이터를 AI가 분석할 수 있도록 확장할 수 있습니다.

다만 AI 분석은 이번 개발에 바로 포함하지 않습니다. 지금은 데이터가 잘 쌓이도록 구조만 신경 쓰고, 실제 AI 기능은 나중에 별도 단계로 논의하는 것이 좋습니다.

## 4. 쉽게 정리하면

기존 1차 ERP는 "엑셀 대신 하루 장부를 안정적으로 입력하고 본사가 확인하는 시스템"이었습니다.

이번 회의에서 나온 요청은 운영 범위를 더 넓히는 내용입니다.

이제는 단순히 장부를 입력하는 것을 넘어서, 본사가 매입, 재고, 직원, 리포트, 손익까지 더 정확하게 관리하고 싶다는 방향입니다.

특히 전체 지점 재고 보기, 급여 정산, 월 손익계산서는 기존보다 업무 범위가 넓어지는 기능입니다. FIFO 재고 계산은 적용하지만 추가 견적에는 포함하지 않는 항목으로 정리했습니다.

따라서 기존 기능의 보완 항목과 추가 개발 항목을 나누어 견적을 산정하는 것이 좋습니다.
