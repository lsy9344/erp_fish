# Review Resolution: ERP Fish PRD

## 2026-05-28

리뷰 입력:

- `reconcile-markdown-sources.md`
- `reconcile-workbooks.md`
- `review-rubric.md`

## 처리 결과

### 1. 지점장 제출 후 잠금 vs 본사 마감 잠금

처리: PRD §3.1에 장부 상태 모델을 추가했다.

- 지점장 제출 또는 검토 대기 전환은 원본 잠금이 아니다.
- 원본 장부가 잠기는 시점은 본사 마감이다.
- 본사 마감 후에는 정정 기록만 추가할 수 있다.

근거: 사용자가 “본사가 마감하면 잠김”이라고 확정했다.

### 2. 재고 조정과 차이 사유 필수 기록

처리: `FR-10: 재고 조정 기록`을 추가했다.

- 조정 전/후 수량과 금액
- 차이 수량/금액
- 차이 사유 필수
- 작성자/작성 시각
- 본사 마감 후에는 정정 기록으로만 추가

### 3. 계산식과 본사 보고 KPI

처리: PRD §4.3에 1차 계산 기준 표를 추가하고, `FR-13`, `FR-28`, `FR-29`의 검증 조건을 보강했다.

추가/명시한 항목:

- 총매출
- 매출원가
- 매출이익
- 이익률
- 영업이익
- 인당생산성
- 재고금액
- 평균재고
- 평균매출
- 매출대비 재고비율
- 최고매출품목
- 매출차액

### 4. 과거 엑셀 데이터 이관

처리: 1차 제외 범위에 `과거 엑셀 데이터 일괄 이관`을 추가했다. 리포트는 1차에서 시스템 입력 신규 데이터를 기준으로 동작한다고 명시했다.

### 5. 직원/근무표 범위

처리: `FR-12`와 1차 제외 범위를 보강했다.

- 1차는 근무인원/특이사항 기록 수준이다.
- 직원 마스터, 근무자별 선택, 월간 근무일수 자동 집계, 지각/조퇴 상세 관리는 제외한다.

근거: 사용자가 “근무인원/특이사항을 기록하는 수준”이라고 확정했다.

### 6. 매입 입력 권한

처리: PRD의 기존 정책을 유지했다.

- 본사와 지점장 모두 매입 정보를 입력할 수 있다.
- 본사는 매입 기준/마스터를 관리한다.

근거: 사용자가 “매입 기준/마스터는 본사 관리, 실제 장부 매입 입력은 지점장과 본사 모두 가능”이라고 확정했다. 워크북의 `본사 스텝 외 작성 금지` 문구는 기존 엑셀 운영 힌트로 보되, 1차 제품 정책은 사용자 확정 답변을 따른다.

### 7. 용어와 FR 번호

처리:

- 제품 용어는 `지점`으로 통일했다.
- `FR-1`부터 `FR-29`까지 번호를 연속 순서로 정리했다.
- 성공 지표의 FR 참조를 새 번호에 맞게 갱신했다.
