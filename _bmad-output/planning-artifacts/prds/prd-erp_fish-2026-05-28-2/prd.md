# PRD: ERP Fish

## 0. 문서 목적

이 PRD는 ERP Fish의 1차 제품 범위를 정의한다. 대상 독자는 PM, 개발자, UX/화면 설계 담당자, 아키텍처 담당자, 에픽/스토리 작성자다.

제품은 원드라이브 엑셀로 운영하던 지점별 일일 장부를 웹 기반 내부 ERP로 전환한다. 이 문서는 엑셀 화면을 그대로 복제하지 않고, 실제 업무 목적과 데이터 흐름을 기준으로 요구사항을 정리한다.

입력 자료 추출 메모는 `addendum.md`에 보존한다. 본문은 확정된 제품 방향과 1차 요구사항만 다룬다.

## 1. 비전

ERP Fish의 1차 제품은 본사가 모든 지점의 일일 장부를 확인하고, 필요한 데이터를 직접 입력하거나 수정하고, 최종 마감할 수 있는 관제탑이다.

지점장은 로그인 후 자기 지점의 일일 장부를 입력한다. 본사는 전체 지점의 장부 상태, 매출/이익률 급락, 매출차액, 재고/손실 이상을 매일 아침 한 화면에서 확인한다. 이 데이터는 매일 아침 지점장 회의의 기준 자료로 사용된다.

1차 성공 기준은 “본사가 매일 아침 회의 전에 전체 지점 장부를 확인하고, 문제 가능성이 높은 지점을 먼저 파악하며, 필요한 장부를 마감할 수 있는 상태”다.

## 2. 대상 사용자

### 2.1 Jobs To Be Done

- 본사 사용자는 매일 아침 회의 전에 모든 지점의 장부 상태와 이상 신호를 빠르게 확인하고 싶다.
- 본사 사용자는 지점 데이터가 누락되거나 잘못된 경우 직접 입력하거나 수정한 뒤 마감하고 싶다.
- 본사 사용자는 마감 후 오류를 발견해도 원본 장부를 보존하면서 정정 기록으로 회의/리포트 숫자를 바로잡고 싶다.
- 지점장은 자기 지점의 매출, 결제, 비용, 매입, 재고, 손실, 근무 정보를 하루 단위로 입력하고 싶다.
- 지점장은 엑셀 수식이나 서식을 깨뜨릴 걱정 없이 장부를 제출하고 싶다.

### 2.2 1차 사용자

- **본사 사용자**: 전체 지점 조회, 직접 입력, 수정, 마감, 정정 기록, 마스터 관리, 이상 신호 기준값 설정 권한을 가진다.
- **지점장**: 자기 지점의 일일 장부를 입력하고, 본사 마감 전까지 수정할 수 있다.

### 2.3 1차 비사용자

- 일반 직원은 1차에서 로그인 사용자로 보지 않는다. 직원 정보는 근무인원/특이사항 기록에 필요한 데이터로만 다룬다.
- 외부 회계 시스템, POS, 카드사 시스템은 1차에서 직접 사용자나 필수 연동 대상으로 보지 않는다.

### 2.4 핵심 사용자 여정

- **UJ-1. 본사 김관리자가 아침 회의 전 문제 지점을 고른다.**
  - **상황:** 김관리자는 매일 아침 지점장 회의 전에 전 지점 장부를 확인해야 한다.
  - **진입:** 본사 계정으로 로그인 후 관제판을 연다.
  - **흐름:** 오늘/어제 기준 전체 지점 현황을 보고, 매출/이익률 급락, 매출차액, 재고/손실 이상이 있는 지점을 먼저 연다.
  - **가치 순간:** 문제 가능성이 높은 지점과 사유가 한눈에 보인다.
  - **결과:** 김관리자는 회의에서 어떤 지점을 먼저 이야기할지 결정한다.

- **UJ-2. 현대 지점장이 하루 장부를 입력한다.**
  - **상황:** 현대 지점장은 퇴근 전 당일 장부를 입력해야 한다.
  - **진입:** 지점장 계정으로 로그인 후 자기 지점의 오늘 장부를 연다.
  - **흐름:** 매출/결제, 비용, 매입, 재고, 손실/폐기/떨이, 근무인원/특이사항을 단계형 화면에서 입력한다.
  - **가치 순간:** 마지막 검토 화면에서 누락 항목과 계산 요약을 확인한다.
  - **결과:** 장부가 본사 검토 대상으로 올라간다.

- **UJ-3. 본사 이스태프가 누락 매입을 보완하고 마감한다.**
  - **상황:** 특정 지점 장부에 매입 정보가 빠졌다.
  - **진입:** 본사 계정으로 해당 지점 장부 상세를 연다.
  - **흐름:** 변경 이력을 확인하고, 매입 정보를 직접 입력한 뒤 계산값과 이상 신호를 다시 확인한다.
  - **가치 순간:** 본사 입력/수정 내역이 지점장 입력과 구분되어 남는다.
  - **결과:** 본사가 장부를 마감하면 원본 장부가 잠긴다.

- **UJ-4. 본사 김관리자가 마감 후 오류를 정정 기록으로 반영한다.**
  - **상황:** 마감 후 회의 준비 중 매출차액 오류가 발견됐다.
  - **진입:** 본사 계정으로 마감된 장부 상세를 연다.
  - **흐름:** 원본 장부는 수정하지 않고 정정 기록을 추가한다.
  - **가치 순간:** 리포트와 관제판 숫자는 정정 반영값으로 바뀌고, 원본값과 정정 내역은 추적 가능하다.
  - **결과:** 회의 자료는 최신 운영 숫자를 보여주면서도 원본 감사 추적을 잃지 않는다.

## 3. 용어

- **본사 사용자** — 전체 지점 데이터를 조회, 입력, 수정, 마감, 정정할 수 있는 사용자.
- **지점장** — 자기 지점의 일일 장부를 입력하고 본사 마감 전까지 수정할 수 있는 사용자.
- **지점** — 장부가 작성되는 매장 단위.
- **일일 장부** — 특정 지점과 특정 일자에 대한 매출, 결제, 비용, 매입, 재고, 손실, 근무 기록.
- **장부 상태** — 일일 장부의 진행 상태. 1차 상태는 미입력, 입력 중, 검토 대기, 본사 마감이다.
- **영업 상태** — 특정 지점+일자의 영업 여부. 1차 상태는 영업일, 휴무일이다.
- **본사 마감** — 본사 사용자가 일일 장부를 확정해 원본 수정을 잠그는 행위.
- **정정 기록** — 본사 마감 후 원본 장부를 바꾸지 않고 추가하는 수정 기록.
- **정정 반영값** — 원본 장부 값에 정정 기록을 반영한 운영 기준 숫자.
- **관제판** — 본사가 전체 지점의 장부 상태와 이상 신호를 한 화면에서 보는 화면.
- **이상 신호** — 본사가 설정한 기준값에 따라 표시되는 주의 표시. 1차 이상 신호는 매출/이익률 급락, 매출차액, 재고/손실 이상이다.
- **품목 마스터** — 본사가 관리하는 품목명, 구분, 규격, 기본 단가 정보.
- **매입 기준** — 본사가 관리하는 매입 관련 기준 정보.
- **재고 흐름** — 전일재고, 매입, 판매/차감, 손실, 당일재고로 이어지는 품목 단위 흐름.
- **재고 조정** — 시스템 계산 재고와 실제 재고가 다를 때 조정 전/후 값과 차이 사유를 기록하는 행위.
- **월초 재고 스냅샷** — 월 첫 장부의 전일재고를 만들기 위한 기준 재고. 기존 엑셀의 `이월` 시트 역할을 대체한다.
- **전일 이월금액** — 총매출에 포함하지 않고 별도 조정/이월 항목으로 저장하는 금액.
- **매출차액** — 총매출과 상품별 판매금액 합계에서 손실/조정 반영액을 차감한 값 사이의 차이. 전일 이월금액은 총매출에 포함하지 않는다.
- **아침 회의 리포트** — 본사가 매일 아침 지점장 회의에서 사용하는 일별 요약 리포트.

### 3.1 장부 상태 모델

| 상태 | 의미 | 수정 가능 여부 |
| --- | --- | --- |
| 미입력 | 해당 지점+일자 장부가 아직 작성되지 않은 상태 | 본사와 해당 지점장 입력 가능 |
| 입력 중 | 일부 값이 저장되었지만 검토 대기 전 상태 | 본사와 해당 지점장 수정 가능 |
| 검토 대기 | 지점장 또는 본사가 입력을 마치고 본사 검토 대상으로 올린 상태 | 본사 마감 전까지 본사와 해당 지점장 수정 가능 |
| 본사 마감 | 본사가 최종 마감한 상태 | 원본 수정 불가, 본사 정정 기록만 추가 가능 |

지점장 제출 또는 검토 대기 전환은 원본 잠금이 아니다. 1차 정책에서 원본 장부가 잠기는 시점은 본사 마감이다.

## 4. 기능 요구사항

### 4.1 인증, 권한, 변경 이력

**설명:** 본사 사용자와 지점장은 로그인 역할에 따라 접근 범위와 입력 기능이 달라진다. 공동 입력 구조이므로 입력자, 수정자, 수정 시각, 변경 이력은 제품 신뢰의 핵심이다.

#### FR-1: 역할 기반 로그인

사용자는 본사 사용자 또는 지점장 역할로 로그인할 수 있다.

**검증 조건:**
- 본사 사용자는 전체 지점 메뉴에 접근할 수 있다.
- 지점장은 자기 지점 장부 메뉴에 접근할 수 있다.
- 로그인하지 않은 사용자는 장부 데이터에 접근할 수 없다.

#### FR-2: 지점 접근 제한

지점장은 자기 지점의 일일 장부만 조회하고 입력할 수 있다.

**검증 조건:**
- 지점장은 다른 지점 장부 목록과 상세에 접근할 수 없다.
- 본사 사용자는 모든 지점 장부 목록과 상세에 접근할 수 있다.

#### FR-3: 입력/수정 이력 기록

시스템은 주요 입력과 수정에 대해 입력자, 수정자, 수정 시각, 변경 전 값, 변경 후 값을 기록한다.

**검증 조건:**
- 지점장 입력과 본사 입력/수정 내역이 구분되어 표시된다.
- 장부 상세에서 변경 이력을 확인할 수 있다.
- 본사 마감 후 정정 기록은 일반 수정 이력과 구분된다.

### 4.2 일일 장부 입력

**설명:** 일일 장부는 지점+일자 단위로 작성된다. 입력 화면은 엑셀식 단일 장표 복제가 아니라 단계형 입력 흐름을 기본으로 한다.

#### FR-4: 지점+일자 장부 생성

본사 사용자와 지점장은 지점+일자 기준으로 일일 장부를 열 수 있다.

**검증 조건:**
- 같은 지점+일자에는 하나의 일일 장부가 존재한다.
- 장부가 없으면 입력 시작 시 생성된다.
- 이미 생성된 장부를 다시 열면 기존 입력값이 유지된다.

#### FR-5: 단계형 입력 흐름

일일 장부 입력은 매출/결제, 비용, 매입, 재고, 손실/폐기/떨이, 근무인원/특이사항, 검토/제출 순서로 제공된다.

**검증 조건:**
- 입력자는 각 단계를 이동할 수 있다.
- 검토/제출 화면에서 주요 누락 항목과 계산 요약을 확인할 수 있다.
- 저장 중인 장부는 입력 중 상태로 표시된다.
- 검토 대기 상태는 본사 마감 전 상태이며, 원본 잠금으로 처리하지 않는다.

#### FR-6: 매출/결제 입력

입력자는 총매출, 현금, 카드, 기타 결제수단, 매출차액 관련 값을 입력할 수 있다.

**검증 조건:**
- 결제수단별 금액과 총매출이 장부에 저장된다.
- 시스템은 매출차액을 계산하거나 표시할 수 있다.
- 매출차액이 본사 기준값을 넘으면 이상 신호 후보로 표시된다.

#### FR-7: 비용 입력

입력자는 비용 항목, 금액, 메모를 입력할 수 있다.

**검증 조건:**
- 하나의 일일 장부에 여러 비용 항목을 등록할 수 있다.
- 비용 합계는 영업이익 계산에 반영된다.

#### FR-8: 매입 입력

입력자는 품목, 규격, 단가, 수량, 매입금액을 입력할 수 있다.

**검증 조건:**
- 지점장과 본사 사용자 모두 매입 정보를 입력할 수 있다.
- 본사 사용자가 관리하는 매입 기준 또는 품목 마스터를 선택해 입력할 수 있다.
- 입력된 매입은 재고 흐름과 일일 요약 계산에 반영된다.

#### FR-9: 품목 단위 재고 입력

입력자는 품목, 규격, 단가, 전일재고, 매입, 판매/차감, 당일재고, 재고금액, 수량을 다룰 수 있다.

**검증 조건:**
- 재고는 단순 금액 요약이 아니라 품목+규격 단위로 저장된다.
- 냉동/생물 같은 품목 구분을 기록할 수 있다.
- 전일재고, 매입, 판매/차감, 손실, 당일재고 흐름을 일자별로 추적할 수 있다.
- 시스템 계산 재고와 실제 입력 재고의 차이를 확인할 수 있다.
- 월 첫 장부의 전일재고는 월초 재고 스냅샷에서 가져온다.
- 이후 영업일의 전일재고는 직전 마감 장부의 당일재고에서 자동 이월된다.

#### FR-10: 재고 조정 기록

본사 사용자와 지점장은 시스템 계산 재고와 실제 재고가 다를 때 재고 조정을 기록할 수 있다.

**검증 조건:**
- 재고 조정에는 품목, 규격, 조정 전 수량/금액, 조정 후 수량/금액, 차이 수량/금액, 차이 사유, 작성자, 작성 시각이 남는다.
- 차이 사유는 필수로 입력하거나 선택해야 한다.
- 재고 조정은 재고 흐름, 이상 신호, 리포트에 반영된다.
- 본사 마감 후 재고 조정은 원본 수정이 아니라 정정 기록으로만 추가된다.

#### FR-11: 손실/폐기/떨이 입력

입력자는 손실, 폐기, 떨이 항목을 품목, 수량, 금액, 처리 유형, 사유/특이사항과 함께 입력할 수 있다.

**검증 조건:**
- 손실 항목은 재고 흐름과 손실 리포트에 반영된다.
- 손실액 또는 손실 수량이 본사 기준값을 넘으면 이상 신호 후보로 표시된다.

#### FR-12: 근무인원/특이사항 입력

입력자는 근무인원과 근무 관련 특이사항을 기록할 수 있다.

**검증 조건:**
- 근무인원은 인당생산성 계산에 반영된다.
- 1차 범위는 직원 마스터, 근무자별 선택, 월간 근무일수 자동 집계, 지각/조퇴, 출퇴근/급여 수준의 상세 근태 관리를 포함하지 않는다.

### 4.3 계산과 검증

**설명:** 시스템은 장부 입력값을 기준으로 본사 회의에 필요한 핵심 지표를 계산하고, 이상 신호 후보를 만든다.

1차 계산 기준은 다음을 기본으로 한다. `30%단가`의 업무 의미처럼 아직 확인이 필요한 항목은 §9 Open Questions에서 별도 관리한다.

| 지표 | 1차 기준 |
| --- | --- |
| 총매출 | 현금 + 카드 + 기타매출 + 당일 조정 매출. 전일 이월금액은 제외 |
| 매출원가 | 전일재고 + 매입 - 당일재고 |
| 매출이익 | 총매출 - 매출원가 |
| 이익률 | 매출이익 / 총매출 |
| 영업이익 | 매출이익 - 비용합계 |
| 인당생산성 | 총매출 / 근무인원 |
| 재고금액 | 품목별 수량 * 단가 |
| 평균재고 | 선택 기간의 일별 재고 스냅샷 평균 |
| 평균매출 | 선택 기간의 일별 총매출 평균 |
| 매출대비 재고비율 | 평균재고 / 평균매출 |
| 최고매출품목 | 상품별 판매금액이 가장 큰 품목 |
| 매출차액 | 총매출 - 상품별 판매금액 합계 - 손실/당일 조정 반영액 |

#### FR-13: 핵심 지표 계산

시스템은 매출, 매출원가, 매출이익, 이익률, 영업이익, 인당생산성, 평균재고, 평균매출, 매출대비 재고비율, 최고매출품목, 최고매출품목 매출액, 매출차액, 재고/손실 관련 지표를 계산한다.

**검증 조건:**
- 각 일일 장부 상세에서 계산값을 확인할 수 있다.
- 관제판과 리포트는 같은 계산 기준을 사용한다.
- 계산 기준이 확정되지 않은 항목은 §9 Open Questions의 결정 등급에 따라 처리한다.

#### FR-14: 입력 검증

시스템은 필수 입력값 누락, 결제수단 합계와 총매출 차이, 상품별 판매금액 합계와 총매출 차이, 재고/손실 이상 후보를 표시한다.

**검증 조건:**
- 검토/제출 화면에서 주요 누락과 이상 후보를 확인할 수 있다.
- 이상 후보가 있어도 본사 사용자는 장부를 열어 판단할 수 있다.
- 검증 결과는 본사 마감 여부와 구분되어 표시된다.
- 영업 상태가 휴무일인 장부는 미입력 장부와 구분되어 표시된다.

### 4.4 본사 관제판과 이상 신호

**설명:** 본사 첫 화면은 오늘/어제 기준 전체 지점 관제판이다. 본사는 전체 순위보다 문제 가능성이 높은 지점부터 확인해야 한다.

#### FR-15: 전체 지점 관제판

본사 사용자는 오늘/어제 기준 전체 지점의 장부 상태를 한 화면에서 볼 수 있다.

**검증 조건:**
- 각 지점은 한 줄로 표시된다.
- 각 행에는 영업 상태, 장부 상태, 이상 신호, 매출, 이익률, 매출차액, 손실 여부, 마지막 수정자, 마지막 수정 시각, 본사 마감 여부가 표시된다.
- 본사 사용자는 지점 행을 선택해 일일 장부 상세로 이동할 수 있다.

#### FR-16: 이상 신호 표시

시스템은 매출/이익률 급락, 매출차액, 재고/손실 이상을 이상 신호로 표시한다.

**검증 조건:**
- 이상 신호는 지점 행에서 눈에 띄게 표시된다.
- 한 지점에 여러 이상 신호가 동시에 표시될 수 있다.
- 이상 신호가 없는 지점도 관제판에서 상태를 확인할 수 있다.

#### FR-17: 이상 신호 기준값 설정

본사 사용자는 이상 신호 기준값을 설정할 수 있다.

**검증 조건:**
- 본사는 매출 하락률, 이익률 하락폭, 매출차액 금액, 손실액, 재고 차이 기준을 설정할 수 있다.
- 기준값 변경 후 새 관제판 조회에는 변경된 기준이 반영된다.
- 기준값 변경 이력이 남는다.
- 기준값은 고정 코드가 아니라 본사 운영 설정으로 저장된다.

### 4.5 본사 입력, 마감, 정정

**설명:** 본사는 관제탑 역할을 하며, 지점 입력 데이터만 보는 사용자가 아니다. 본사는 직접 입력/수정하고 최종 마감한다.

#### FR-18: 본사 직접 입력/수정

본사 사용자는 모든 지점의 본사 마감 전 일일 장부를 직접 입력하거나 수정할 수 있다.

**검증 조건:**
- 본사 사용자는 지점장 입력값을 보완하거나 수정할 수 있다.
- 본사 수정은 변경 이력에 본사 사용자 수정으로 기록된다.
- 수정 후 계산값과 이상 신호가 갱신된다.

#### FR-19: 본사 마감과 원본 잠금

본사 사용자는 일일 장부를 본사 마감 처리할 수 있다.

**검증 조건:**
- 본사 마감 후 원본 장부 입력값은 수정할 수 없다.
- 본사 마감 상태는 관제판과 장부 상세에서 표시된다.
- 지점장은 본사 마감된 장부를 수정할 수 없다.

#### FR-20: 마감 후 정정 기록

본사 마감 후 오류가 발견되면 본사 사용자는 원본을 수정하지 않고 정정 기록을 추가할 수 있다.

**검증 조건:**
- 정정 기록에는 작성자, 작성 시각, 대상 항목, 원본값, 정정값, 사유가 남는다.
- 정정 기록은 삭제되거나 원본 수정으로 합쳐지지 않는다.
- 장부 상세에서 원본값과 정정 반영값을 구분해 볼 수 있다.

#### FR-21: 정정 반영값 사용

관제판과 리포트의 기본 숫자는 정정 반영값을 사용한다.

**검증 조건:**
- 정정 기록 추가 후 관제판과 리포트 숫자가 정정 반영값으로 표시된다.
- 사용자는 상세에서 원본값과 정정 반영값의 차이를 확인할 수 있다.

### 4.6 마스터와 관리 설정

**설명:** 본사는 지점, 사용자, 품목, 매입 기준, 코드, 이상 신호 기준을 관리한다. 지점장은 본사 마스터를 기준으로 장부를 입력한다.

#### FR-22: 지점 마스터 관리

본사 사용자는 지점명과 활성/비활성 상태를 관리할 수 있다.

**검증 조건:**
- 비활성 지점은 신규 장부 입력 대상에서 제외할 수 있다.
- 과거 장부 조회를 위해 비활성 지점 데이터는 보존된다.

#### FR-23: 사용자/권한 관리

본사 사용자는 본사 사용자와 지점장 계정, 지점 접근 권한을 관리할 수 있다.

**검증 조건:**
- 지점장 계정은 하나 이상의 지점에 연결된다.
- 권한 변경 후 접근 범위가 즉시 반영된다.

#### FR-24: 품목 마스터 관리

본사 사용자는 품목명, 구분, 규격, 기본 단가를 관리할 수 있다.

**검증 조건:**
- 장부 입력자는 품목 마스터에서 품목을 선택할 수 있다.
- 장부 입력 시 실제 단가와 수량은 장부 상황에 맞게 입력/조정할 수 있다.
- 품목 마스터 변경은 과거 장부의 원본 입력값을 임의로 바꾸지 않는다.

#### FR-25: 매입 기준 관리

본사 사용자는 매입 관련 기준 또는 기본 정보를 관리할 수 있다.

**검증 조건:**
- 매입 입력 시 본사 기준 정보를 선택하거나 참조할 수 있다.
- 본사와 지점장 모두 장부에 매입 정보를 입력할 수 있다.

#### FR-26: 코드 관리

본사 사용자는 결제수단, 비용 항목, 손실 유형 같은 장부 입력 코드를 관리할 수 있다.

**검증 조건:**
- 장부 입력 화면에서 관리 코드가 선택지로 제공된다.
- 비활성 코드는 신규 입력 선택지에서 제외할 수 있다.
- 과거 장부의 기존 코드 표시는 유지된다.

### 4.7 리포트

**설명:** 1차 리포트는 운영 회의와 월간 관리에 필요한 범위로 제한한다.

#### FR-27: 일별 아침 회의 리포트

본사 사용자는 전체 지점의 일별 회의용 요약을 볼 수 있다.

**검증 조건:**
- 리포트는 지점별 마감 상태, 이상 신호, 매출, 이익률, 매출차액, 손실 현황을 보여준다.
- 기본 숫자는 정정 반영값을 사용한다.
- 원본값과 정정 반영값 차이는 상세에서 확인할 수 있다.

#### FR-28: 지점별 기간 비교

본사 사용자는 선택 기간의 지점별 실적을 비교할 수 있다.

**검증 조건:**
- 비교 항목은 매출, 매출이익, 이익률, 영업이익, 인당생산성, 평균재고, 평균매출, 매출대비 재고비율, 손실을 포함한다.
- 지점별 비교가 가능해야 한다.
- 1차 리포트는 시스템에 입력된 신규 데이터를 기준으로 하며, 과거 엑셀 일괄 이관 데이터는 1차 필수 전제로 삼지 않는다.
- 미입력, 입력 중, 검토 대기, 본사 마감, 휴무일 데이터는 구분되어 집계된다.

#### FR-29: 월간 지점 요약

본사 사용자는 지점별 월간 실적, 마감 현황, 주요 이상 항목, 손실/재고 흐름 요약을 볼 수 있다.

**검증 조건:**
- 월과 지점을 선택해 요약을 볼 수 있다.
- 마감되지 않은 장부가 있으면 요약에서 구분된다.
- 월간 요약은 최고매출품목, 최고매출품목 매출액, 평균재고, 평균매출, 매출대비 재고비율을 포함한다.

## 5. 전역 비기능 요구사항

- **권한 보안:** 본사 사용자와 지점장의 접근 범위는 서버 기준으로 강제되어야 한다.
- **감사 추적:** 마감, 정정, 주요 입력/수정, 기준값 변경은 누가 언제 무엇을 바꿨는지 추적 가능해야 한다.
- **데이터 보존:** 본사 마감 후 원본 장부는 보존되어야 하며, 정정 기록으로 덮어쓰면 안 된다.
- **반응형 웹:** 지점장 입력 화면은 PC, 태블릿, 모바일 웹에서 사용할 수 있어야 한다.
- **운영 속도:** 본사 관제판 기본 조회는 10개 내외 지점 기준 3초 안에 지점 목록과 이상 신호를 표시하는 것을 목표로 한다.
- **모바일 사용성:** 단계형 장부 입력의 핵심 흐름은 최소 390px 폭의 모바일 화면에서 사용할 수 있어야 한다.
- **엑셀 의존 감소:** 1차 제품은 기존 엑셀의 목적을 대체하되, 엑셀 수식이나 서식 구조를 그대로 복제하는 것을 목표로 하지 않는다.

## 6. 1차 제외 범위

다음 항목은 1차 범위에서 제외한다.

- 텔레그램 알림
- 이카운트 연동
- POS/카드 매출 자동 연동
- AI 이미지 식별
- 설날 등 특수 기간 비교 리포트
- 출퇴근/급여 수준의 상세 근태 관리
- 직원 마스터, 근무자별 선택, 월간 근무일수 자동 집계, 지각/조퇴 상세 관리
- 고급 재고 신선도/입고일 추적
- 과거 엑셀 데이터 일괄 이관

이 항목들은 후순위 또는 추후 검토 항목이며, 아직 확정 기능으로 보지 않는다. 1차는 수동 입력, 검증, 본사 관제, 마감, 정정 기록, 기본 리포트를 안정화하는 데 집중한다.

## 7. MVP 범위

### 7.1 포함 범위

- 본사/지점장 로그인과 역할 기반 권한
- 지점+일자 일일 장부 생성
- 단계형 장부 입력
- 매출/결제, 비용, 매입, 재고, 손실/폐기/떨이, 근무인원/특이사항 입력
- 품목/규격/단가/재고/수량 기반 재고 관리
- 본사 관제판
- 매출/이익률 급락, 매출차액, 재고/손실 이상 신호
- 재고 조정 기록과 차이 사유 필수 입력
- 본사 기준값 설정
- 본사 직접 입력/수정
- 본사 마감과 원본 잠금
- 마감 후 정정 기록
- 정정 반영값 기반 관제판/리포트
- 지점/사용자/품목/매입 기준/코드 관리
- 일별 아침 회의 리포트
- 지점별 기간 비교
- 월간 지점 요약

### 7.2 제외 범위

제외 범위는 §6과 동일하다. UX, 아키텍처, 에픽/스토리 작성 시 §6을 1차 범위 방어선으로 사용한다.

## 8. 성공 지표

**Primary**

- **SM-1: 아침 회의 준비 가능성** — 본사가 회의 전 전체 지점의 마감 상태와 이상 신호를 확인할 수 있다. 검증: FR-15, FR-16, FR-27.
- **SM-2: 본사 마감 통제** — 본사가 각 지점 장부를 검토하고, 필요한 수정 후 마감할 수 있다. 검증: FR-18, FR-19.
- **SM-3: 마감 후 추적 가능성** — 마감 후 오류는 원본 수정이 아니라 정정 기록으로만 반영된다. 검증: FR-20, FR-21.

**Secondary**

- **SM-4: 지점 입력 완결성** — 지점장 또는 본사가 1차 장부 입력 범위 전체를 한 장부 안에서 입력할 수 있다. 검증: FR-5~FR-12.
- **SM-5: 본사 기준 운영** — 본사가 품목, 권한, 코드, 이상 신호 기준을 관리할 수 있다. 검증: FR-17, FR-22~FR-26.

**Counter-metrics**

- **SM-C1: 빠른 입력만을 위해 필수 검증을 줄이지 않는다.** 장부 입력 속도는 중요하지만, 매출차액과 재고/손실 이상 확인을 건너뛰는 방향으로 최적화하지 않는다.
- **SM-C2: 이상 신호 수를 줄이는 것을 성공으로 보지 않는다.** 이상 신호는 운영 위험을 찾기 위한 장치이며, 기준값을 느슨하게 해서 조용한 화면을 만드는 것은 성공이 아니다.
