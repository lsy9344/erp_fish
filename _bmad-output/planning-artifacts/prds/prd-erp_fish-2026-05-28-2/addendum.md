# Addendum: 입력 자료 추출 메모

이 문서는 PRD 본문에 모두 넣기에는 세부적인 입력 자료 관찰 내용을 보존한다. 엑셀 파일은 수식 검증이 아니라 업무 형태와 목적 파악 기준으로만 해석한다.

## 1. 고객 정리 문서

출처:

- `docs/reference_from_customer/desc.md`
- `docs/reference_from_customer/feature_analysis.md`

핵심 관찰:

- 현재 원드라이브 엑셀은 일일 장부 마감, 재고 이월, 월간 근무/영업 집계, 본사 분석 리포트까지 담당한다.
- 목표는 엑셀 포맷을 그대로 복제하는 것이 아니라, 지점 일일 마감과 본사 검토 흐름을 웹 기반 ERP로 안정화하는 것이다.
- 주요 사용자 후보는 지점장, 본사 관리자/대표, 본사 스텝이다. 직원은 근무자 마스터 대상이며 로그인 사용자는 선택 사항이다.
- 필수 기능 후보는 지점장 입력 포털, 일일 장부 생성/마감/잠금, 재고 이월, 손실 기록, 본사 대시보드, 리포트, 알림, 텔레그램 발송이다.
- 이카운트 연동, POS/카드 매출 연동, 입고일/신선도 추적, AI 진열 이미지 식별은 확장 또는 추후 검토 후보로 분리된다.

## 2. `total_sales.xlsx`

핵심 관찰:

- 전체 매장 일별 실적과 본사 분석/보고용 파일로 보인다.
- 주요 시트는 `입력`, `분석`, `매장 별(년도)`, `매장 별(달)`, `보고양식1`, `설날매출비교`, `보고 양식`, `직원이름`이다.
- 숨김 보조 시트는 `매출`, `매출이익`, `이익률`, `인당생산성`, `평균 재고`, `Sheet3`이다.
- `입력` 시트는 일자, 요일, 매장, 매출, 현금, 카드, 매출이익, 이익률, 일지출, 영업이익, 인당생산성, 최고매출품목, 근무인원, 팀장, 팀원, 매출차액 등을 담는다.
- `분석` 시트는 대조 기간과 현재 기간을 비교하고, 매장별 KPI와 차트를 제공한다.
- `직원이름` 시트는 직원 마스터 성격이다.

제품 요구사항 후보:

- 일별 매장 실적 입력
- 직원/매장 마스터 관리
- 전기 대비 현재, 연도별, 월별 KPI 비교
- 매출, 매출이익, 이익률, 영업이익, 평균 근무인원, 인당생산성, 평균재고, 매출대비 재고비율 자동 집계
- 경영 보고용 표와 차트
- 설날 등 특수 기간 D-day 비교

## 3. `장부-202605현대 (1).xlsx`

핵심 관찰:

- 현대 지점의 월간 일일 장부 파일로 보인다.
- 시트는 `근무표`, `이월`, `1`~`31`로 구성된다.
- 일자 시트는 좌측 현장 입력/보고 영역과 우측 관리자 분석 영역으로 나뉘는 형태다.
- 매일 수산물 재고, 매입, 판매, 현금/카드 매출, 비용, 근무자, 폐기/떨이, 전달사항을 기록한다.
- 품목은 냉동/생물로 나뉘고, 전일재고 → 매입 → 판매/폐기 → 당일재고 흐름을 추적한다.
- `근무표`는 월간 근무/영업 요약, 관리자모드 영업자료조회, 일일 매입금액 집계표 역할을 한다.
- `이월`은 전월 이월 또는 월초 기준 재고 마스터 성격으로 보인다.

제품 요구사항 후보:

- 지점+월 단위 장부 생성과 일자별 장부 자동 생성
- 품목+규격 단위 냉동/생물 재고 관리
- 전일재고, 매입, 판매, 폐기/떨이, 당일재고 흐름 기록
- 결제수단별 매출과 비용 입력
- 근무자 입력 및 월간 근무일수 집계
- 전달사항/특이사항/본사 코멘트 기록
- 품목명, 구분, 규격, 단가 자동완성 또는 마스터 관리

## 4. 정책 확인이 필요한 항목

- `30%단가`의 업무 의미
- `전일 이월금액`이 총매출에 포함되는지 여부
- `매출차액`의 정의와 허용 기준
- 매입 입력 주체가 본사인지, 지점인지, 혼합인지
- 폐기/떨이 손실액 저장 방식
- 과거 결제수단 공란 이관 방식
- 휴무일과 미입력일 구분 방식
- `이월` 시트의 정확한 역할
- 보고 절취선 좌우 영역의 사용자 역할 구분
- 품목명, 구분, 규격 분리 기준

## 5. `docs/meeting/change.md`

핵심 관찰:

- 시스템 초기 세팅에는 본사 마스터 계정과 지점장 마스터 계정이 필요하다.
- 본사는 10개 이상 지점을 동적으로 등록하고 활성/비활성 상태를 제어해야 하며, 운영 매뉴얼 제공이 요구된다.
- 본사 관제판은 4단계 장부 상태 인디케이터와 리사이징 가능한 그리드/레이아웃을 제공해야 한다.
- 본사는 지점 입력 데이터를 강제 수정할 수 있고, 수동 검토를 생략한 즉시 일괄 마감 옵션을 가져야 한다.
- 변경 이력은 작성자/수정자, 일시, 기존 값, 변경 값을 운영자가 이해할 수 있는 텍스트 형태로 누적해야 한다.
- 이카운트 엑셀 업로드를 통해 본사가 매입 장부를 자동 생성하며, 업로드 검수 중 출고 단가를 수동 변경할 수 있어야 한다.
- 본사 업로드 매입은 지점장에게 기본적으로 보기 전용으로 제공하되, 비상 입력 허용 여부는 별도 정책 확인이 필요하다.
- FIFO 재고 금액 계산은 매입 잔량 근거를 추적하고, 전일재고/재고금액 클릭 시 기본 1개월 범위의 근거 이력을 보여줘야 한다.
- 지점 입력은 7단계 모바일 호환 위자드로 구성되고, 작성자 이름은 1단계 입력 후 전체 저장 흐름에 유지되어야 한다.
- 지점장 화면에서는 매출원가, 매출이익, 인당생산성, 영업이익 같은 민감 회계 지표가 원천 차단되어야 한다.
- 손실액은 지점장이 영업 개시 전에 설정한 희망 판매가와 실제 처리 금액의 차이를 기준으로 추적해야 한다.
- 직원 관리는 일당제/일급 운영을 전제로 입사일, 지각, 조퇴, 특수 근태 메모, 급여 차액 수동 입력을 포함한다.
- 냉동품/생물 그룹별 총판매액과 이익률은 표뿐 아니라 그래프/차트로도 볼 수 있어야 한다.
- 외부 알림은 오전 8시 등 설정된 시간에 당일 적자, 목표 마진율 미달, 장기 체화 재고를 LINE 또는 텔레그램으로 요약 발송하는 구조를 고려한다.
- AI 기능 자체는 이번 범위가 아니지만, 장기적으로 자연어 분석을 할 수 있도록 특이사항과 근무 배치 같은 데이터를 구조화해 보존해야 한다.
- 서버 인프라 세팅, 장애 대응, 시인성 개선 요청, 월 유지보수 비용은 제품 기능과 별도 운영/계약 정책으로 정리해야 한다.
