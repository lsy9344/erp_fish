# Story 1.3: 본사가 지점 정보를 관리한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 본사 사용자,<br>
I want 지점명과 활성 상태를 관리하고 싶다,<br>
so that 운영 중인 지점만 신규 업무 대상에 포함하면서 과거 지점 데이터는 보존할 수 있다.

## Acceptance Criteria

1. Given 본사 사용자가 기준정보의 지점 관리 화면에 접근할 때, when 화면이 열리면, then 지점 목록에는 지점명, 활성/비활성 상태, 마지막 수정자, 마지막 수정 시각이 표시되어야 한다, and 목록은 지점명이 많아져도 찾을 수 있도록 검색 또는 필터를 제공해야 한다.
2. Given 본사 사용자가 새 지점을 등록할 때, when 필수 값인 지점명을 입력하고 저장하면, then 새 지점이 활성 상태로 생성되어야 한다, and 이후 장부 입력 대상 옵션에 사용할 수 있는 active 지점 목록에 포함되어야 한다.
3. Given 본사 사용자가 기존 지점명을 수정할 때, when 변경 내용을 저장하면, then 새 지점명이 지점 목록에 표시되어야 한다, and 과거 장부와 연결된 지점 참조는 끊기지 않아야 한다.
4. Given 본사 사용자가 지점을 비활성 처리할 때, when 비활성 상태로 저장하면, then 해당 지점은 신규 장부 입력 대상에서 제외될 수 있어야 한다, and 과거 장부 조회를 위해 지점 데이터는 삭제되지 않아야 한다.
5. Given 본사 사용자가 지점 정보를 변경할 때, when 생성, 이름 변경, 활성 상태 변경이 완료되면, then actor, timestamp, target, before, after를 포함하는 감사 기록이 남아야 한다.
6. Given 지점 관리 폼에 잘못된 값이 입력될 때, when 저장을 시도하면, then 필드 하단에 한국어 검증 메시지가 표시되어야 한다, and 포커스는 첫 번째 오류 필드로 이동해야 한다.

### Latest Technical Information

- 2026-05-29 확인: Next.js App Router mutation 문서는 Server Functions/Server Actions를 form action 또는 client event에서 호출할 수 있고, mutation 후 `revalidatePath` 또는 `revalidateTag`로 stale data를 갱신하라고 안내한다. Story 1.3 Server Actions는 store 변경 후 `/app/master-data/stores`를 revalidate한다. [External: `https://nextjs.org/docs/app/getting-started/updating-data`]
- 2026-05-29 확인: Next.js authentication guide는 빠른 UI redirect와 별개로 민감한 작업은 database-backed secure authorization check를 DAL/server boundary에서 수행하라고 설명한다. Story 1.3은 page와 action 모두 `requireHeadquartersUser()`를 호출한다. [External: `https://nextjs.org/docs/app/guides/authentication`]
- 2026-05-29 확인: Prisma transactions guarantee grouped read/write operations succeed or fail as a whole. Story 1.3 store update and audit write should be one transaction. [External: `https://www.prisma.io/docs/orm/prisma-client/queries/transactions`]
