# Story 2.1: 지점장이 오늘 장부를 열고 매출/결제를 저장한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 지점장,
I want 오늘 장부를 열고 매출/결제 정보를 저장하고 싶다,
so that 하루 장부 입력을 중복 없이 시작하고 이어서 작성할 수 있다.

## Acceptance Criteria

1. Given 지점장이 자기 지점의 오늘 장부를 열 때, when 해당 지점+일자 장부가 없으면, then 새 일일 장부가 생성되어야 한다, and `storeId + closingDate` 기준으로 중복 장부가 생기면 안 된다.
2. Given 기존 장부가 있을 때, when 지점장이 다시 장부를 열면, then 저장된 매출/결제 값이 유지되어 표시되어야 한다.
3. Given 지점장이 총매출, 현금, 카드, 기타 결제수단 금액을 입력할 때, when 저장하면, then 값이 장부에 저장되어야 한다, and 장부 상태는 `입력중`으로 표시되어야 한다.
4. Given 결제수단 합계와 총매출이 다를 때, when 매출/결제 단계가 표시되면, then 결제 합계 차액이 계산되어 보여야 한다, and 이 경고는 저장을 막지 않아야 한다.
5. Given 지점장이 권한 없는 지점 장부 URL에 직접 접근할 때, when 서버가 요청을 처리하면, then 장부 데이터가 반환되지 않아야 한다, and "접근 권한이 없습니다." 화면이 표시되어야 한다.
6. Given 390px 모바일 화면에서 매출/결제 단계를 사용할 때, when 숫자 입력 필드가 표시되면, then 숫자 키패드가 열리도록 입력 모드가 설정되어야 한다, and 주요 터치 대상은 44x44px 이상이어야 한다.
7. Given 저장 중 오류가 발생할 때, when 저장을 시도하면, then 한국어 오류 메시지와 재시도 동작이 제공되어야 한다.
