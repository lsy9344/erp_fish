# Story 3.3: 본사가 매출/이익률/매출차액 이상 신호를 본다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 본사 사용자,
I want 관제판에서 매출, 이익률, 매출차액 이상 신호를 확인하고 싶다,
so that 회의 전에 매출 흐름이나 계산 차이가 큰 지점을 먼저 볼 수 있다.

## Acceptance Criteria

1. **Given** 본사 사용자가 관제판을 열 때
   **When** 장부의 매출, 이익률, 매출차액 데이터와 기준값이 존재하면
   **Then** 매출 급락, 이익률 급락, 매출차액 이상 여부가 서버 측 공유 이상 신호 계산으로 평가되어야 한다
   **And** UI 전용 계산으로 신호를 따로 만들면 안 된다.

2. **Given** 매출 하락률 또는 이익률 하락폭이 본사 기준값을 넘을 때
   **When** 관제판 행이 표시되면
   **Then** 해당 지점 행에 경고 등급 이상 신호 칩이 표시되어야 한다
   **And** 칩에는 아이콘과 한국어 레이블이 함께 표시되어야 한다.

3. **Given** 매출차액 금액이 본사 기준값을 넘을 때
   **When** 관제판 행이 표시되면
   **Then** 매출차액 이상 신호 칩이 표시되어야 한다
   **And** 칩 또는 상세에는 기준값과 실제 매출차액이 표시되어야 한다.

4. **Given** 한 지점에 매출 급락, 이익률 급락, 매출차액 이상이 동시에 발생할 때
   **When** 관제판 행이 표시되면
   **Then** 여러 신호 칩이 동시에 표시될 수 있어야 한다
   **And** 행 강조는 가장 높은 심각도를 기준으로 적용되어야 한다.

5. **Given** 비교 기준일 데이터가 없거나 기준 정책이 미정일 때
   **When** 매출/이익률 급락 신호를 평가하면
   **Then** 임의로 정상 처리하지 않아야 한다
   **And** "계산 기준 확인 필요" 또는 "계산 불가" 상태를 표시해야 한다.

6. **Given** 총매출이 0이거나 이익률 계산이 불가능한 장부일 때
   **When** 이상 신호를 평가하면
   **Then** 나누기 오류나 잘못된 무한대 값이 표시되면 안 된다
   **And** 계산 불가 상태와 실제 입력 상태가 구분되어야 한다.

7. **Given** 사용자가 이상 신호가 있는 관제판 행을 선택할 때
   **When** 행을 클릭하거나 키보드로 활성화하면
   **Then** 해당 지점+일자의 장부 상세로 이동하거나 Sheet가 열려야 한다
   **And** 상세에도 같은 이상 신호 요약이 표시되어야 한다.

### Signal Calculation Rules

- 기준값이 없으면 row signal은 `기준값 설정 전` info 상태를 유지한다.
- 기준값이 있어도 비교 기준일 정책이 없으면 매출 급락/이익률 급락은 정상으로 처리하지 않고 `계산 기준 확인 필요` 상태를 만든다.
- 매출 하락률은 baseline 총매출이 0보다 클 때만 계산한다. baseline이 0, null, missing이면 계산 불가다.
- 이익률 하락폭은 current/baseline gross margin이 모두 finite number일 때만 계산한다. 하나라도 null이면 계산 불가다.
- 매출차액 threshold는 `salesDifference.value`가 number일 때만 비교한다. 현재 codebase에서는 `salesDifference`가 `계산 기준 확인 필요`이므로 이 상태를 그대로 보여준다.
- `paymentDifference`는 총매출과 결제수단 합계의 차이다. PRD의 매출차액과 다르므로 Story 3.3 anomaly source로 쓰면 안 된다.
- warning signal label 예시는 `매출 급락`, `이익률 급락`, `매출차액 초과`다. detail에는 actual과 threshold가 함께 들어가야 한다.
