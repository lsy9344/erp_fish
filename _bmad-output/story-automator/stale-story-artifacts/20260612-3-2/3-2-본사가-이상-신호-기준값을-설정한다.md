# Story 3.2: 본사가 이상 신호 기준값을 설정한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 본사 사용자,
I want 이상 신호 기준값을 운영 상황에 맞게 설정하고 싶다,
so that 관제판이 본사 기준에 따라 문제 가능성이 높은 지점을 표시할 수 있다.

## Acceptance Criteria

1. **Given** 본사 사용자가 이상 신호 기준값 설정 화면에 접근할 때
   **When** 화면이 열리면
   **Then** 매출 하락률, 이익률 하락폭, 매출차액 금액, 손실액, 재고 차이 기준을 볼 수 있어야 한다
   **And** 각 기준값에는 단위와 설명이 표시되어야 한다.

2. **Given** 매출차액 기준을 설정할 때
   **When** 금액 기준을 입력하고 저장하면
   **Then** 이후 관제판 조회는 변경된 금액 기준을 사용해야 한다
   **And** 비율 기준이 필요하면 별도 Open Question으로 남겨야 하며 임의 구현하면 안 된다.

3. **Given** 매출/이익률 급락 기준을 설정할 때
   **When** 매출 하락률 또는 이익률 하락폭을 저장하면
   **Then** 이후 관제판 조회는 변경된 기준을 사용해야 한다
   **And** 비교 기준일 정책이 미정이면 "기준일 정책 확인 필요"로 표시해야 한다.

4. **Given** 손실액 또는 재고 차이 기준을 설정할 때
   **When** 기준값을 저장하면
   **Then** 이후 재고/손실 이상 신호 계산에 반영되어야 한다.

5. **Given** 기준값에 잘못된 숫자나 허용 범위 밖 값이 입력될 때
   **When** 저장을 시도하면
   **Then** 필드 하단에 한국어 검증 메시지가 표시되어야 한다
   **And** 잘못된 기준값은 저장되지 않아야 한다.

6. **Given** 기준값이 변경될 때
   **When** 저장이 완료되면
   **Then** actor, timestamp, target, before, after를 포함하는 감사 기록이 남아야 한다
   **And** 다음 관제판 조회부터 변경된 기준이 적용되어야 한다.

7. **Given** 지점장 또는 권한 없는 사용자가 기준값 설정 화면에 접근할 때
   **When** 서버가 요청을 처리하면
   **Then** 기준값 데이터가 반환되지 않아야 한다
   **And** 사용자는 "접근 권한이 없습니다." 화면을 봐야 한다.

### Validation Rules

- 비율 필드 `매출 하락률`, `이익률 하락폭`: 빈 값은 오류. 숫자 문자열만 허용. 표시 단위는 `%`와 `%p`. 서버 저장은 bps integer. 허용 범위는 `0` 이상 `10000` bps 이하(0.0%~100.0%)다.
- 금액 필드 `매출차액 금액`, `손실액`: 빈 값은 오류. 콤마는 입력 편의로 제거할 수 있지만 서버 저장은 integer KRW. 허용 범위는 `0` 이상 `2_147_483_647` 이하다.
- 수량 필드 `재고 차이 기준`: 빈 값은 오류. 정수만 허용. 허용 범위는 `0` 이상 `2_147_483_647` 이하다.
- 오류 메시지는 한국어로 필드별 표시한다. 예: "매출 하락률은 0.0% 이상 100.0% 이하로 입력해 주세요.", "손실액은 0원 이상의 정수여야 합니다."
- 클라이언트 검증은 편의를 위한 것이고 서버 schema가 최종 기준이다.

### Latest Technical Information

- Next.js 15 `revalidatePath`는 Server Functions와 Route Handlers에서 호출할 수 있고 Client Component나 Middleware에서는 사용할 수 없다. 기준값 저장 action은 Server Action이므로 저장 후 `/app/master-data/anomaly-thresholds`와 `/app/dashboard`를 revalidate한다. [External: `https://nextjs.org/docs/15/app/api-reference/functions/revalidatePath`]
- Prisma Client는 필요한 필드만 `select`할 수 있고, `upsert`로 단일 설정 row를 생성 또는 갱신할 수 있다. threshold query는 audit 전체나 ledger 상세를 같이 읽지 않는다. [External: `https://www.prisma.io/docs/orm/prisma-client/queries/crud`]
- Zod는 coercion과 schema validation을 지원한다. 이 프로젝트의 기존 code schema처럼 문자열 입력을 서버 schema에서 숫자 payload로 정규화하고 field error를 반환한다. [External: `https://zod.dev/api`]
