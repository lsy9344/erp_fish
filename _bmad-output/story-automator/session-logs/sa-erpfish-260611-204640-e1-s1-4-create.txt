+# Story 1.4: 서버 권한 헬퍼와 민감 필드 응답 차단
+
+Status: ready-for-dev
+
+## Story
+
+As a 개발자,
+I want 권한 확인과 민감 필드 차단을 공통 서버 헬퍼로 사용할 수 있기를 원한다,
+so that 이후 장부, 리포트, export 기능이 각자 다른 방식으로 보안을 구현하지 않는다.
+
+## Acceptance Criteria
+
+1. Given 서버 코드가 사용자 권한을 확인해야 할 때, when 기능 코드가 공통 authorization helper를 호출한다, then helper는 로그인 여부, 권한 프로파일, 지점 접근 범위, action 권한을 함께 확인해야 한다, a
nd 권한 없는 요청은 업무 로직 실행 전에 중단되어야 한다.
+2. Given 본사 action 권한이 필요한 요청이 있을 때, when close, correction, upload, settings, report/export 같은 action을 검사한다, then 각 action은 명시된 permission key 또는 동등한 구조로 확인되어야
 한다, and 단순히 `본사 사용자`라는 큰 역할만으로 모든 action을 허용하면 안 된다.
+3. Given 지점장 경로에서 장부나 리포트 데이터를 조회할 때, when 서버가 응답 데이터를 만든다, then 매출원가, 매출이익, 영업이익, 이익률, 인당생산성, 재고금액, lot 근거, 본사 고정비, 타 지점 비교 같은
민감 필드는 응답에서 제거되어야 한다, and 클라이언트에서 숨기는 방식에 의존하면 안 된다.
+4. Given 본사 조회 전용 권한이 있는 사용자가 리포트나 export를 요청할 때, when 서버가 응답 필드와 대상 지점을 결정한다, then 허용된 지점/기간/컬럼만 포함해야 한다, and 권한 밖 지점이나 권한 없는 민감
 컬럼은 반환하지 않아야 한다.
+5. Given 새로운 feature action이 추가될 때, when 개발자가 권한 검사를 구현한다, then 기존 공통 helper 패턴을 재사용해야 한다, and one-off role check를 만들면 코드 리뷰에서 차단할 수 있어야 한다.
