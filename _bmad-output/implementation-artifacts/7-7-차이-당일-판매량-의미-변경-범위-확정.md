# Story 7.7: `차이` → `당일 판매량` 의미 변경 범위 확정

Status: done

## Story

As a PM과 본사 운영자,
I want 기존 재고 테이블의 `차이`를 `당일 판매량`으로 바꾸는 의미를 확정하고 싶다,
so that 단순 라벨 변경인지 계산 의미 변경인지 혼동 없이 구현할 수 있다.

## Acceptance Criteria

1. **Given** OQ-14가 열려 있을 때, **When** 이 discovery story를 수행한다, **Then** `차이`를 `당일 판매량`으로 바꾸는 것이 단순 문구 변경인지, 계산 의미 변경인지 결정해야 한다, **And** 결정 근거를 예시 재고 행으로 설명해야 한다.
2. **Given** 단순 라벨 변경으로 결정될 때, **When** 정책 산출물을 작성한다, **Then** 변경 대상 화면, 리포트, export 컬럼명을 명시해야 한다, **And** 계산식은 바꾸지 않는다고 기록해야 한다.
3. **Given** 계산 의미 변경으로 결정될 때, **When** 정책 산출물을 작성한다, **Then** 재고 흐름, 리포트, 이상 신호, 손실/조정 반영 방식에 미치는 영향을 정의해야 한다, **And** 관련 구현 스토리는 계산 정책 승인 후에만 생성해야 한다.
4. **Given** 지점장 화면의 이해도를 개선하려 할 때, **When** 문구를 확정한다, **Then** 운영자에게 보일 설명 문구와 도움말을 함께 정해야 한다, **And** 본사 내부 계산 용어와 화면 표시 용어가 다를 경우 매핑표를 포함해야 한다.
5. **Given** 정책 산출물이 완료되었을 때, **When** PM과 본사 운영자가 승인한다, **Then** 산출물에는 라벨/계산 의미 결정, 영향 범위, 예시, 표시 문구, 승인자가 포함되어야 한다, **And** MVP-S10 또는 후속 구현 스토리로 승격할 수 있는지 명시해야 한다.
