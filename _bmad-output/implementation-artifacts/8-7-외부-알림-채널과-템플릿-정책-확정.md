# Story 8.7: 외부 알림 채널과 템플릿 정책 확정

Status: done

## Story

As a PM과 본사 운영자,
I want LINE 또는 텔레그램 알림의 채널, 템플릿, 발송 기준을 확정하고 싶다,
so that CAP-11이 과다 발송이나 민감 정보 노출 없이 운영 위험을 알려줄 수 있다.

## Acceptance Criteria

1. **Given** CAP-11이 외부 알림을 요구할 때, **When** 이 discovery story를 수행한다, **Then** 우선 채널, 계정 준비, 발송 주체, 테스트 발송 방식, 실패/재시도 기준을 정의해야 한다, **And** OQ-13이 닫히기 전에는 채널 연동 구현 스토리를 만들면 안 된다.
2. **Given** 장기 체화 재고 알림이 필요할 때, **When** 알림 조건을 정한다, **Then** 1주, 1개월, 품목별 기준 중 어떤 기준을 사용할지 결정해야 한다, **And** OQ-16이 닫히기 전에는 장기 체화 재고 자동 알림을 구현하면 안 된다.
3. **Given** 알림 템플릿이 민감 지표를 포함할 수 있을 때, **When** 템플릿 정책을 작성한다, **Then** 지점장 수신 가능 필드와 본사 전용 필드를 분리하고, 링크 접근 권한과 감사 로그 기준을 포함해야 한다, **And** 알림 템플릿 예외 정책은 OQ-10B의 영향을 받아야 한다.
4. **Given** 산출물이 완료되었을 때, **When** PM과 본사 운영자가 승인한다, **Then** 산출물에는 채널 결정, 알림 조건표, 템플릿 allowed/blocked field matrix, 승격 가능 여부가 포함되어야 한다.
