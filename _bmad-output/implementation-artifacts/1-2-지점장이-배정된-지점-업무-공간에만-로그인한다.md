# Story 1.2: 지점장이 배정된 지점 업무 공간에만 로그인한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 지점장,<br>
I want 로그인 후 내가 배정된 지점의 업무 공간에만 접근하고 싶다,<br>
so that 다른 지점의 장부나 기준정보가 노출되지 않는다.

## Acceptance Criteria

1. Given 지점장 계정이 하나 이상의 활성 지점에 배정되어 있을 때, when 지점장이 올바른 인증 정보로 로그인하면, then 지점장용 업무 화면으로 진입해야 한다, and 기본 진입점은 자기 지점의 오늘 장부 시작 화면이어야 한다.
2. Given 지점장이 로그인했을 때, when 내비게이션이 표시되면, then 모바일에서는 `장부`, `재고`, `손실` 하단 탭이 보여야 한다, and 데스크탑에서는 지점장에게 필요한 업무 항목만 보여야 한다.
3. Given 지점장이 자기 지점이 아닌 장부나 업무 경로에 직접 URL로 접근할 때, when 서버가 요청을 처리하면, then 해당 데이터를 반환하지 않아야 한다, and 사용자는 "접근 권한이 없습니다." 화면을 봐야 한다.
4. Given 지점장 권한 검사가 수행될 때, when 장부, 재고, 손실 화면의 데이터를 조회하면, then 서버는 세션의 사용자 역할과 지점 배정을 기준으로 접근을 허용하거나 거부해야 한다, and 클라이언트에서 숨긴 메뉴만을 보안 경계로 삼으면 안 된다.
5. Given 지점장이 로그인 후 자기 지점 업무 화면을 볼 때, when 화면이 390px 폭 모바일에서 렌더링되면, then 핵심 내비게이션과 화면 구조가 사용할 수 있어야 한다, and 터치 대상은 최소 44x44px 기준을 따라야 한다.
6. Given 지점장 계정에 배정된 지점이 없거나 비활성 지점만 배정되어 있을 때, when 로그인 후 업무 화면에 접근하면, then 장부 데이터가 노출되지 않아야 한다, and 본사에 문의하라는 간결한 한국어 안내가 표시되어야 한다.
