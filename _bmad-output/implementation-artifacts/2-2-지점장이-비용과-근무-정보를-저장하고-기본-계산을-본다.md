# Story 2.2: 지점장이 비용과 근무 정보를 저장하고 기본 계산을 본다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 지점장,
I want 비용 항목과 근무인원/특이사항을 저장하고 기본 계산값을 보고 싶다,
so that 매출 이후 장부의 운영 비용과 인당생산성을 이어서 확인할 수 있다.

## Acceptance Criteria

1. **Given** 지점장이 입력중 장부의 비용 단계에 접근할 때
**When** 비용 항목, 금액, 메모를 입력하고 저장하면
**Then** 하나의 장부에 여러 비용 항목이 저장되어야 한다
**And** 비용 항목은 active 비용 코드 목록에서 선택할 수 있어야 한다.

2. **Given** 비용 항목이 여러 개 저장되어 있을 때
**When** 비용 단계가 다시 열리면
**Then** 저장된 비용 항목과 비용 합계가 표시되어야 한다.

3. **Given** 지점장이 근무인원/특이사항 단계에 접근할 때
**When** 근무인원과 특이사항 메모를 입력하고 저장하면
**Then** 값이 장부에 저장되어야 한다
**And** 근무인원은 인당생산성 계산에 사용할 수 있어야 한다.

4. **Given** 총매출과 비용 합계가 저장되어 있을 때
**When** 비용 또는 근무 단계가 표시되면
**Then** 현재 입력값 기준 비용 합계와 기본 영업이익이 표시되어야 한다.

5. **Given** 총매출과 근무인원이 저장되어 있을 때
**When** 근무 단계가 표시되면
**Then** 인당생산성이 계산되어 표시되어야 한다
**And** 근무인원이 0이거나 비어 있으면 나누기 오류 대신 "계산 불가" 상태가 표시되어야 한다.

6. **Given** 비용 금액이나 근무인원에 잘못된 값이 입력될 때
**When** 저장을 시도하면
**Then** 필드 하단에 한국어 검증 메시지가 표시되어야 한다
**And** 첫 번째 오류 필드로 포커스가 이동해야 한다.

7. **Given** 390px 모바일 화면에서 비용 또는 근무 단계를 사용할 때
**When** 숫자 입력 필드가 표시되면
**Then** 금액과 인원 입력에 맞는 숫자 입력 모드가 적용되어야 한다
**And** 주요 버튼과 행 추가 컨트롤은 터치 가능한 크기여야 한다.

### Architecture Guardrails

- 스택/패턴은 Create T3 App + Next.js Server Component + Server Actions + Prisma + shadcn/ui 그대로 유지.
- Mutation은 공통 `ActionResult`, `requireStoreAccess`, `writeAuditLog`, `db.$transaction`, `revalidatePath` 가드라인을 따른다.
- 입력 코드 재사용: 비용 코드/비용 항목 선택은 `getActiveLedgerInputCodeOptions("EXPENSE_ITEM")` 사용.
- 금액은 integer KRW로 저장하고 문자열/포맷 문자열 그대로 저장하지 않는다.

### Current Repository State

- `prisma/schema.prisma`는 현재 `DailyLedger`에 매출/결제 필드, 상태, `createdBy/updatedBy` relation만 존재.

### Data Model Guardrails

- 비용 항목은 장부와 N:1 관계로 저장해 항목 수를 자유롭게 확장 가능해야 함.
- 비용 항목은 활성 코드 목록에서 선택 가능해야 하며, 코드 비활성화 후에도 과거 항목 표시 보존 정책을 고려한다.
- `근무인원`은 0과 null/빈 값에 대한 계산 정책을 명시적으로 처리한다.
- 고유 제약은 중복 장부(지점+일자) 생성 방지 원칙을 유지하고, 이번 스토리에서는 기존 `DailyLedger` 유일성 정책을 변경하지 않는다.

### UX Guardrails
