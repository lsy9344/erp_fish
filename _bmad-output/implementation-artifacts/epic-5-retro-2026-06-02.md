## 1) Epic Review

### Epic Summary
Epic 5은 본사 운영 리포트 기능을 일별 회의 리포트, 숫자 근거 추적, 기간 비교, 월간 마감/이상, 월간 손실·재고 흐름으로 확장했다. 총 5개 Story(5.1~5.5)가 완료되어 스코프가 정리되었다.

### 성공 포인트
- 핵심 수치가 **관제판/장부 상세와 동일한 shared server calculation**을 재사용해 화면별 계산 분기 위험을 줄였다.
- 일별→기간 비교→월간 리포트로 기능이 누적되었고, 각 단계가 이전 구현(5.1/5.2)에서 만든 데이터 모델을 재활용해 확장되었다.
- 액세스, 날짜/필터 상태, 링크 재검증 등 운영 플로우를 통합해 일관된 UX를 확보했다.
- 단위/통합 테스트와 e2e 커버리지가 각 Story에서 누적되며 회귀 리스크를 적극 관리했다.
- 각 Story의 [Review][Patch] 항목이 완료되어 출시 전 품질 정리를 수행했다.

### 회고 대상 핵심 이슈 (근거 기반)
- `5.1`의 날짜 선택·미입력/휴무 구분·정정 반영 값 처리에서 기준 불일치 이슈가 발견되어 수정됨.
- `5.2`에서 정정 근거 표현 로직이 여러 지표/상태로 오염되거나 링크 경로가 고정되는 문제를 해결함.
- `5.3`에서 기간 비교 파라미터 유효성·evidence 링크·누락 상태 반영 처리 개선.
- `5.4`에서 월간 이상 항목 표시 정확성(지점 맥락, 아이콘/카드, 빈 상태 메시지)을 보완함.
- `5.5`에서 월간 손실/재고 흐름의 근거 링크 및 계산 불가 처리의 정합성 강화.

### 완료된 Story 요약
- **5.1 본사가 일별 아침 회의 리포트를 본다**: 화면 라우팅, 로딩/빈 상태, 상태/신호 배지, 정정 반영 기반 집계 구현.
- **5.2 본사가 리포트 숫자의 근거와 정정 차이를 추적한다**: 원본/정정 값 분리 증빙, 모바일 근거 노출, 정정 타임라인 연결.
- **5.3 본사가 선택 기간의 지점별 실적을 비교한다**: 기간 URL 상태 유지, 지표 집계, 링크 유효성 보강.
- **5.4 본사가 월간 지점별 마감과 이상 현황을 본다**: 월간 상태/이상 화면, 월 필터, 재검증 범위 확장.
- **5.5 본사가 월간 손실/재고 흐름과 핵심 성과를 본다**: 월간 KPI·손실 유형 집계·재고 흐름 증빙 강화.

## 2) 성과와 개선점

### 무엇이 잘 되었는가
1. 계산/표시 계약을 중앙 집권화해 화면별 계산 편차를 크게 줄였다.
2. 정정 로직을 증빙 가능한 형태로 확장해 “왜 이 숫자인가”를 추적 가능하게 만들었다.
3. 상태(미입력/휴무/입력중/검토대기/본사마감) 구분과 빈 상태 메시지를 운영 관점으로 정교하게 다듬었다.
4. 테스트 케이스를 점진적으로 누적해, 회귀 조건(권한·경로·필터·데이터 예외)을 반복 검증했다.

### 개선해야 할 점
1. 화면 증빙(근거)이 많은 만큼, story 간 공통 타입/헬퍼 계약의 사소한 변경이 잦아 유지비가 발생함.
2. 날짜/필터 기반 조회는 작은 파라미터 오차가 큰 범위 조회로 번질 수 있어, 입력 유효성 장치가 더 강해야 함.
3. 휴무/미입력/계산 불가 분기에서 사용자 메시지 규칙이 story 간 분산되면 회귀 위험이 커짐.
4. e2e 테스트 재실행 시 테스트 환경(포트, 상태 정리)에 민감한 특성이 있어 안정화가 필요.
