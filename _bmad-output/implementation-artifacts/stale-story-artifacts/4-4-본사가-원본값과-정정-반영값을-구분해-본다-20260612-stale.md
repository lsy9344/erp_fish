# Story 4.4: 본사가 원본값과 정정 반영값을 구분해 본다

Status: done

## Story

As a 본사 사용자,
I want 마감된 장부 상세에서 원본값과 정정 반영값을 구분해 보고 싶다,
so that 회의 숫자는 최신 운영값으로 보면서도 원본 감사 추적을 잃지 않을 수 있다.

## Acceptance Criteria

1. **Given** 본사 사용자가 정정 기록이 있는 마감 장부 상세를 열 때
   **When** 대상 항목이 표시되면
   **Then** 원본값과 정정 반영값이 구분되어 표시되어야 한다
   **And** 정정 반영값에는 "정정 반영" 레이블이 표시되어야 한다.

2. **Given** 사용자가 정정 기록 패널을 열 때
   **When** 정정 이력을 확인하면
   **Then** 대상 항목, 원본값, 정정값, 사유, 작성자, 작성 시각이 시간순 또는 최신순 타임라인으로 표시되어야 한다.

3. **Given** 장부 상세가 Sheet로 열려 있는 상태일 때
   **When** 사용자가 정정 기록 패널을 열면
   **Then** Sheet 위에 또 다른 Sheet를 중첩하지 않아야 한다
   **And** 상세 안 확장 영역, 하단 패널, 또는 별도 페이지 중 하나로 표시해야 한다.

4. **Given** 원본 입력 필드가 마감 상태에서 표시될 때
   **When** 사용자가 장부 상세를 보면
   **Then** 원본 입력 필드는 disabled 상태여야 한다
   **And** 정정 기록 추가가 가능한 항목에는 명확한 정정 액션이 제공되어야 한다.

5. **Given** 같은 대상 항목에 여러 정정 기록이 있을 때
   **When** 사용자가 상세를 확인하면
   **Then** 최신 정정값이 현재 정정 반영값으로 표시되어야 한다
   **And** 이전 정정값도 타임라인에서 확인할 수 있어야 한다.

6. **Given** 정정 반영값과 원본값이 모두 표시될 때
   **When** 스크린 리더나 키보드 사용자가 탐색하면
   **Then** 두 값의 의미가 텍스트 레이블로 구분되어야 한다
   **And** 색상이나 취소선만으로 차이를 전달하면 안 된다.

7. **Given** 정정 기록이 없는 마감 장부를 볼 때
   **When** 정정 패널을 열면
   **Then** 정정 기록이 없다는 빈 상태 메시지를 표시해야 한다
   **And** 본사 사용자는 새 정정 기록 추가 동작을 시작할 수 있어야 한다.

## Dev Notes

### 핵심 목표

- Story 4.4는 정정 기록 생성 기능을 다시 만드는 story가 아니다. Story 4.3에서 만든 append-only `CorrectionRecord`, 최신값 helper, `CorrectionPanel`을 재사용해 상세 화면에서 원본값과 현재 정정 반영값을 구분해 보여주는 story다.
- 원본 장부 값은 계속 감사 기준 원본이다. 상세 화면이 정정 반영값을 보여주더라도 원본 DB row를 덮어쓰면 안 된다.
- 관제판 기본 숫자를 정정 반영값으로 바꾸는 일은 Story 4.5 범위다. 이번 story는 장부 상세과 정정 패널의 표시 구분에 집중한다.

### 현재 구현 상태

- `CorrectionRecord` 모델은 이미 있다. 필드는 `dailyLedgerId`, `targetType`, `targetId`, `fieldKey`, `originalValue`, `previousAppliedValue`, `correctedValue`, `reason`, `createdById`, `createdAt`이다.
- `src/features/corrections/actions.ts`는 본사 권한, `HEADQUARTERS_CLOSED` 상태, target별 original value resolve, shape 검증, advisory lock, serializable transaction, `correction.created` 감사 로그를 처리한다.
- `src/features/corrections/queries.ts`에는 `getLatestCorrectionByTargetInTx`, `getCorrectionRecordsForLedgerInTx`, `getCorrectionRecordsForLedger`, `buildCorrectionTargetKey`가 있다.
- `src/features/corrections/components/correction-panel.tsx`는 현재 정정 입력 폼과 기록 표를 제공한다. 기록 표는 `대상 / 이전값 / 정정값 / 사유 / 작성` 컬럼을 사용한다.
- `src/app/app/ledgers/[ledgerId]/page.tsx`는 `HEADQUARTERS_CLOSED`일 때 `CorrectionPanel`을 렌더링하고, 주요 숫자 `MetricCard`는 아직 원본/정정 반영 구분 없이 `detail` 계산값만 표시한다.
- `src/features/dashboard/queries.ts`의 `getHqLedgerDetail`과 `getHqDashboardRows`는 아직 원본 `DailyLedger` 값으로 `calculateLedgerReviewSummary`를 호출한다. Story 4.4에서는 상세 표시용 정정 구분만 추가하고, dashboard row 기본 계산 변경은 Story 4.5로 남긴다.

### 구현 가드레일

1. 정정 표시를 위해 새로운 테이블이나 별도 수정 모델을 만들지 않는다. 기존 `CorrectionRecord`를 읽는다.
2. 최신 정정값 선택 규칙은 Story 4.3과 동일하게 `createdAt desc, id desc`다.
3. UI에서 계산 규칙을 새로 만들지 않는다. 값 overlay 또는 요약 계산이 필요하면 shared server helper를 만들고 `src/server/calculations`나 `src/features/corrections`에서 재사용 가능하게 둔다.
4. "원본", "정정 반영"은 반드시 화면 텍스트로 표시한다. 색상, 취소선, 배지 색만으로 의미를 전달하지 않는다.
5. Sheet/Dialog 중첩 금지. 현재 상세 페이지 안 section 또는 확장 영역 패턴을 유지한다.
6. 정정 기록이 없는 상태도 정상 상태다. 빈 상태를 오류처럼 보이게 만들지 않는다.
7. 지점장 또는 권한 없는 사용자는 상세와 정정 데이터 조회 모두 서버 권한에서 차단되어야 한다.

### Data and Calculation Notes

- `CorrectionRecord.correctedValue`는 `{ kind, value, label }` 형태의 JSON으로 저장된다. 숫자 표시 helper는 `kind: "money" | "quantity" | "metric"`과 string label을 안전하게 처리해야 한다.
- `previousAppliedValue`는 같은 대상의 직전 최신 정정값 또는 원본값이다. 타임라인에서는 "이전 반영값"으로 유용하지만, AC의 원본값 표시는 `originalValue`를 직접 보여줘야 한다.
- 현재 `getCorrectionTargetOptions`는 결제 필드, 근무인원, 비용 amount, 매입 amount를 대상으로 만든다. 재고/손실 정정 표시는 후속 확장 여지를 남기되, 이번 story에서 대상 목록을 넓힐 때는 `actions.ts`의 target resolver와 UI label이 같은 target identity를 써야 한다.
- 주요 숫자 `매출`, `마진율`, `매출 차이`, `손실` 중 `매출`은 `totalSalesAmount` 정정으로 직접 overlay할 수 있다. `마진율`, `매출 차이`처럼 계산값인 항목은 shared calculation에서 정정 반영 입력을 받기 전까지 임의 계산하지 않는다. 계산 불가/확인 필요 상태면 그렇게 표시한다.

### UX and Accessibility Notes

- `CorrectionPanel`은 기존 일반 section이다. 이를 유지하면 AC 3의 모달 중첩 금지를 만족한다.
- `DESIGN.md`에 따라 shadcn 기본 primitive를 사용하고 radius는 업무 도구 느낌에 맞게 유지한다. pill은 배지와 신호 칩에만 쓴다.
- `EXPERIENCE.md`는 정정 기록 추가 후 수치 옆 "정정 반영" 레이블과 원본값 확인을 요구한다.
- 스크린 리더 사용자가 두 값의 의미를 알 수 있도록 "원본", "정정 반영", "이전 반영값" 같은 텍스트를 셀/카드 안에 넣는다.
- 모바일/좁은 화면에서 원본값과 정정 반영값이 겹치지 않게 줄바꿈과 `break-words`, `tabular-nums`를 유지한다.
