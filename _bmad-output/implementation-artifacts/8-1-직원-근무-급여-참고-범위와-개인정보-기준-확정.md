# Story 8.1: 직원/근무/급여 참고 범위와 개인정보 기준 확정

Status: done

## Story

As a PM과 본사 운영자,
I want 직원 마스터와 급여 참고 자료의 범위와 개인정보 기준을 확정하고 싶다,
so that CAP-1/CAP-9가 실제 급여 지급 확정 기능으로 잘못 구현되지 않게 할 수 있다.

## Acceptance Criteria

1. **Given** CAP-1과 CAP-9가 승인 확장 범위에 있을 때, **When** 이 discovery story를 수행한다, **Then** 직원 마스터, 월간 근무일수 집계, 급여 참고 자료가 포함할 필드를 정의해야 한다, **And** 실제 급여 지급 확정, 세무/노무 판단, 자동 지급 기능은 제외 범위로 표시해야 한다.
2. **Given** OQ-12가 열려 있을 때, **When** 급여 참고 자료의 목적을 정한다, **Then** 참고 자료만 제공하는지, 지급 확정까지 필요한지 결정해야 한다, **And** 지급 확정이 필요하면 별도 PRD 또는 change proposal 없이는 구현 스토리로 승격하지 않아야 한다.
3. **Given** 직원 개인정보를 저장할 가능성이 있을 때, **When** 데이터 보존표를 작성한다, **Then** 저장 필드, 보존 기간, 접근 권한, 감사 로그 대상, 삭제/비활성 처리 기준을 포함해야 한다, **And** 기존 MVP 장부의 근무인원 숫자를 특정 직원 근무 기록으로 자동 분해하지 않는 원칙을 포함해야 한다.
4. **Given** 산출물이 완료되었을 때, **When** PM과 본사 운영자가 승인한다, **Then** 산출물에는 CAP-1/CAP-9 범위표, OQ-12 결정, 개인정보 보존표, 승격 가능 여부가 포함되어야 한다, **And** 구현 스토리는 승인된 참고 범위 안에서만 생성할 수 있어야 한다.
