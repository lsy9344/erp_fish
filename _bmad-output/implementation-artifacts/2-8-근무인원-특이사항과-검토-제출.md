# Story 2.8: 근무인원/특이사항과 검토 제출

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 지점장,
I want 근무인원과 특이사항을 입력하고 장부를 검토 대기 상태로 제출할 수 있기를 원한다,
so that 본사가 회의 전 장부의 운영 맥락과 제출 상태를 확인할 수 있다.

## Acceptance Criteria

1. Given 사용자가 근무인원/특이사항 단계에 있을 때, when 근무인원과 특이사항을 입력한다, then 값은 해당 장부에 저장되어야 한다, and 근무인원은 승인된 기본 계산에서 사용할 수 있어야 한다.
2. Given 일반 직원 로그인 기능이나 직원별 근태 관리가 요구될 때, when 이 스토리를 구현한다, then 직원별 선택, 지각/조퇴 상세, 실제 급여 지급 확정 기능은 포함하지 않아야 한다, and MVP는 근무인원/특이사항 기록 수준으로 제한해야 한다.
3. Given 사용자가 검토/제출 단계에 도달했을 때, when 제출 전 검토 화면이 표시된다, then 입력된 매출, 비용, 매입, 재고, 손실, 근무 정보의 요약과 저장 상태가 보여야 한다, and 필수 누락이나 `기준 확인 필요` 상태가 있는 항목은 숨기지 않아야 한다.
4. Given 사용자가 장부를 제출할 때, when 서버 검증이 통과한다, then 장부 상태는 `검토대기`로 변경되어야 한다, and 지점장 제출은 원본 잠금이 아니며, 본사 마감 전까지 권한 있는 사용자가 수정할 수 있어야 한다.
5. Given 제출 시 필수 값이 누락되었을 때, when 서버가 제출 요청을 검증한다, then 제출은 거부되어야 한다, and 사용자는 어느 단계의 어떤 필드를 고쳐야 하는지 알 수 있어야 한다.
6. Given 장부가 제출되거나 제출 상태에서 수정될 때, when 저장 요청이 성공한다, then 제출자/수정자, 시각, 변경 전/후 상태는 감사 로그에 남아야 한다, and 장부 version 또는 edit token이 갱신되어야 한다.
