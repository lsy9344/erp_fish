# Story 1.5: 본사가 품목 마스터와 매입 기준을 관리한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 본사 사용자,
I want 품목 마스터와 매입 기준을 관리하고 싶다,
so that 지점장과 본사가 장부 입력 시 같은 기준 정보를 참조할 수 있다.

## Acceptance Criteria

1. Given 본사 사용자가 품목 마스터 화면에 접근할 때, when 화면이 열리면, then 품목 목록에는 품목명, 구분, 규격, 기본 단가, 활성/비활성 상태, 마지막 수정 시각이 표시되어야 한다, and 품목명, 구분, 활성 상태로 검색 또는 필터링할 수 있어야 한다.
2. Given 본사 사용자가 새 품목을 등록할 때, when 품목명, 구분, 규격, 기본 단가를 입력하고 저장하면, then 품목이 활성 상태로 생성되어야 한다, and 장부 입력에서 사용할 수 있는 active 품목 목록에 포함될 수 있어야 한다.
3. Given 본사 사용자가 기존 품목의 기본 정보를 수정할 때, when 변경 내용을 저장하면, then 변경 후 값이 품목 마스터에 표시되어야 한다, and 과거 장부에 이미 저장된 원본 품목명, 규격, 단가 값은 임의로 바뀌지 않아야 한다.
4. Given 본사 사용자가 품목을 비활성 처리할 때, when 비활성 상태로 저장하면, then 해당 품목은 신규 장부 입력 선택지에서 제외될 수 있어야 한다, and 과거 장부 조회와 감사 추적을 위해 품목 데이터는 삭제되지 않아야 한다.
5. Given 본사 사용자가 매입 기준 관리 화면에 접근할 때, when 매입 기준을 등록하거나 수정하면, then 품목, 기준 단가 또는 참조 정보, 활성/비활성 상태를 관리할 수 있어야 한다, and 장부 입력에서 참조할 수 있는 active 매입 기준 목록을 제공할 수 있어야 한다.
6. Given 품목 마스터 또는 매입 기준 정보가 변경될 때, when 생성, 수정, 활성 상태 변경이 완료되면, then actor, timestamp, target, before, after를 포함하는 감사 기록이 남아야 한다.
7. Given 품목 또는 매입 기준 폼에 잘못된 값이 입력될 때, when 저장을 시도하면, then 필드 하단에 한국어 검증 메시지가 표시되어야 한다, and 숫자 입력 필드는 금액 입력에 맞는 입력 모드를 사용해야 한다.
