# Story 8.2: 품목 정규화와 이카운트 업로드 계약 확정

Status: done

## Story

As a PM과 개발 리드,
I want 품목 정규화와 이카운트 업로드의 파일/매핑 계약을 확정하고 싶다,
so that CAP-5/CAP-6 구현 전에 품목명, 규격, 업로드 실패 기준이 흔들리지 않게 할 수 있다.

## Acceptance Criteria

1. **Given** CAP-5와 CAP-6이 승인 확장 범위에 있을 때, **When** 이 discovery story를 수행한다, **Then** 품목명, 구분, 규격 분리 기준과 본사 검수 흐름을 정의해야 한다, **And** OQ-3과 OQ-8이 닫히기 전에는 자동 정규화 구현 스토리를 만들면 안 된다.
2. **Given** 이카운트 샘플 파일이 있을 때, **When** 업로드 파일 계약을 작성한다, **Then** 필수 논리 필드, 실제 헤더명, 날짜/금액/수량 형식, 중복 판단 키, 누락 처리 기준을 포함해야 한다, **And** OQ-6과 OQ-15의 결정 상태를 명시해야 한다.
3. **Given** 업로드 preview/commit 흐름을 검토할 때, **When** 실패와 재처리 기준을 정한다, **Then** 매핑 실패, 필수 컬럼 누락, 중복 후보, commit 실패, reprocess 조건을 정의해야 한다, **And** commit/void/reprocess는 감사 로그 대상이어야 한다.
4. **Given** 산출물이 완료되었을 때, **When** PM과 개발 리드가 승인한다, **Then** 산출물에는 품목 정규화 기준표, 업로드 파일 계약서, preview/commit 상태표, 승격 가능 여부가 포함되어야 한다, **And** CAP-6 구현 스토리는 CAP-5 선행 관계와 업로드 감사 계약을 참조해야 한다.
