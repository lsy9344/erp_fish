# Story 2.3: 지점장이 매입 품목을 기준정보에서 선택해 입력한다

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 지점장,
I want 매입 품목을 기준정보에서 선택해 수량과 단가를 입력하고 싶다,
so that 매입 내역이 장부와 이후 재고 흐름에 반영될 수 있다.

## Acceptance Criteria

1. **Given** 지점장이 입력중 장부의 매입 단계에 접근할 때  
   **When** 화면이 열리면  
   **Then** active 품목 목록과 active 매입 기준을 선택지로 사용할 수 있어야 한다  
   **And** 비활성 품목이나 비활성 매입 기준은 신규 입력 선택지에서 제외되어야 한다.

2. **Given** 지점장이 매입 품목을 선택할 때  
   **When** 품목을 고르면  
   **Then** 품목명, 구분, 규격, 기본 단가 또는 매입 기준 참조 정보가 입력 보조값으로 표시되어야 한다  
   **And** 실제 장부 단가와 수량은 사용자가 장부 상황에 맞게 입력/조정할 수 있어야 한다.

3. **Given** 지점장이 품목, 규격, 단가, 수량을 입력할 때  
   **When** 매입 항목을 저장하면  
   **Then** 매입금액이 계산되어 장부에 저장되어야 한다  
   **And** 하나의 장부에 여러 매입 항목을 저장할 수 있어야 한다.

4. **Given** 기존 매입 항목이 저장되어 있을 때  
   **When** 매입 단계가 다시 열리면  
   **Then** 저장된 항목과 매입 합계가 표시되어야 한다.

5. **Given** 품목 마스터가 이후 변경되었을 때  
   **When** 과거 장부의 매입 항목을 조회하면  
   **Then** 장부에 저장된 원본 품목명, 규격, 단가 값이 유지되어야 한다.

6. **Given** 단가나 수량에 잘못된 값이 입력될 때  
   **When** 저장을 시도하면  
   **Then** 필드 하단에 한국어 검증 메시지가 표시되어야 한다  
   **And** 첫 번째 오류 필드로 포커스가 이동해야 한다.

7. **Given** 390px 모바일 화면에서 매입 단계를 사용할 때  
   **When** 품목 검색과 수량/단가 입력을 수행하면  
   **Then** 터치 가능한 선택지와 숫자 입력 모드가 제공되어야 한다  
   **And** 저장 후 성공 또는 실패 피드백이 표시되어야 한다.

## Dev Notes

### Source Context

- 2.1/2.2에서 `getTodayStoreLedger`, `revalidatePath`, `requireStoreAccess`, `writeAuditLog`, `inputMode="numeric"` 같은 패턴이 고정됨.
- `src/app/app/store-entry/page.tsx`는 현재 `sales|cost|work` 단계만 노출하고 있어 `purchase`가 추가될 예정인 핵심 진입점임.
- `src/features/master-data/purchase-standard-queries.ts`는 현재 본사 조회 가드(`requireHeadquartersUser`)가 걸려 있어, 지점 입력화면용 활성 목록 함수는 추가 설계 필요.

### Architecture Guardrails

- Create T3 App 스타일 유지: Server Action + Prisma + shadcn/ui + Zod 검증을 변경 없이 확장.
- 동일 장부 idempotency는 `getTodayStoreLedger` 트랜잭션 패턴을 유지.
- 감사 추적은 기존 `writeAuditLog` 중심으로 `before`/`after` payload 규격 유지.
- 저장 실패 시 기존 패턴(문구: "저장에 실패했습니다. 다시 시도해 주세요.")과 동일한 UX 라인 유지.

### Current Repository State

- 매입 단계 모델/컴포넌트가 아직 미구현 상태.
- 단계형 진입은 현재 매출/비용/근무 UI까지만 실제 입력/저장 흐름 완성됨.
- 비용/근무 입력은 이미 장부 합계/이익/인당생산성 계산과 연결된 상태.

### Data Model Guardrails

- 과거 장부의 매입 항목은 품목명/규격/단가/참조 정보를 보존(스냅샷)해야 함.
- 매입 기준이 비활성/품목이 비활성이라도 기존 장부 항목을 지우지 않음.
- 390px 모바일에서 필드 간 오입력 방지를 위해 숫자 입력은 정규화/범위 검증.
