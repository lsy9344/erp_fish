# Story 8.4: 민감 지표 고도화와 희망 판매가 손실액 정책 확정

Status: done

## Story

As a PM과 개발 리드,
I want 민감 지표 노출 고도화와 희망 판매가 손실액 정책을 확정하고 싶다,
so that CAP-13/CAP-14가 보안 기본값을 깨지 않고 구현될 수 있다.

## Acceptance Criteria

1. **Given** CAP-13 고도화가 지표별 노출 정책을 조정할 수 있을 때, **When** 이 discovery story를 수행한다, **Then** 조정 가능한 지표, 기본 차단값, 승인자, 변경 사유, 적용 시작일, 롤백 기준을 정의해야 한다, **And** OQ-10B가 닫히기 전에는 지점장 노출 허용 기능을 구현하면 안 된다.
2. **Given** CAP-14가 희망 판매가 기준 손실액 계산을 요구할 때, **When** 손실액 정책을 정한다, **Then** 희망 판매가 미입력, 영업 중 가격 변경, 손실 발생 시점, version/lock 기준을 포함해야 한다, **And** OQ-9가 닫히기 전에는 손실액 계산 구현 스토리를 만들면 안 된다.
3. **Given** 민감 지표와 손실액이 export나 API에 포함될 수 있을 때, **When** 서버 응답 계약을 작성한다, **Then** 본사 전용 응답과 지점장 응답을 분리하고, export와 캐시 응답에서도 같은 차단 기준을 적용해야 한다, **And** UI 숨김만으로는 승인 기준을 만족하지 않아야 한다.
4. **Given** 산출물이 완료되었을 때, **When** PM과 개발 리드가 승인한다, **Then** 산출물에는 CAP-13 고도화 정책 메모, CAP-14 손실액 계산 정책, 서버 차단 테스트 기준, 승격 가능 여부가 포함되어야 한다, **And** 후속 구현 스토리는 Epic 7 Story 7.6의 MVP 민감 필드 매트릭스를 깨지 않아야 한다.

[Source: `_bmad-output/planning-artifacts/architecture.md#Starter Template Evaluation`] [Source: `package.json`]
