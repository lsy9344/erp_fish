# Story 5.5: 이상 신호 기준값 설정 구조

Status: done

## Story

As a 본사 설정 관리자,  
I want 이상 신호 기준값을 저장하고 변경 이력을 남길 수 있기를 원한다,  
so that 정책이 확정된 뒤 관제판과 리포트에서 같은 기준을 적용할 수 있다.

## Acceptance Criteria

1. **Given** 설정 관리자 권한을 가진 본사 사용자가 이상 신호 기준값 화면에 접근할 때, **When** 기준값 목록을 조회한다, **Then** 기준 유형, 현재 값, 적용 범위, 활성/비활성 상태, 마지막 변경 시각을 볼 수 있어야 한다, **And** 권한 없는 사용자는 기준값 mutation을 수행할 수 없어야 한다.
2. **Given** 설정 관리자가 기준값을 추가하거나 수정할 때, **When** 값과 적용 범위가 유효하다, **Then** 기준값은 저장되어야 한다, **And** 변경 전/후 값, 변경자, 시각, 사유가 감사 로그에 남아야 한다.
3. **Given** OQ-1 매출차액/이익률 이상 기준이 아직 닫히지 않았을 때, **When** 이 스토리를 구현한다, **Then** 기준값 저장 구조는 만들 수 있지만 확정 이상 판정 로직으로 사용하면 안 된다, **And** 관제판은 관련 항목을 `기준 확인 필요`로 계속 표시해야 한다.
4. **Given** 기준값이 비활성화되었을 때, **When** 관제판이나 검토 화면이 상태를 표시한다, **Then** 비활성 기준은 확정 이상 판정에 사용되면 안 된다, **And** 필요한 경우 `기준 확인 필요` 상태로 보여줘야 한다.
5. **Given** 기준값 변경이 과거 장부에 영향을 줄 수 있을 때, **When** 설정 관리자가 변경을 저장한다, **Then** 변경 시점과 적용 범위가 기록되어야 한다, **And** 과거 장부 원본값은 변경되면 안 된다.
6. **Given** 잘못된 기준값이 입력될 때, **When** 서버가 요청을 검증한다, **Then** 저장은 거부되어야 한다, **And** UI는 어떤 값이 잘못되었는지 필드 단위로 표시해야 한다.
